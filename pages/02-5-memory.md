# 02-5. 메모리

## 1. 이 강의에서 배울 내용

이번 강의에서는 ChatGPT 의 **메모리** 기능을 살펴봅니다.

ChatGPT 는 기본적으로 대화방마다 독립적으로 작동합니다. 하지만 메모리 기능을 사용하면 사용자의 중요한 정보나 선호 사항을 기억해 두었다가, 다른 대화에서도 활용할 수 있습니다.

이 강의를 통해 다음 내용을 익힐 수 있습니다.

* ChatGPT 메모리 기능의 개념을 이해할 수 있습니다.
* 저장된 메모리와 채팅 기록 참고의 차이를 이해할 수 있습니다.
* 메모리 설정 메뉴에 접근할 수 있습니다.
* ChatGPT 에게 특정 정보를 기억하도록 요청할 수 있습니다.
* 저장된 메모리를 확인하고 삭제할 수 있습니다.
* 메모리 기능을 사용할 때 주의해야 할 개인정보와 보안 기준을 이해할 수 있습니다.

## 2. 메모리란 무엇인가

기본적으로 ChatGPT 의 각 채팅은 서로 독립적으로 동작합니다.

즉, A 채팅방에서 했던 이야기를 B 채팅방에서는 알지 못하는 것이 원칙입니다. 예를 들어 어떤 채팅방에서 “나는 누구고, 어떤 일을 하고, 어떤 방식의 답변을 좋아한다” 라고 알려줬다고 해도, 다른 채팅방에서는 그 정보를 다시 설명해야 할 수 있습니다.

이 방식은 개인정보 보호 관점에서는 자연스럽지만, 매번 같은 정보를 반복해서 알려줘야 한다는 불편함이 있습니다.

예를 들어 다음과 같은 정보를 매번 다시 설명해야 한다면 번거롭습니다.

* 나는 기업 교육 강사입니다.
* Excel, Power BI, ChatGPT 를 자주 사용합니다.
* 답변은 한국어로 받고 싶습니다.
* 실무 예시 중심의 설명을 선호합니다.
* 저는 채식주의자입니다.
* 저는 특정 음식에 알레르기가 있습니다.

메모리 기능은 이 문제를 줄여주는 기능입니다.

ChatGPT 가 사용자와의 대화에서 중요하다고 판단한 정보를 기억해 두었다가, 이후 다른 대화에서도 그 정보를 활용할 수 있게 해 줍니다.

쉽게 말하면 메모리는 ChatGPT 를 **나를 기억하는 AI 비서** 로 만드는 기능입니다.

## 3. 왜 메모리가 필요한가

ChatGPT 를 단순히 한 번 질문하고 답변받는 도구로만 사용한다면 메모리는 크게 필요하지 않을 수 있습니다.

하지만 ChatGPT 를 업무 보조 도구, 학습 파트너, 글쓰기 도우미, 데이터 분석 보조자처럼 지속적으로 사용한다면 메모리의 중요성이 커집니다.

메모리가 없으면 매번 같은 정보를 반복해야 합니다.

예를 들어 보고서 작성을 요청할 때마다 다음처럼 설명해야 할 수 있습니다.

* 나는 제조업 기획팀에서 일합니다.
* 임원 보고용 문서를 자주 작성합니다.
* 답변은 표와 목록 중심으로 정리해 주세요.
* 너무 친근한 표현보다 정제된 비즈니스 문체를 선호합니다.

반대로 메모리가 적절히 작동하면, ChatGPT 는 이런 정보를 바탕으로 더 개인화된 답변을 제공할 수 있습니다.

즉, 메모리의 목적은 단순히 정보를 저장하는 것이 아닙니다.

핵심 목적은 **반복 설명을 줄이고, 사용자의 맥락에 맞는 답변을 제공하는 것** 입니다.

## 4. 저장된 메모리와 채팅 기록 참고

ChatGPT 의 개인화 기능을 이해할 때는 두 가지를 구분해야 합니다.

| 구분       | 설명                                 | 예시                              |
| -------- | ---------------------------------- | ------------------------------- |
| 저장된 메모리  | ChatGPT 가 중요하다고 판단해 별도로 저장한 사용자 정보 | “사용자는 Excel 과 Power BI 강의를 진행함” |
| 채팅 기록 참고 | 과거 대화의 흐름을 참고해 답변에 활용하는 기능         | 예전에 나눈 프로젝트 대화를 바탕으로 후속 답변 생성   |

저장된 메모리는 일종의 메모장처럼 별도로 관리되는 정보입니다. 사용자가 “기억해 줘” 라고 요청하거나, ChatGPT 가 중요하다고 판단한 정보를 저장할 수 있습니다.

채팅 기록 참고는 과거 대화의 내용을 답변에 활용하는 방식입니다. 특정 정보가 저장된 메모리로 등록되어 있지 않더라도, 과거 대화의 맥락이 답변에 반영될 수 있습니다.

둘 다 개인화를 위한 기능이지만, 관리 방식은 다릅니다.

저장된 메모리는 사용자가 직접 확인하고 삭제할 수 있습니다. 반면 채팅 기록 참고는 과거 대화의 흐름을 바탕으로 답변이 개인화되는 방식에 가깝습니다.

## 5. 메모리 설정 위치

메모리 기능이 켜져 있는지 확인하려면 개인 맞춤 설정 화면으로 이동합니다.

기본 경로는 다음과 같습니다.

1. ChatGPT 화면 좌측 하단의 계정 아이콘을 클릭합니다.
2. `개인 맞춤 설정` 을 선택합니다.
3. 화면 아래쪽의 `메모리` 항목을 확인합니다.
4. 또는 `설정` 메뉴로 들어간 뒤 `개인 맞춤 설정` 항목에서 메모리 설정을 확인합니다.

개인 맞춤 설정 화면에서는 다음과 같은 메모리 관련 설정을 확인할 수 있습니다.

* 저장된 메모리 참고
* 채팅 기록 참고
* 저장된 메모리 관리

설정 화면의 이름이나 위치는 ChatGPT 업데이트와 사용 플랜에 따라 조금 달라질 수 있습니다. 중요한 것은 `개인 맞춤 설정` 영역에서 메모리 관련 설정을 확인한다는 점입니다.

## 6. 메모리 저장 실습

이제 메모리 기능을 직접 확인해 보겠습니다.

새 채팅을 열고 아래 문장을 입력합니다.

<pre><code>나는 채식주의자입니다. 반드시 기억해 주세요.</code></pre>

이 문장은 ChatGPT 가 “채식주의자” 라는 사용자 정보를 메모리에 저장하도록 유도하는 테스트용 프롬프트입니다.

프롬프트를 전송하면 답변 상단 또는 근처에 다음과 같은 안내가 표시될 수 있습니다.

<pre><code>메모리 업데이트됨</code></pre>

이 문구가 표시되면 ChatGPT 가 해당 정보를 메모리에 저장했다는 뜻입니다.

중요한 점은 ChatGPT 가 사용자의 말을 그대로 저장하는 것이 아니라, 의미를 해석해 다른 문장으로 저장할 수 있다는 것입니다.

예를 들어 사용자가 다음처럼 말해도,

<pre><code>나는 채식주의자입니다. 반드시 기억해 주세요.</code></pre>

저장된 메모리에는 다음처럼 표현될 수 있습니다.

<pre><code>사용자는 채식주의자입니다.</code></pre>

즉, 메모리는 사용자의 문장을 그대로 복사해 두는 기능이라기보다, ChatGPT 가 이후 대화에서 참고할 수 있도록 의미를 정리해 저장하는 기능입니다.

## 7. 메모리 저장 시 알아둘 점

메모리는 사용자가 설정 화면에서 직접 문장을 타이핑해 강제로 넣는 방식과는 다릅니다.

기본적으로는 대화 중에 사용자가 알려준 정보나, 사용자가 명시적으로 기억해 달라고 요청한 정보를 ChatGPT 가 판단해 저장합니다.

따라서 다음과 같은 표현을 사용할 수 있습니다.

<pre><code>이 내용을 기억해 주세요.</code></pre>

<pre><code>앞으로 이 점을 참고해 주세요.</code></pre>

<pre><code>나는 기업 교육 강사입니다. 이 정보를 기억해 주세요.</code></pre>

<pre><code>나는 답변을 표와 단계별 목록으로 받는 것을 선호합니다. 기억해 주세요.</code></pre>

다만 모든 정보가 항상 저장되는 것은 아닙니다. ChatGPT 는 어떤 정보가 이후 대화에 도움이 될지 판단해 일부 정보를 메모리에 저장합니다.

따라서 메모리를 저장했다면 반드시 관리 화면에서 실제로 어떤 내용이 저장되었는지 확인하는 것이 좋습니다.

## 8. 메모리 확인하기

저장된 메모리는 설정 화면에서 확인할 수 있습니다.

기본 경로는 다음과 같습니다.

1. 좌측 하단의 계정 아이콘을 클릭합니다.
2. `개인 맞춤 설정` 으로 들어갑니다.
3. `메모리` 항목을 찾습니다.
4. `관리` 버튼을 선택합니다.
5. 현재 저장된 메모리 목록을 확인합니다.

또는 ChatGPT 에게 직접 물어볼 수도 있습니다.

<pre><code>나에 대해 무엇을 기억하고 있어?</code></pre>

이 질문을 입력하면 ChatGPT 가 현재 자신이 참고하고 있는 사용자 관련 정보를 요약해서 알려줄 수 있습니다.

다만 이 답변은 설정 화면의 메모리 목록과 완전히 같은 형식으로 보이지 않을 수 있습니다. 정확한 관리를 위해서는 설정 화면의 메모리 관리 메뉴를 함께 확인하는 것이 좋습니다.

## 9. 메모리 삭제하기

메모리는 저장할 수도 있지만, 필요하면 삭제할 수도 있습니다.

예를 들어 강의 실습을 위해 “나는 채식주의자입니다” 라고 저장했지만 실제로는 채식주의자가 아니라면, 이 메모리는 삭제해야 합니다.

잘못된 메모리를 그대로 두면 이후 답변에 계속 영향을 줄 수 있습니다.

예를 들어 맛집 추천을 요청했을 때, ChatGPT 가 계속 채식주의자 기준으로 식당을 추천할 수 있습니다.

메모리 삭제 절차는 다음과 같습니다.

1. `개인 맞춤 설정` 으로 이동합니다.
2. `메모리` 항목에서 `관리` 를 선택합니다.
3. 저장된 메모리 목록을 확인합니다.
4. 삭제하려는 메모리 옆의 옵션 버튼을 선택합니다.
5. `삭제` 를 선택합니다.

삭제 후에는 해당 정보가 이후 답변에 반영되지 않는지 확인해 보는 것이 좋습니다.

## 10. 메모리가 답변에 미치는 영향

메모리가 있으면 ChatGPT 의 답변은 더 개인화됩니다.

예를 들어 ChatGPT 가 사용자를 채식주의자로 기억하고 있다면, 사용자가 다음처럼 질문했을 때 답변이 달라질 수 있습니다.

<pre><code>점심 메뉴를 추천해 주세요.</code></pre>

메모리에 “채식주의자” 정보가 있을 때는 채식 식단을 고려한 메뉴를 추천할 가능성이 높습니다.

반면 해당 메모리가 없다면 일반적인 점심 메뉴를 추천할 수 있습니다.

이처럼 메모리는 다음과 같은 방식으로 답변에 영향을 줍니다.

* 음식 추천에서 식성 반영
* 업무 조언에서 직무와 산업 반영
* 글쓰기 요청에서 선호 문체 반영
* 학습 계획에서 관심 분야 반영
* 도구 추천에서 자주 사용하는 프로그램 반영

메모리가 잘 관리되면 ChatGPT 는 점점 더 나에게 맞는 답변을 제공할 수 있습니다.

반대로 잘못된 메모리가 저장되어 있으면, 답변도 잘못된 방향으로 개인화될 수 있습니다.

## 11. 메모리와 맞춤 설정의 차이

이전 강의에서 맞춤 설정을 다루었습니다. 맞춤 설정과 메모리는 모두 개인화를 위한 기능이지만, 역할이 다릅니다.

| 구분    | 맞춤 설정             | 메모리                             |
| ----- | ----------------- | ------------------------------- |
| 입력 방식 | 사용자가 설정 화면에 직접 입력 | 대화 중 ChatGPT 가 저장하거나 사용자가 기억 요청 |
| 성격    | 기본 응답 규칙          | 장기 기억                           |
| 예시    | “답변은 한국어로 해줘”     | “사용자는 Excel 강의를 진행함”            |
| 관리 방식 | 설정 화면에서 직접 수정     | 메모리 관리 화면에서 확인·삭제               |
| 목적    | 기본 답변 방식 설정       | 사용자 맥락 유지                       |

맞춤 설정은 ChatGPT 가 기본적으로 어떻게 답변해야 하는지를 정하는 데 적합합니다.

예를 들어 다음과 같은 내용은 맞춤 설정에 적합합니다.

* 한국어로 답변해 주세요.
* 답변은 표와 목록을 활용해 주세요.
* 업무 문서에 맞게 정제된 표현을 사용해 주세요.
* 초보자도 이해할 수 있게 설명해 주세요.

반면 다음과 같은 내용은 메모리에 적합할 수 있습니다.

* 사용자는 제조업 기획팀에서 일함
* 사용자는 Power BI 강의를 자주 진행함
* 사용자는 채식주의자임
* 사용자는 보고서 초안을 자주 작성함

둘을 함께 사용하면 ChatGPT 를 더 안정적으로 개인화할 수 있습니다.

## 12. 실무에서 사용할 때의 주의사항

메모리는 편리한 기능이지만, 업무에서 사용할 때는 주의가 필요합니다.

### 12.1 민감정보는 기억시키지 않습니다

다음과 같은 정보는 메모리에 저장하지 않는 것이 좋습니다.

* 비밀번호
* API 키
* 주민등록번호
* 계좌번호
* 신용카드 정보
* 고객 개인정보
* 회사 기밀
* 내부 시스템 접속 정보
* 비공개 계약 조건
* 공개되지 않은 인사 정보

메모리는 개인화를 위한 기능이지, 민감정보 저장소가 아닙니다.

### 12.2 잘못된 메모리는 삭제합니다

잘못 저장된 메모리는 답변 품질을 떨어뜨릴 수 있습니다.

예를 들어 테스트용으로 입력한 정보가 실제 정보처럼 저장되면 이후 답변이 엉뚱한 방향으로 개인화될 수 있습니다.

따라서 가끔은 메모리 관리 화면을 열어 어떤 정보가 저장되어 있는지 확인하는 것이 좋습니다.

### 12.3 하나의 계정은 한 사람이 사용하는 것이 좋습니다

메모리는 사용자 개인의 맥락을 기반으로 작동합니다.

여러 사람이 하나의 계정을 함께 사용하면 서로의 정보가 섞일 수 있습니다. 그러면 ChatGPT 가 누구의 맥락을 기준으로 답변해야 하는지 혼란스러워질 수 있습니다.

따라서 가능하면 하나의 계정은 한 사람이 사용하는 것이 좋습니다.

### 12.4 회사 정책을 확인합니다

기업이나 조직에서 ChatGPT 를 사용할 때는 메모리 기능 사용 여부가 내부 보안 정책과 연결될 수 있습니다.

회사에서 AI 사용 가이드라인을 제공한다면, 메모리 기능을 켜도 되는지, 어떤 정보를 입력해도 되는지, 업무 데이터가 포함된 대화를 어떻게 관리해야 하는지 확인해야 합니다.

## 13. 실습하기

이번 강의에서는 메모리를 직접 저장하고 확인하고 삭제해 봅니다.

### 13.1 메모리 저장 실습

새 채팅에서 아래 프롬프트를 입력합니다.

<pre><code>나는 채식주의자입니다. 반드시 기억해 주세요.</code></pre>

답변 근처에 `메모리 업데이트됨` 이 표시되는지 확인합니다.

### 13.2 저장된 메모리 확인

설정 메뉴에서 메모리 관리 화면으로 이동하거나, 아래 프롬프트를 입력해 확인합니다.

<pre><code>나에 대해 무엇을 기억하고 있어?</code></pre>

### 13.3 테스트 메모리 삭제

이번 실습에서 입력한 “채식주의자” 정보가 실제 정보가 아니라면, 메모리 관리 화면에서 삭제합니다.

삭제 후 다시 아래와 같이 질문해 봅니다.

<pre><code>점심 메뉴를 추천해 주세요.</code></pre>

채식주의자 조건이 더 이상 반영되지 않는지 확인합니다.

## 14. 실습 완료 기준

이번 강의의 실습은 다음 기준으로 완료할 수 있습니다.

* 개인 맞춤 설정에서 메모리 설정 위치를 확인했다.
* 메모리 기능이 켜져 있는지 확인했다.
* 테스트 문장을 입력해 메모리 저장을 시도했다.
* `메모리 업데이트됨` 안내를 확인했다.
* 저장된 메모리 관리 화면을 열어 보았다.
* 테스트용 메모리를 삭제했다.
* 메모리가 답변에 영향을 줄 수 있다는 점을 이해했다.

## 15. 핵심 정리

* 메모리는 ChatGPT 가 사용자의 중요한 정보를 기억해 이후 대화에서 활용하는 기능입니다.
* 기본적으로 각 채팅은 독립적으로 동작하지만, 메모리를 사용하면 대화방이 달라져도 사용자 맥락을 유지할 수 있습니다.
* 저장된 메모리와 채팅 기록 참고는 서로 다른 방식의 개인화 기능입니다.
* 메모리는 사용자가 명시적으로 기억해 달라고 요청하거나, ChatGPT 가 중요하다고 판단한 정보를 저장할 수 있습니다.
* 저장된 메모리는 관리 화면에서 확인하고 삭제할 수 있습니다.
* 잘못된 메모리는 답변 품질을 떨어뜨릴 수 있으므로 주기적으로 확인하는 것이 좋습니다.
* 비밀번호, 계좌번호, 고객 개인정보, 회사 기밀 같은 민감정보는 메모리에 저장하지 않습니다.
* 메모리는 ChatGPT 를 단순한 도구가 아니라 나를 이해하는 개인화된 AI 비서로 만드는 핵심 기능입니다.
