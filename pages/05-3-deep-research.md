# 05-3. 심층 리서치(Deep Research)

## 1. 이 강의에서 배울 내용

경쟁사 세 곳을 비교해 달라는 업무 요청을 받았다고 가정해 보겠습니다.

일반적으로는 검색어를 여러 번 바꾸어 입력하고, 기사와 보고서를 하나씩 열어보고, 필요한 내용을 복사한 뒤 출처를 정리합니다. 이후 자료를 표로 만들고 보고서 형태로 다시 다듬습니다.

이 과정에서 가장 큰 문제는 정보가 부족한 것이 아닙니다.

오히려 정보가 너무 많고 여러 곳에 흩어져 있어서, 업무 판단에 필요한 자료만 골라내고 서로 비교하는 데 많은 시간이 소요됩니다.

ChatGPT의 심층 리서치를 사용하면 이러한 조사 과정의 상당 부분을 AI에 위임할 수 있습니다.

다만 심층 리서치는 단순히 `조사해 주세요`라고 요청한다고 해서 좋은 결과가 만들어지는 기능은 아닙니다.

무엇을 조사해야 하는지, 어떤 자료를 우선해야 하는지, 결과를 어떤 형태로 받아야 하는지를 명확히 설명해야 합니다.

이 강의를 통해 다음 내용을 익힐 수 있습니다.

* 심층 리서치가 무엇인지 이해할 수 있습니다.
* 일반 ChatGPT 답변, 웹 검색과 심층 리서치의 차이를 구분할 수 있습니다.
* 심층 리서치가 적합한 업무와 적합하지 않은 업무를 판단할 수 있습니다.
* 심층 리서치를 검색이 아니라 조사 업무 위임으로 활용할 수 있습니다.
* C.P.R. 프레임워크로 조사 의뢰서형 프롬프트를 작성할 수 있습니다.
* 웹, 업로드 파일, 연결된 앱과 특정 웹사이트 중 조사 출처를 선택할 수 있습니다.
* ChatGPT가 제안한 조사 계획을 검토하고 수정할 수 있습니다.
* 심층 리서치의 진행 상황을 확인하고 필요하면 조사 방향을 조정할 수 있습니다.
* 보고서의 핵심 주장과 출처를 검증할 수 있습니다.
* 검증 결과를 신뢰, 보류, 제외로 구분할 수 있습니다.
* 긴 리서치 보고서를 1페이지 브리핑으로 압축할 수 있습니다.
* 조사 결과를 30-60-90일 액션 플랜으로 변환할 수 있습니다.
* 내부 자료와 외부 자료를 결합할 때의 보안 기준을 이해할 수 있습니다.

## 2. 심층 리서치란 무엇인가

심층 리서치는 복잡한 조사 과제를 여러 단계로 수행하고, 그 결과를 출처가 포함된 보고서로 만드는 ChatGPT의 리서치 기능입니다.

사용자가 조사 목적과 조건을 설명하면 ChatGPT는 다음과 같은 과정을 수행합니다.

1. 사용자의 조사 목적을 파악합니다.
2. 필요한 조사 범위와 하위 질문을 정리합니다.
3. 조사 계획을 제안합니다.
4. 여러 자료를 탐색합니다.
5. 발견한 내용에 따라 다음 조사 방향을 조정합니다.
6. 서로 다른 자료를 비교하고 종합합니다.
7. 출처가 연결된 구조화된 보고서를 작성합니다.

따라서 심층 리서치는 한 번 검색하고 끝나는 기능이 아닙니다.

조사라는 과제 자체를 여러 단계로 수행한다는 점에서 **자율형 리서치 에이전트**로 이해할 수 있습니다.

심층 리서치의 핵심은 자료의 양이 아닙니다.

> 여러 자료를 어떤 기준으로 탐색하고, 어떻게 종합하며, 어떤 업무 판단에 사용할 수 있도록 정리하는지가 핵심입니다.

## 3. 일반 답변·웹 검색·심층 리서치 비교

일반 ChatGPT 답변, 웹 검색과 심층 리서치는 서로 대체하는 기능이 아닙니다.

과제의 복잡성과 필요한 결과에 따라 구분해서 사용해야 합니다.

| 구분       | 일반 ChatGPT 답변  | 웹 검색          | 심층 리서치               |
| -------- | -------------- | ------------- | -------------------- |
| 주요 목적    | 설명, 아이디어, 글 작성 | 최신 정보 확인      | 복잡한 조사 과제 수행         |
| 조사 깊이    | 비교적 짧고 빠름      | 특정 질문 중심      | 다단계 탐색과 종합           |
| 최신 웹 정보  | 필수적이지 않음       | 사용함           | 사용함                  |
| 여러 출처 비교 | 제한적일 수 있음      | 요청에 따라 가능     | 핵심 기능                |
| 조사 계획    | 일반적으로 없음       | 일반적으로 없음      | 실행 전 제안 가능           |
| 조사 방향 수정 | 후속 질문으로 수행     | 후속 검색으로 수행    | 진행 중에도 조정 가능         |
| 결과 형태    | 대화형 답변         | 출처가 포함된 답변    | 구조화된 종합 보고서          |
| 적합한 과제   | 개념 설명, 글쓰기     | 최근 뉴스, 가격, 정책 | 시장 조사, 경쟁사 분석, 규제 조사 |
| 소요 시간    | 짧음             | 비교적 짧음        | 복잡도에 따라 더 오래 걸림      |

다음과 같은 질문은 일반 답변이나 웹 검색으로도 충분합니다.

* 현재 제품 가격은 얼마인가?
* 최근 발표된 뉴스는 무엇인가?
* 특정 용어의 의미는 무엇인가?
* 특정 회사의 공식 홈페이지는 어디인가?

반면 다음과 같은 과제는 심층 리서치가 적합합니다.

* 최근 18개월의 시장 동향과 경쟁 구도를 분석합니다.
* 세 개 벤더의 기능, 가격, 보안과 고객 반응을 비교합니다.
* 여러 국가의 규제를 비교하고 사업 영향을 정리합니다.
* 내부 매출 보고서와 외부 시장 자료를 함께 분석합니다.
* 여러 출처를 바탕으로 경영진 의사결정 자료를 작성합니다.

## 4. 심층 리서치는 검색이 아니라 위임입니다

심층 리서치를 사용할 때는 사용자의 역할에 대한 관점이 달라져야 합니다.

기존 조사 방식에서 사용자는 직접 검색하는 사람이었습니다.

* 검색어를 만듭니다.
* 검색 결과를 엽니다.
* 필요한 자료를 찾습니다.
* 내용을 복사합니다.
* 자료를 비교합니다.
* 출처를 정리합니다.
* 보고서를 작성합니다.

심층 리서치를 사용하면 이 과정의 상당 부분을 ChatGPT에 위임할 수 있습니다.

대신 사용자는 다음 역할을 맡습니다.

1. 무엇을 조사해야 하는지 정의합니다.
2. 조사가 어떤 의사결정에 사용되는지 설명합니다.
3. 조사할 기간, 지역과 대상을 지정합니다.
4. 신뢰할 출처와 제외할 출처를 정합니다.
5. 결과물의 구조와 깊이를 지정합니다.
6. 조사 계획이 목적에 맞는지 검토합니다.
7. 핵심 주장과 출처를 검증합니다.
8. 보고서를 실제 업무 산출물로 변환합니다.

즉, 실무자의 역할이 **검색 작업자**에서 **조사 의뢰자와 검토자**로 이동합니다.

심층 리서치를 사용할 때 기억할 핵심 문장은 다음과 같습니다.

> 심층 리서치는 질문에 답하는 기능이 아니라 조사 과제를 수행하는 기능입니다.

## 5. 심층 리서치가 적합한 업무

심층 리서치는 여러 자료를 종합하고 비교해야 하는 복잡한 조사에 적합합니다.

| 업무 유형       | 활용 방법                  | 대표 산출물       |
| ----------- | ---------------------- | ------------ |
| 시장 조사       | 시장 규모, 성장 요인, 경쟁 구도 종합 | 시장 진입 검토 보고서 |
| 경쟁사 분석      | 기능, 가격, 고객 반응, 포지셔닝 비교 | 경쟁사 배틀카드     |
| 규제 조사       | 공식 기관 자료와 법률 분석 자료 비교  | 규제 영향 분석표    |
| 벤더 평가       | 공식 문서, 인증, 리뷰, 가격 비교   | 벤더 스코어카드     |
| 산업 동향 분석    | 기사, 통계, 보고서와 전문가 의견 종합 | 경영진 브리핑      |
| 고객 조사       | 시장 자료와 고객 반응 비교        | 고객 요구사항 요약   |
| 내부·외부 자료 결합 | 내부 실적과 외부 시장 요인 비교     | 원인 가설 검증표    |
| 교육 수요 조사    | 채용 공고와 산업 역량 자료 분석     | 교육 과정 기획안    |

다음 조건에 해당할수록 심층 리서치의 활용 가치가 높아집니다.

* 하나의 검색 결과만으로 답하기 어렵습니다.
* 여러 출처의 내용을 비교해야 합니다.
* 최신 자료가 필요합니다.
* 공식 출처가 중요합니다.
* 조사 결과에 근거가 있어야 합니다.
* 결과를 보고서나 의사결정 자료로 만들어야 합니다.
* 조사에 상당한 시간이 소요됩니다.

## 6. 심층 리서치가 적합하지 않은 업무

모든 조사에 심층 리서치를 사용할 필요는 없습니다.

| 업무             | 부적합한 이유           | 적합한 대안         |
| -------------- | ----------------- | -------------- |
| 간단한 사실 확인      | 조사 범위가 좁음         | 웹 검색           |
| 일반 개념 설명       | 최신 출처 종합이 필요하지 않음 | 일반 ChatGPT 대화  |
| 짧은 문장 수정       | 조사 과제가 아님         | 일반 ChatGPT 대화  |
| 단순 계산          | 리서치가 필요하지 않음      | 계산 또는 데이터 분석   |
| 최종 법률 판단       | 전문적인 책임 판단 필요     | 법무 검토          |
| 최종 투자 결정       | 정량 검증과 승인 절차 필요   | 내부 의사결정 절차     |
| 최종 재무 수치 확정    | 원장과 계산 검증 필요      | 재무 시스템과 담당자 검증 |
| 내부 기밀 기반 단독 결정 | 보안과 책임 문제         | 승인된 내부 절차      |

특히 다음 업무는 심층 리서치 결과를 그대로 사용하면 안 됩니다.

* 계약 체결 여부 결정
* 투자 실행 결정
* 법률 위반 여부 확정
* 의료 진단
* 직원 평가와 징계
* 회계 결산 수치 확정
* 보안 사고의 최종 판정

심층 리서치는 의사결정을 돕는 자료를 만들 수 있지만, 최종 의사결정권자나 담당 전문가를 대신하지 않습니다.

## 7. 심층 리서치의 전체 업무 흐름

심층 리서치를 활용한 조사 업무는 다음 일곱 단계로 진행할 수 있습니다.

> 의사결정 정의 → 질문 구조화 → 조사 의뢰서 작성 → 조사 계획 검토 → 보고서 검토 → 출처 검증 → 실무 산출물 재가공

| 단계           | 주요 작업              | 산출물       | 핵심 질문                 |
| ------------ | ------------------ | --------- | --------------------- |
| 1. 의사결정 정의   | 조사 결과의 사용 목적 정의    | 의사결정 문장   | 이 조사로 무엇을 결정할 것인가?    |
| 2. 질문 구조화    | 큰 질문을 하위 질문으로 분해   | 리서치 질문 목록 | 어떤 세부 질문에 답해야 하는가?    |
| 3. 조사 의뢰서 작성 | 맥락, 우선순위, 결과 형식 작성 | 실행 프롬프트   | 기간과 출처가 명확한가?         |
| 4. 조사 계획 검토  | ChatGPT가 제안한 계획 확인 | 확정된 조사 계획 | 조사 범위가 목적에 맞는가?       |
| 5. 보고서 검토    | 구조, 누락, 불확실성 확인    | 보완 요청 목록  | 빠진 관점이나 대상이 있는가?      |
| 6. 출처 검증     | 핵심 주장과 원문 비교       | 출처 검증표    | 수치와 날짜가 맞는가?          |
| 7. 재가공       | 브리핑과 액션 플랜 작성      | 실무 산출물    | 바로 회의와 실행에 사용할 수 있는가? |

이 흐름을 한 문장으로 정리하면 다음과 같습니다.

> AI가 초안을 만들고, 사람이 검증한 뒤, 업무 산출물로 바꿉니다.

## 8. 좋은 결과는 좋은 조사 의뢰서에서 나옵니다

다음과 같이 입력한다고 가정해 보겠습니다.

<pre><code>클라우드 보안 시장을 조사해 주세요.</code></pre>

이 요청에는 다음 정보가 없습니다.

* 조사 목적
* 보고 대상
* 의사결정 상황
* 조사 기간
* 조사 지역
* 비교 대상
* 우선 출처
* 제외할 출처
* 결과물의 구성
* 불확실한 정보의 처리 방법

따라서 결과가 넓고 일반적인 시장 설명으로 흐를 가능성이 높습니다.

심층 리서치에서는 검색어가 아니라 **조사 의뢰서**를 작성해야 합니다.

조사 의뢰서에는 최소한 다음 내용이 필요합니다.

1. 왜 조사하는가?
2. 누가 결과를 사용할 것인가?
3. 어떤 결정을 내리기 위한 조사인가?
4. 어떤 기간과 시장을 조사할 것인가?
5. 어떤 출처를 우선할 것인가?
6. 무엇을 제외할 것인가?
7. 어떤 질문에 답해야 하는가?
8. 어떤 결과물로 받아야 하는가?
9. 불확실한 내용은 어떻게 표시해야 하는가?

## 9. C.P.R. 프롬프트 프레임워크

심층 리서치용 조사 의뢰서를 작성할 때 C.P.R. 프레임워크를 사용할 수 있습니다.

C.P.R.은 다음 세 요소의 첫 글자를 조합한 것입니다.

| 요소            | 의미      | 작성해야 할 내용                     |
| ------------- | ------- | ----------------------------- |

## 11. 심층 리서치 시작하기

2026년 7월 현재 OpenAI 공식 도움말에서는 다음 방법으로 심층 리서치를 시작할 수 있다고 안내합니다.

* 프롬프트 입력창에 `/Deepresearch` 입력

영상 촬영 당시에는 플러스 버튼과 `@` 메뉴에서 심층 리서치를 선택하는 방법을 사용했습니다. 메뉴 위치와 실행 방식은 앱, 화면 크기와 업데이트에 따라 달라질 수 있습니다.

실행 순서는 다음과 같습니다.

1. 새로운 채팅을 시작합니다.
2. 심층 리서치를 선택합니다.
3. 조사 의뢰서형 프롬프트를 입력합니다.
4. 사용할 자료와 출처 범위를 선택합니다.
5. ChatGPT가 제안한 조사 계획을 확인합니다.
6. 필요한 경우 조사 계획을 수정합니다.
7. 조사를 시작합니다.

실제 화면에서는 버튼의 위치보다 다음 상태를 확인하는 것이 중요합니다.

## 12. 조사에 사용할 출처 선택하기

심층 리서치는 다음 자료를 조사에 활용할 수 있습니다.

* 공개 웹
* 사용자가 업로드한 파일
* 사용자가 연결한 ChatGPT 앱의 자료
* 사용자가 지정한 특정 웹사이트

공식 도움말에 따르면 특정 웹사이트만 조사하도록 제한하거나, 지정한 사이트를 우선하면서 전체 웹도 함께 검색하게 할 수 있습니다. 연결된 앱은 심층 리서치에서 읽기 작업에 사용되며, 앱의 데이터를 수정하는 작업에는 사용되지 않습니다.

### 12.1 전체 웹 사용

시장 전체의 동향이나 여러 관점을 넓게 조사할 때 사용합니다.

예시는 다음과 같습니다.

* 산업 동향
* 경쟁사 분석
* 고객 반응
* 시장 규모
* 제품 비교

### 12.2 특정 웹사이트 우선

공식 자료를 중심으로 조사하고 싶을 때 사용합니다.

예를 들어 다음 사이트 유형을 우선할 수 있습니다.

* 정부 기관
* 규제 기관
* 기업 공식 홈페이지
* 공시 시스템
* 학술 기관
* 공식 제품 문서
* 공식 가격 페이지

### 12.3 업로드 파일 활용

내부 자료와 외부 자료를 함께 분석할 때 사용합니다.

예시는 다음과 같습니다.

* 내부 매출 보고서와 외부 시장 자료 비교
* 고객 설문 결과와 산업 동향 결합
* 기존 전략 보고서와 최근 경쟁사 발표 비교
* 제안요청서와 벤더 공식 자료 비교

### 12.4 연결된 앱 활용

사용자가 접근 권한을 가진 문서 저장소나 데이터 서비스를 연결한 경우 해당 자료를 조사에 포함할 수 있습니다.

조사에 사용할 자료는 많을수록 좋은 것이 아닙니다.

목적과 관련된 출처만 선택해야 합니다.

## 13. 조사 계획 검토하기

심층 리서치를 실행하면 ChatGPT가 곧바로 최종 보고서를 작성하는 것이 아니라, 먼저 조사 계획을 제안할 수 있습니다.

공식 도움말에 따르면 사용자는 조사 시작 전에 제안된 계획을 검토하고 수정할 수 있습니다. 조사 목적을 확인하기 위한 명확화 질문이 함께 제시될 수도 있습니다.

조사 계획에서는 다음 내용을 확인합니다.

| 점검 항목 | 확인 질문                    |
| ----- | ------------------------ |
| 조사 목적 | 내가 의도한 의사결정과 연결되어 있는가?   |
| 조사 범위 | 기간과 지역이 너무 넓거나 좁지 않은가?   |
| 조사 대상 | 핵심 경쟁사와 기관이 포함되어 있는가?    |
| 핵심 질문 | 반드시 답해야 할 질문이 모두 포함되었는가? |
| 출처    | 공식 자료와 신뢰할 자료가 우선되는가?    |
| 제외 조건 | 홍보성·중복 자료의 처리 기준이 있는가?   |
| 결과 형식 | 필요한 표와 브리핑이 포함되어 있는가?    |

조사 계획이 목적과 다르면 그대로 시작하지 않습니다.

다음과 같이 수정할 수 있습니다.

<pre><code>조사 계획을 다음 기준으로 수정해 주세요.

1. 북미 전체가 아니라 미국과 캐나다를 구분해 주세요.
2. 경쟁사는 중견기업 대상 솔루션 3개로 제한해 주세요.

5. 최종 결과에 가격, 인증, 통합 기능 비교표를 포함해 주세요.</code></pre>

## 14. 명확화 질문에 구체적으로 답하기

심층 리서치는 조사 방향을 명확하게 하기 위해 추가 질문을 할 수 있습니다.

이 질문에 모호하게 답하면 리서치 결과도 모호해집니다.

* `최근` 대신 구체적인 기간을 씁니다.

* `신뢰할 자료` 대신 출처 유형을 지정합니다.
* `잘 정리` 대신 결과물의 목차를 제시합니다.

## 15. 조사 진행 상태 확인과 방향 수정

심층 리서치가 실행되면 진행 상황을 확인할 수 있습니다.

현재 공식 안내에 따르면 사용자는 조사 과정을 추적하고, 진행 중인 작업을 중단하거나 추가 지시와 새로운 출처를 제공해 조사 초점을 조정할 수 있습니다.

진행 과정에서 다음 상황이 발견되면 조사 방향을 수정합니다.

* 관련 없는 시장을 조사하고 있습니다.
* 특정 기업에 자료가 편중되어 있습니다.
* 오래된 자료가 많이 사용되고 있습니다.
* 공식 자료보다 블로그 자료가 많습니다.
* 중요한 비교 대상이 빠져 있습니다.
* 지역 범위가 너무 넓습니다.
* 내부 자료가 제대로 반영되지 않았습니다.

수정 요청 예시는 다음과 같습니다.

<pre><code>현재 조사 방향을 다음과 같이 조정해 주세요.

- 글로벌 시장 전체보다 북미와 한국을 우선합니다.
- 공식 기업 문서와 보안 인증 자료의 비중을 높입니다.
- 최근 18개월 이전의 자료는 배경 정보로만 사용합니다.
- 중견기업 고객의 구매 기준과 도입 장벽을 더 깊게 조사합니다.
- 가격 정보가 없는 경우 추정하지 않습니다.</code></pre>

## 16. 완성된 리서치 보고서 검토하기

리서치가 완료되면 구조화된 보고서가 제공됩니다.

현재 공식 도움말에서는 완성된 보고서에 다음 검토 기능이 제공된다고 안내합니다.

* 긴 보고서를 이동할 수 있는 목차
* 인용 또는 출처 링크
* 사용된 출처 목록
* 조사 진행 과정을 보여주는 작업 기록
* Markdown, Word와 PDF 형식의 다운로드

보고서는 전체 화면에서 검토하고 재사용할 수 있습니다.

보고서를 받은 뒤 다음을 확인합니다.

### 16.1 요청한 구조 준수

* 핵심 요약이 있는가?
* 경쟁사 비교표가 있는가?
* 고객 반응이 포함되어 있는가?
* 전략적 시사점이 있는가?
* 리스크와 불확실성이 구분되어 있는가?
* 의사결정 질문이 포함되어 있는가?

### 16.2 조사 범위 준수

* 요청한 기간을 지켰는가?
* 요청한 지역을 지켰는가?
* 대상 고객군이 맞는가?
* 비교 대상이 적절한가?
* 제외하기로 한 자료가 핵심 근거에 사용되지 않았는가?

### 16.3 불확실성 표시

* 가격 공개 여부가 구분되어 있는가?
* 계획과 확정 사항이 구분되어 있는가?
* 공식 정보와 분석 의견이 구분되어 있는가?
* 단일 출처에만 의존한 주장이 표시되어 있는가?
* 충돌하는 출처가 별도로 설명되어 있는가?

## 17. 출처가 있다는 것과 검증되었다는 것은 다릅니다

심층 리서치 보고서에는 출처가 포함됩니다.

그러나 출처가 있다는 이유만으로 보고서의 모든 내용이 사실이라고 판단하면 안 됩니다.

다음과 같은 문제가 발생할 수 있습니다.

* 검색된 원문 자체가 오래되었습니다.
* 원문이 부정확하거나 편향되어 있습니다.
* AI가 원문의 표현을 과도하게 일반화했습니다.
* 계획이나 전망을 확정된 사실처럼 표현했습니다.
* 서로 다른 기준의 숫자를 비교했습니다.
* 통화와 단위를 잘못 해석했습니다.
* 동일한 보도자료를 여러 독립 출처처럼 사용했습니다.
* 원문과 보고서의 맥락이 다릅니다.

특히 다음 정보는 반드시 직접 검증합니다.

* 시장 규모
* 성장률
* 투자 금액
* 제품 가격
* 정책과 규제명
* 시행일
* 보안 인증
* 계약 여부
* 고객 수
* 제품 출시일

> 출처는 검증의 시작점이며, 검증의 완료를 의미하지 않습니다.

## 20. 보고서는 끝이 아니라 재료입니다

심층 리서치가 만든 보고서는 최종 업무 산출물이 아닐 수 있습니다.

보고서는 다양한 자료와 분석을 모아둔 **조사 결과물**입니다.

실제 업무에서는 이를 목적에 맞는 문서로 다시 변환해야 합니다.

| 리서치 보고서  | 변환할 수 있는 실무 산출물   |
| -------- | ----------------- |
| 시장 조사    | 시장 진입 검토 메모       |
| 경쟁사 분석   | 영업 배틀카드           |
| 벤더 조사    | 구매 검토표와 PoC 체크리스트 |
| 규제 조사    | 법무 질의 목록과 대응 일정   |
| 고객 조사    | 고객 인터뷰 질문지        |
| 직무 조사    | 교육 로드맵            |
| 내부·외부 분석 | 원인 가설 검증표         |
| 산업 동향    | 경영진 브리핑           |

이 단계에서 중요한 것은 보고서의 길이가 아닙니다.

의사결정자가 필요한 내용을 얼마나 빠르게 파악할 수 있는지가 중요합니다.

| 결과가 너무 일반적임      | 조사 목적과 범위가 없음    | C.P.R. 구조로 다시 작성    |
| 보고서가 지나치게 김      | 질문과 결과 형식이 넓음    | 대상과 목차를 제한          |
| 중요한 경쟁사가 빠짐      | 비교 대상을 지정하지 않음   | 대상 기업을 직접 지정        |
| 오래된 자료가 포함됨      | 기간 조건이 없음        | 최근 12개월 등 기간 지정     |
| 홍보성 자료가 많음       | 우선 출처 기준이 없음     | 공식 문서와 원천 자료 우선     |
| 가격이 추정됨          | 비공개 정보 처리 기준이 없음 | 공식 가격이 없으면 확인 불가 표시 |
| 출처는 있지만 내용이 다름   | 원문 맥락을 과도하게 해석   | 핵심 주장 직접 검증         |
| 같은 보도자료를 반복 인용   | 독립 출처 기준이 없음     | 서로 독립적인 자료 요구       |
| AI 결론이 과도하게 단정적임 | 사실과 해석이 혼합됨      | 사실·해석·가설로 분리        |
| 보고서만 만들고 끝남      | 후속 산출물 계획이 없음    | 브리핑과 액션 플랜으로 변환     |
| 명확화 질문에 모호하게 답함  | 조사 기준이 정리되지 않음   | 기간, 시장과 결과 형식 구체화   |
| 내부 정보가 과다하게 포함됨  | 보안 점검 누락         | 비식별화와 최소 입력 원칙 적용   |

## 28. 실습 완료 기준

이번 강의의 실습은 다음 기준으로 완료할 수 있습니다.

* 심층 리서치가 필요한 조사 주제를 선정했다.
* 이 조사로 결정할 사항을 한 문장으로 정의했다.
* 보고 대상을 명확하게 정했다.
* 조사 기간, 지역과 비교 대상을 제한했다.
* 우선 출처와 제외할 출처를 지정했다.
* C.P.R. 구조의 조사 의뢰서를 작성했다.
* 조사에 필요한 핵심 질문을 작성했다.
* 심층 리서치 모드를 선택했다.
* 조사에 사용할 출처를 선택했다.
* ChatGPT가 제안한 조사 계획을 검토했다.
* 필요한 부분을 수정한 뒤 조사를 시작했다.
* 완성된 보고서의 구조와 누락 항목을 확인했다.
* 핵심 주장 다섯 개를 선택했다.
* 각 주장의 원문을 직접 확인했다.
* 검증 결과를 신뢰, 보류, 제외로 구분했다.
* 검증 결과를 반영해 보고서를 수정했다.
* 같은 대화에서 1페이지 브리핑을 작성했다.
* 검증되지 않은 내용이 결론에서 제외되었는지 확인했다.
* 내부 자료 사용 시 보안 정책을 확인했다.

## 29. 핵심 정리

* 심층 리서치는 단순 검색이 아니라 복잡한 조사 과제를 수행하는 기능입니다.
* 일반 검색은 자료의 위치를 찾는 데 적합하고, 심층 리서치는 여러 자료의 종합과 비교에 적합합니다.
* 심층 리서치를 사용하면 실무자는 검색 작업자가 아니라 조사 의뢰자와 검토자가 됩니다.
* 시장 조사, 경쟁사 분석, 규제 조사와 벤더 평가처럼 다단계 조사가 필요한 업무에 적합합니다.
* 단순 사실 확인은 웹 검색이나 일반 대화가 더 빠를 수 있습니다.
* 심층 리서치의 품질은 조사 의뢰서의 품질에 크게 영향을 받습니다.
* C.P.R.은 Context, Prioritize와 Refine으로 구성됩니다.
* Context에는 조사 배경과 의사결정 목적을 작성합니다.
* Prioritize에는 기간, 지역, 출처와 제외 조건을 작성합니다.
* Refine에는 핵심 질문, 출력 형식과 검증 기준을 작성합니다.
* 심층 리서치는 공개 웹, 업로드 파일, 연결된 앱과 특정 사이트를 조사에 활용할 수 있습니다.
* 실행 전 조사 계획을 검토하고 수정할 수 있습니다.
* 진행 중에도 조사 방향과 출처를 조정할 수 있습니다.
* 완성된 보고서에는 인용과 사용 출처가 포함되지만, 출처가 있다는 것이 검증 완료를 의미하지는 않습니다.
* 숫자, 날짜, 가격, 정책명과 인증은 원문을 직접 확인해야 합니다.
* 핵심 주장을 신뢰, 보류와 제외로 구분하면 보고서의 품질을 통제할 수 있습니다.
* 심층 리서치 보고서는 최종 산출물이 아니라 업무 자료의 재료입니다.
* 검증한 보고서는 1페이지 브리핑, 체크리스트와 액션 플랜으로 재가공해야 합니다.
* 내부 자료와 외부 자료를 결합할 때는 기간, 기준과 보안을 확인해야 합니다.
* 심층 리서치의 기본 흐름은 `AI 조사 → 사람 검증 → 업무 산출물 변환`입니다.
* AI가 조사 시간을 줄여주면 사람은 판단, 검증과 의사결정 설계에 더 많은 시간을 사용할 수 있습니다.
