# 04-6. 채팅 공유: ChatGPT Shared Links

## 1. 이 강의에서 배울 내용

이번 강의에서는 ChatGPT에서 나눈 대화를 다른 사람에게 전달하는 **채팅 공유 기능**을 다룹니다.

ChatGPT를 업무에 활용하다 보면 잘 만든 프롬프트, 분석 결과, 문제 해결 과정 또는 참고할 만한 대화를 동료에게 보여줘야 하는 상황이 생깁니다.

이때 화면을 캡처하거나 대화 내용을 복사해 전달할 수도 있습니다. 하지만 이런 방식은 대화의 앞뒤 맥락이 잘리고, 어떤 질문에 어떤 답변이 이어졌는지 파악하기 어렵다는 문제가 있습니다.

ChatGPT의 공유 링크를 사용하면 특정 대화를 하나의 URL로 만들어 전달할 수 있습니다.

이 강의를 통해 다음 내용을 익힐 수 있습니다.

* ChatGPT 공유 링크가 무엇인지 이해할 수 있습니다.
* 화면 캡처 및 복사·붙여넣기 방식과 공유 링크의 차이를 구분할 수 있습니다.
* 사이드바와 대화 화면에서 공유 링크를 만들 수 있습니다.
* 공유 링크에 포함되는 대화 범위를 이해할 수 있습니다.
* 공유받은 사람에게 대화가 어떻게 보이는지 확인할 수 있습니다.
* 공유 링크와 ChatGPT가 생성한 외부 링크를 구분할 수 있습니다.
* 설정에서 공유 링크 목록을 확인하고 삭제할 수 있습니다.
* 공유 링크를 삭제해도 이미 만들어진 복사본이 남을 수 있다는 점을 이해할 수 있습니다.
* 개인용 ChatGPT와 Enterprise 워크스페이스의 공유 방식 차이를 이해할 수 있습니다.
* 개인정보와 내부 기밀을 보호하며 안전하게 대화를 공유할 수 있습니다.

## 2. 왜 대화를 링크로 공유해야 하는가

ChatGPT로 작업하다 보면 다른 사람과 공유할 가치가 있는 대화가 만들어집니다.

예를 들어 다음과 같은 대화입니다.

* 팀에서 반복해서 사용할 수 있는 프롬프트
* 보고서나 제안서를 작성한 과정
* 데이터 분석을 위해 사용한 질문과 결과
* 고객 응대 문안을 여러 번 개선한 과정
* 아이디어를 비교하고 최종안을 선택한 과정
* 롤플레잉이나 시뮬레이션 실습 결과
* 문제의 원인을 찾고 해결 방법을 도출한 과정

이런 결과를 화면 캡처로 공유하면 대화의 일부만 전달될 수 있습니다.

텍스트를 복사해 붙여넣는 경우에도 다음과 같은 문제가 생깁니다.

* 질문과 답변의 구분이 흐려집니다.
* 대화 순서가 바뀔 수 있습니다.
* 표, 목록, 인용문 등의 형식이 깨질 수 있습니다.
* 중간에 사용한 추가 프롬프트가 누락될 수 있습니다.
* 결과가 만들어진 과정을 확인하기 어렵습니다.
* 공유 과정에서 내용을 잘못 수정할 수 있습니다.

공유 링크는 대화의 구조와 흐름을 그대로 전달한다는 점에서 이런 문제를 줄여줍니다.

| 공유 방식   | 장점                 | 한계                               |
| ------- | ------------------ | -------------------------------- |
| 화면 캡처   | 빠르고 간단함            | 대화 일부만 보이며 맥락이 잘릴 수 있음           |
| 텍스트 복사  | 필요한 부분만 전달 가능      | 서식과 대화 순서가 깨질 수 있음               |
| 문서로 재작성 | 내용을 목적에 맞게 편집 가능   | 시간과 추가 작업이 필요함                   |
| 공유 링크   | 질문과 답변의 흐름을 그대로 전달 | 링크를 가진 사람이 대화를 볼 수 있으므로 보안 점검 필요 |

공유 링크의 핵심은 단순히 편하게 보내는 것이 아닙니다.

> ChatGPT 대화를 공유한다는 것은 대화의 맥락과 접근 범위를 함께 전달하는 것입니다.

## 3. ChatGPT 공유 링크란 무엇인가

ChatGPT 공유 링크는 특정 ChatGPT 대화를 열람할 수 있도록 사용자가 직접 생성하는 URL입니다.

일반적인 공유 링크는 다음과 같은 구조를 가집니다.

<pre><code>https://chatgpt.com/share/대화식별번호</code></pre>

공유 링크를 받은 사람은 링크에 포함된 질문과 답변의 흐름을 확인할 수 있습니다.

공유 링크는 다음 용도로 활용할 수 있습니다.

* 프롬프트 템플릿을 팀원에게 전달
* 분석 과정을 검토자에게 공유
* 강의 실습 결과 제출
* 고객 응대 문안의 수정 과정 공유
* 회의 전에 참고할 AI 대화 제공
* 프로젝트 아이디어의 검토 과정 공유
* ChatGPT 활용 우수 사례 전파

다만 공유 링크는 비공개 파일 전송 기능이 아닙니다.

일반적인 개인용 공유 링크는 링크를 가진 사람이 대화를 열람할 수 있으므로, 링크를 만들기 전에 대화 내용을 반드시 점검해야 합니다.

## 4. 공유 링크는 공개 범위를 가진 링크입니다

공유 링크를 사용할 때 가장 먼저 이해해야 할 점은 **링크를 가진 사람이 대화를 볼 수 있다**는 것입니다.

공유 링크에는 비밀번호를 설정하거나 특정 이메일 주소만 지정하는 방식이 기본적으로 제공되지 않을 수 있습니다.

따라서 링크가 다른 사람에게 전달되면 원래 의도하지 않았던 사람도 대화를 볼 수 있습니다.

공유해서는 안 되는 정보의 예시는 다음과 같습니다.

* 고객 이름과 연락처
* 임직원의 개인정보
* 계약 금액과 세부 계약 조건
* 공개되지 않은 매출과 재무 정보
* 제품 출시 전 전략
* 내부 시스템의 주소나 접속 정보
* 계정과 인증에 관련된 정보
* 회사의 영업비밀
* 개인적인 상담이나 민감한 대화
* 보안 정책상 외부 공유가 금지된 파일 내용

공유할 대화에 이런 정보가 있다면 해당 대화를 그대로 공유하지 않습니다.

대신 다음 중 하나를 선택합니다.

1. 민감한 정보를 제거한 새 대화를 만듭니다.
2. 실제 이름을 A사, B사, 담당자 C처럼 익명화합니다.
3. 필요한 결과만 별도의 문서로 정리합니다.
4. 조직에서 승인한 협업 공간을 사용합니다.
5. 공유 대상과 목적에 맞게 대화 범위를 줄입니다.

> 공유 버튼을 누르기 전에 “이 대화를 링크를 가진 누구나 보아도 괜찮은가?”를 먼저 확인해야 합니다.

## 5. 공유 링크에 포함되는 대화 범위

공유 링크는 일반적으로 링크를 생성한 시점까지의 대화 내용을 스냅숏 형태로 포함합니다.

예를 들어 하나의 대화에서 다음 과정을 진행했다고 가정하겠습니다.

1. 보고서 초안을 요청합니다.
2. 결과를 표로 수정합니다.
3. 분량을 절반으로 줄입니다.
4. 임원 보고용 문체로 다시 작성합니다.
5. 공유 링크를 만듭니다.

이 경우 공유 링크에는 링크를 만든 시점까지의 질문과 답변이 포함될 수 있습니다.

공유 링크를 만든 이후 원본 대화에서 추가 질문을 하더라도 그 내용이 기존 링크에 자동으로 추가되는 것은 아닙니다.

추가한 내용까지 공유하려면 공유 링크를 다시 만들거나, 현재 제공되는 공유 화면에서 링크를 업데이트해야 합니다.

### 5.1 전체 대화 공유

사이드바 또는 대화 화면 상단에서 공유하면 일반적으로 공유 시점까지의 대화 흐름이 포함됩니다.

따라서 대화가 길다면 처음부터 마지막까지 공유해도 되는 내용인지 확인해야 합니다.

### 5.2 특정 응답 공유

사용 환경에 따라 특정 ChatGPT 응답만 선택해 공유할 수 있는 메뉴가 제공될 수 있습니다.

이 방식을 사용하면 전체 대화가 아니라 선택한 응답을 중심으로 링크를 만들 수 있습니다.

화면 구성은 웹, 모바일 앱, 계정 또는 업데이트 상태에 따라 달라질 수 있습니다. 버튼의 위치보다 **어떤 범위가 공유되는지 미리보기로 확인하는 것**이 더 중요합니다.

## 6. 대화 목록에서 공유 링크 만들기

먼저 사이드바의 대화 목록에서 공유하는 방법을 살펴보겠습니다.

### 6.1 공유할 대화 선택하기

왼쪽 사이드바에서 공유하려는 대화를 찾습니다.

이번 강의 영상에서는 이전 강의에서 진행했던 음성 모드 롤플레잉 대화를 예시로 사용합니다.

실제 업무에서 연습할 때는 개인정보나 내부 기밀이 없는 테스트 대화를 선택하는 것이 좋습니다.

### 6.2 더보기 메뉴 열기

공유할 대화 제목 오른쪽의 더보기 메뉴를 선택합니다.

더보기 메뉴에서 `공유` 또는 `공유하기`를 선택합니다.

화면 구성에 따라 메뉴 이름은 달라질 수 있습니다.

### 6.3 공유할 내용 확인하기

공유 창이 열리면 공유될 대화 내용을 확인합니다.

다음 항목을 점검합니다.

* 대화 처음부터 공유해도 되는가?
* 실제 회사명이나 고객명이 포함되어 있지 않은가?
* 민감한 파일 내용이 답변에 표시되어 있지 않은가?
* 개인적인 메모나 불필요한 질문이 포함되어 있지 않은가?
* 공유할 필요가 없는 이전 대화가 포함되어 있지 않은가?

### 6.4 링크 복사하기

공유할 내용에 문제가 없다면 `링크 복사`를 선택합니다.

복사한 링크는 다음과 같은 방법으로 전달할 수 있습니다.

* 이메일
* 사내 메신저
* Microsoft Teams
* Slack
* 문서
* 업무 게시판
* 교육 과제 제출 폼

일부 화면에서는 LinkedIn, X, Reddit 등의 서비스로 바로 공유할 수 있는 버튼도 제공될 수 있습니다.

업무에서는 일반적으로 링크를 복사한 뒤, 공유 목적과 주의사항을 함께 작성해서 전달하는 방식이 안전합니다.

## 7. 대화 화면에서 바로 공유하기

공유 링크는 사이드바뿐 아니라 현재 열어 둔 대화 화면에서도 만들 수 있습니다.

기본 절차는 다음과 같습니다.

1. 공유할 대화를 엽니다.
2. 대화 화면 오른쪽 상단의 공유 버튼을 선택합니다.
3. 공유될 대화 범위를 확인합니다.
4. 링크 복사를 선택합니다.
5. 복사한 링크를 필요한 사람에게 전달합니다.

화면에 따라 공유 버튼을 선택하는 즉시 링크가 클립보드에 복사될 수 있고, 먼저 공유 미리보기가 나타날 수도 있습니다.

중요한 것은 버튼 위치를 외우는 것이 아닙니다.

다음 세 가지를 확인하는 것이 핵심입니다.

1. 어떤 대화가 공유되는가?
2. 링크를 받은 사람은 무엇을 볼 수 있는가?
3. 대화 안에 외부로 나가면 안 되는 정보가 있는가?

## 8. 공유받은 사람에게는 어떻게 보이는가

공유 링크를 받은 사람은 웹 브라우저에서 해당 링크를 열 수 있습니다.

일반적인 개인용 공유 링크는 링크를 알고 있는 사람이 공유된 대화를 열람할 수 있습니다.

공유된 화면에서는 다음 내용을 확인할 수 있습니다.

* 사용자가 입력한 프롬프트
* ChatGPT의 답변
* 질문과 답변의 순서
* 공유 시점까지 포함된 대화 범위
* 표, 목록, 코드 등 답변의 구조

다만 공유받은 사람이 원본 대화를 편집할 수 있는 것은 아닙니다.

원본 대화의 소유권과 관리 권한은 공유 링크를 만든 사용자에게 있습니다.

### 8.1 영상 촬영 이후 변경된 사항

영상 촬영 당시에는 공유 링크를 연 화면에서 기존 대화의 맥락을 이어 추가 질문을 입력하는 실습을 진행했습니다.

현재 OpenAI 공식 안내에 따르면 일반 공유 링크에서 기존 대화를 직접 이어가는 기능은 중단되었습니다.

따라서 현재는 다음과 같이 이해해야 합니다.

* 공유 링크는 대화 내용을 열람하는 용도로 사용합니다.
* 공유받은 화면에서 원본 대화에 메시지를 추가할 수 없습니다.
* 공유받은 사람이 별도로 활용하려면 필요한 프롬프트나 내용을 자신의 새 대화에서 사용해야 합니다.
* 제품 업데이트에 따라 화면과 제공 기능이 달라질 수 있습니다.

영상의 공유 링크 생성, 열람 및 관리 과정은 참고할 수 있지만, 공유 링크 화면에서 직접 대화를 이어가는 부분은 현재 기능과 다를 수 있습니다.

## 9. 공유 링크와 생성 링크의 차이

ChatGPT에서는 서로 다른 두 종류의 링크를 볼 수 있습니다.

하나는 사용자가 직접 만드는 **공유 링크**이고, 다른 하나는 ChatGPT 답변 안에 포함되는 **생성 링크 또는 외부 링크**입니다.

| 구분    | 공유 링크                   | ChatGPT가 생성한 외부 링크     |
| ----- | ----------------------- | ---------------------- |
| 생성 주체 | 사용자                     | ChatGPT                |
| 목적    | ChatGPT 대화 공유           | 외부 웹페이지나 자료 확인         |
| 대표 형태 | `chatgpt.com/share/...` | 답변 안의 외부 웹사이트 주소       |
| 핵심 위험 | 대화의 과도한 공개              | 신뢰할 수 없는 사이트 접속과 정보 전달 |
| 확인 사항 | 공유 내용과 대상               | 링크 도메인과 전달되는 정보        |

### 9.1 공유 링크의 위험

공유 링크의 주된 위험은 내 대화 내용을 너무 넓게 공개하는 것입니다.

따라서 공유하기 전에 대화의 전체 범위를 점검해야 합니다.

### 9.2 생성 링크의 위험

ChatGPT가 답변 안에 외부 사이트 링크를 제시할 수 있습니다.

링크를 클릭하면 외부 웹사이트에 접속하게 되며, 링크 주소 자체에 검색어, 지역 또는 대화에서 사용된 정보가 포함될 수 있습니다.

외부 링크를 열기 전에는 다음을 확인합니다.

* 공식 사이트 또는 신뢰할 수 있는 도메인인가?
* 요청한 업무와 직접 관련된 링크인가?
* 링크 주소에 민감한 정보가 포함되어 있지 않은가?
* 접속 후 회사 계정 로그인을 요구하는가?
* 불필요한 개인정보를 입력하도록 요구하는가?
* ChatGPT가 링크 접속 승인을 요청하는가?

공유 링크와 생성 링크는 모두 링크이지만, 위험의 종류가 다릅니다.

## 10. 공유 전 확인해야 할 항목

대화를 공유하기 전에 다음 체크리스트를 사용하면 좋습니다.

| 점검 항목   | 확인 질문                             |
| ------- | --------------------------------- |
| 공유 목적   | 이 대화를 왜 공유하는가?                    |
| 공유 대상   | 누가 이 링크를 받는가?                     |
| 재전달 가능성 | 링크가 다른 사람에게 전달되어도 괜찮은가?           |
| 대화 범위   | 공유할 필요가 없는 앞부분이 포함되어 있지 않은가?      |
| 개인정보    | 고객명, 개인명, 연락처가 포함되어 있지 않은가?       |
| 내부 정보   | 매출, 일정, 전략, 계약 조건이 포함되어 있지 않은가?   |
| 파일 내용   | 업로드한 파일의 민감한 내용이 답변에 노출되지 않았는가?   |
| 최신성     | 오래된 정보나 폐기된 정책이 포함되어 있지 않은가?      |
| 표현      | 오해를 일으킬 수 있는 미완성 답변이 포함되어 있지 않은가? |
| 공유 기간   | 언제까지 이 링크가 필요할 것인가?               |

특히 실제 업무에서는 다음 세 단계로 관리하는 것이 좋습니다.

> 공유 전 점검 → 공유 중 접근 범위 관리 → 필요할 때 링크 회수

## 11. 공유 링크와 함께 보낼 안내문

링크만 전달하면 받는 사람이 공유 목적과 사용 방법을 이해하기 어려울 수 있습니다.

아래와 같이 간단한 안내문을 함께 작성하는 것이 좋습니다.

<pre><code>[공유 목적]

ChatGPT를 활용해 작성한 신제품 캠페인 기획 프롬프트와 결과 예시를 공유합니다.

[확인할 내용]

1. 프롬프트에 포함된 역할과 맥락
2. 캠페인 아이디어를 생성한 과정
3. 결과를 수정한 추가 프롬프트
4. 최종 결과물의 구성

[활용 방법]

공유된 대화를 참고해 자신의 제품과 고객 조건에 맞는 새 프롬프트를 작성해 주세요.

[주의사항]

- 실제 고객명, 계약 금액, 출시 일정은 입력하지 마세요.
- 링크를 외부 파트너에게 재전달하지 마세요.
- 답변에 포함된 외부 링크는 도메인을 확인한 뒤 접속하세요.

[공유 링크]

여기에 ChatGPT 공유 링크를 입력합니다.</code></pre>

이렇게 안내하면 단순한 링크 전달이 아니라, 목적이 있는 협업 자료로 사용할 수 있습니다.

## 12. 공유 링크 목록 확인하기

생성한 공유 링크는 설정에서 확인할 수 있습니다.

기본적인 접근 경로는 다음과 같습니다.

> 계정 아이콘 → 설정 → 데이터 제어 → 공유 링크 → 관리

화면의 메뉴 이름은 업데이트에 따라 조금 달라질 수 있습니다.

공유 링크 관리 화면에서는 일반적으로 다음 작업을 수행할 수 있습니다.

* 지금까지 생성한 공유 링크 확인
* 링크가 연결된 원본 대화 확인
* 불필요한 공유 링크 삭제
* 여러 공유 링크 일괄 점검
* 오래된 링크 정리

공유 링크를 자주 사용한다면 정기적으로 이 화면을 확인하는 것이 좋습니다.

특히 프로젝트가 끝났거나 더 이상 공유할 필요가 없는 링크는 삭제합니다.

## 13. 공유 링크 삭제하기

더 이상 대화를 공유하고 싶지 않다면 공유 링크를 삭제할 수 있습니다.

기본 절차는 다음과 같습니다.

1. 설정을 엽니다.
2. 데이터 제어로 이동합니다.
3. 공유 링크의 관리 화면을 엽니다.
4. 삭제할 링크를 찾습니다.
5. 휴지통 또는 링크 삭제 버튼을 선택합니다.
6. 삭제 여부를 확인합니다.

공유 링크 삭제와 원본 대화 삭제는 서로 다른 작업입니다.

| 작업       | 결과                                |
| -------- | --------------------------------- |
| 공유 링크 삭제 | 해당 링크로 대화를 열 수 없게 됨               |
| 원본 대화 유지 | 내 ChatGPT 대화 목록에는 계속 남음           |
| 원본 대화 삭제 | 원본 대화와 연결된 공유 링크도 사용할 수 없게 될 수 있음 |

공유 링크만 삭제하면 원래의 ChatGPT 대화는 그대로 유지됩니다.

즉, 공유 링크 관리 화면은 대화 자체를 삭제하는 곳이 아니라, 외부에서 대화에 접근하는 통로를 관리하는 곳입니다.

## 14. 공유 링크 삭제의 한계

공유 링크를 삭제하면 해당 URL을 통해 대화를 더 이상 볼 수 없게 됩니다.

하지만 공유받은 사람이 이미 대화 내용을 복사하거나 자신의 기록으로 가져갔다면 그 복사본까지 자동으로 삭제되지는 않을 수 있습니다.

예를 들어 받는 사람이 다음 행동을 했다면 링크를 회수한 뒤에도 내용이 남을 수 있습니다.

* 화면을 캡처했습니다.
* 대화 내용을 문서에 복사했습니다.
* 프롬프트를 자신의 ChatGPT 대화에 붙여넣었습니다.
* 공유 대화의 복사본을 자신의 기록으로 가져갔습니다.
* 다른 사람에게 다시 전달했습니다.

따라서 민감한 정보를 뒤늦게 발견했다면 링크를 삭제하는 것만으로 충분하지 않을 수 있습니다.

이 경우 다음 조치를 함께 고려해야 합니다.

1. 공유 링크를 즉시 삭제합니다.
2. 공유받은 사람에게 사용 중단을 요청합니다.
3. 저장하거나 복사한 자료의 폐기를 요청합니다.
4. 조직의 보안 담당자에게 상황을 알립니다.
5. 필요한 경우 정보 노출 범위를 확인합니다.
6. 민감정보가 제거된 새 자료를 다시 공유합니다.

공유 링크의 안전성은 링크를 회수할 수 있다는 데만 있지 않습니다.

처음부터 공유할 내용을 최소화하는 것이 더 중요합니다.

## 15. 개인용 공유 링크와 Enterprise 공유 링크

ChatGPT의 공유 링크는 사용 중인 플랜과 워크스페이스 정책에 따라 동작 방식이 다를 수 있습니다.

### 15.1 개인용 ChatGPT

개인용 ChatGPT에서는 공유 링크를 가진 사람이 해당 대화를 열람할 수 있습니다.

따라서 다음 원칙이 중요합니다.

* 외부 공개 가능한 내용만 공유합니다.
* 링크가 재전달될 가능성을 고려합니다.
* 공유할 필요가 끝나면 링크를 삭제합니다.
* 회사 기밀은 개인용 공유 링크로 전달하지 않습니다.

### 15.2 Enterprise 및 조직 워크스페이스

Enterprise 공유 링크는 일반적으로 같은 워크스페이스 구성원만 열 수 있습니다.

외부 사용자가 링크를 받더라도 워크스페이스 구성원이 아니라면 접근할 수 없을 수 있습니다.

또한 워크스페이스 관리자는 조직 정책에 따라 공유 링크 기능을 제한하거나 비활성화할 수 있습니다.

Enterprise 환경에서는 다음 사항을 확인해야 합니다.

* 공유 대상이 같은 워크스페이스 구성원인가?
* 조직 관리자가 공유 기능을 허용했는가?
* 업로드된 파일이나 이미지가 포함된 대화인가?
* 데이터 분석 대화에서 원본 파일 접근이 필요한가?
* 공유받은 사람이 상호작용할 때 별도의 비공개 대화가 만들어지는가?

Enterprise에서 고급 데이터 분석을 사용한 대화는 공유할 수 있더라도, 받는 사람이 업로드된 원본 파일 자체에 접근하지 못할 수 있습니다.

이미지가 업로드되거나 생성된 대화는 공유가 제한될 수 있습니다.

기업 환경에서는 기능의 편의성보다 조직의 보안 정책과 워크스페이스 설정을 우선합니다.

## 16. 공유 링크, 그룹 채팅, 프로젝트 비교

ChatGPT에는 협업에 사용할 수 있는 여러 기능이 있습니다.

모든 상황에 공유 링크를 사용하는 것이 가장 좋은 것은 아닙니다.

| 업무 목적            | 적합한 기능    | 이유                    |
| ---------------- | --------- | --------------------- |
| 특정 대화나 결과를 전달    | 공유 링크     | 완성된 대화 흐름을 보여주기 적합    |
| 여러 사람이 동시에 의견 교환 | 그룹 채팅     | 같은 대화방에서 공동 논의 가능     |
| 파일과 지침을 장기간 축적   | 프로젝트      | 지속적인 업무 맥락 관리에 적합     |
| 결과만 공식적으로 전달     | 문서 또는 이메일 | 필요한 정보만 선별하여 전달 가능    |
| 교육 실습 과정 제출      | 공유 링크     | 프롬프트와 개선 과정을 함께 확인 가능 |
| 장기적인 부서 지식 관리    | 프로젝트      | 반복 작업과 자료 축적에 적합      |

선택 기준은 다음과 같습니다.

* 이미 완료된 대화를 보여주려면 공유 링크
* 여러 사람이 함께 대화하려면 그룹 채팅
* 장기적으로 자료와 맥락을 관리하려면 프로젝트
* 외부에 확정된 결과만 보내려면 별도 문서

## 17. 업무 활용 예시

### 17.1 프롬프트 템플릿 공유

팀에서 사용할 표준 프롬프트를 공유할 때 사용할 수 있습니다.

예를 들어 다음과 같은 프롬프트입니다.

* 고객 사과 이메일 작성
* 주간 업무 보고서 요약
* 신제품 캠페인 기획
* 회의록 액션 아이템 정리
* 제안서 검토
* 데이터 분석 요구사항 작성

공유 링크에는 프롬프트뿐 아니라 결과를 수정한 과정도 함께 포함할 수 있습니다.

팀원은 대화 구조를 참고해 자신의 업무에 맞는 새 프롬프트를 작성할 수 있습니다.

### 17.2 교육 실습 결과 제출

수강생이 ChatGPT 실습을 완료한 뒤 공유 링크를 제출하게 할 수 있습니다.

강사는 다음 내용을 확인할 수 있습니다.

* 최초 프롬프트
* ChatGPT의 첫 답변
* 수강생이 추가한 수정 지시
* 결과 개선 과정
* 최종 산출물

다만 실습용 대화에는 실제 회사 정보나 개인정보를 입력하지 않도록 안내해야 합니다.

### 17.3 분석 결과 검토

데이터 분석 과정에서 사용한 질문과 해석을 검토자에게 공유할 수 있습니다.

이 경우 다음 사항을 함께 안내합니다.

* 분석에 사용한 데이터의 범위
* 수치가 샘플인지 실제 데이터인지
* 원본 파일의 공유 여부
* 검토가 필요한 가정
* 결과를 확정하기 전 확인할 사항

공유 링크는 분석 흐름을 보여주는 용도로 사용하고, 최종 보고 결과는 별도 문서로 확정하는 것이 좋습니다.

### 17.4 고객 응대 문안 검토

고객 이메일이나 공지문을 여러 번 수정한 대화를 동료에게 공유할 수 있습니다.

다만 실제 고객명과 연락처는 제거하고, A사 또는 고객사처럼 익명화합니다.

| 공유 이후 추가한 메시지가 보이지 않음            | 공유 링크가 스냅숏 방식으로 생성됨                    | 공유 링크를 다시 생성하거나 업데이트합니다.           |
| 영상처럼 공유 대화를 이어갈 수 없음             | 해당 기능이 현재 중단됨                          | 필요한 내용을 자신의 새 채팅에서 사용합니다.          |
| 링크를 삭제했는데 상대방이 내용을 가지고 있음        | 이미 캡처하거나 복사본을 만든 상태                    | 상대방에게 별도로 폐기와 사용 중단을 요청합니다.        |
| 공유 링크와 외부 링크를 혼동함                | 두 링크의 목적을 구분하지 못함                      | `chatgpt.com/share` 형식인지 확인합니다.    |
| 이미지가 포함된 Enterprise 대화를 공유할 수 없음 | 이미지가 있는 대화에 공유 제한이 적용됨                 | 텍스트 중심의 새 대화를 만들어 공유합니다.           |
| 데이터 분석 대화에서 원본 파일을 받을 수 없음       | 공유 대화와 원본 파일 접근 권한이 분리됨                | 승인된 별도 파일 공유 경로를 사용합니다.            |
| 오래된 링크가 계속 사용됨                   | 링크 생애주기를 관리하지 않음                       | 프로젝트 종료 시 공유 링크 목록을 점검하고 삭제합니다.    |

## 19. 실습하기

이번 강의에서는 테스트 대화를 공유하고, 받는 사람의 화면에서 확인한 뒤, 공유 링크를 삭제해 보겠습니다.

### 19.1 공유할 테스트 대화 준비하기

실제 기밀 정보가 없는 ChatGPT 대화를 하나 준비합니다.

기존에 만들어 둔 실습 대화를 사용하거나, 새 채팅에서 아래 프롬프트를 실행합니다.

<pre><code>나는 마케팅팀 실무자입니다.

신제품 출시 캠페인 기획 회의에서 사용할 아이디어 초안을 만들려고 합니다.

가상의 제품명은 "AlphaNote"입니다.
타깃 고객은 중소기업 영업팀장입니다.
목적은 제품 인지도 향상입니다.

다음 형식으로 캠페인 아이디어를 작성해 주세요.

1. 핵심 메시지
2. 타깃 고객의 문제
3. 캠페인 콘셉트 3개
4. 채널별 실행 아이디어
5. 회의에서 검토할 질문</code></pre>

답변이 생성되면 이어서 아래 프롬프트를 입력합니다.

<pre><code>위 답변을 팀원이 바로 활용할 수 있도록 프롬프트 템플릿 형태로 다시 정리해 주세요.

제품명, 타깃 고객, 캠페인 목적은 대괄호로 비워 주세요.</code></pre>

### 19.2 공유 전 대화 점검하기

다음 내용을 확인합니다.

* 가상 제품만 사용했는가?
* 실제 고객명이나 회사명이 없는가?
* 실제 매출과 예산이 없는가?
* 내부 출시 일정이 없는가?
* 개인정보가 없는가?
* 외부로 공유해도 되는 내용인가?

### 19.3 공유 링크 만들기

다음 두 방법 중 하나를 사용합니다.

#### 방법 1: 사이드바에서 공유하기

1. 대화 제목 오른쪽의 더보기 메뉴를 선택합니다.
2. 공유하기를 선택합니다.
3. 공유 내용을 확인합니다.
4. 링크 복사를 선택합니다.

#### 방법 2: 대화 화면에서 공유하기

1. 공유할 대화를 엽니다.
2. 오른쪽 상단의 공유 버튼을 선택합니다.
3. 공유 내용을 확인합니다.
4. 링크 복사를 선택합니다.

### 19.4 받는 사람의 화면에서 확인하기

복사한 공유 링크를 다른 브라우저 또는 시크릿 창에서 엽니다.

다음 내용을 확인합니다.

* 공유한 대화가 정상적으로 보이는가?
* 공유 시점까지의 질문과 답변이 포함되었는가?
* 불필요한 대화가 함께 보이지 않는가?
* 원본 대화를 편집할 수 없는가?
* 현재 화면에서 대화를 직접 이어가는 기능이 제공되는가?

현재 일반 공유 링크에서는 영상 촬영 당시와 달리 대화를 직접 이어가는 기능이 제공되지 않을 수 있습니다.

따라서 공유 화면은 **대화 열람용**으로 확인합니다.

### 19.5 공유 링크 관리 화면 열기

다음 경로로 이동합니다.

> 계정 아이콘 → 설정 → 데이터 제어 → 공유 링크 → 관리

방금 만든 링크가 목록에 표시되는지 확인합니다.

### 19.6 공유 링크 삭제하기

1. 방금 만든 공유 링크를 찾습니다.
2. 휴지통 또는 링크 삭제 버튼을 선택합니다.
3. 삭제를 확인합니다.
4. 원래 대화가 사이드바에 남아 있는지 확인합니다.
5. 복사한 공유 링크를 다시 열어 접근할 수 없는지 확인합니다.

이 과정을 통해 공유 링크 삭제와 원본 대화 삭제가 서로 다르다는 점을 확인할 수 있습니다.

## 20. 실습 완료 기준

이번 강의의 실습은 다음 기준으로 완료할 수 있습니다.

* 실제 기밀 정보가 없는 테스트 대화를 만들었다.
* 사이드바 또는 대화 화면에서 공유 링크를 생성했다.
* 공유하기 전에 대화의 전체 범위를 확인했다.
* 별도 브라우저 또는 시크릿 창에서 링크를 열었다.
* 공유 시점까지의 대화가 보이는 것을 확인했다.
* 원본 대화를 편집할 수 없다는 점을 확인했다.
* 현재는 공유 링크에서 직접 대화를 이어가는 기능이 중단되었다는 점을 이해했다.
* 설정의 데이터 제어에서 공유 링크 관리 화면을 열었다.
* 공유 링크를 삭제했다.
* 링크를 삭제해도 원본 대화가 유지되는 것을 확인했다.
* 공유 링크를 삭제해도 상대방이 만든 복사본까지 삭제되지는 않을 수 있음을 이해했다.
* 공유 링크와 ChatGPT가 생성한 외부 링크의 차이를 설명할 수 있다.

## 21. 핵심 정리

* ChatGPT 공유 링크는 특정 대화를 URL로 만들어 다른 사람에게 전달하는 기능입니다.
* 공유 링크는 화면 캡처나 복사·붙여넣기보다 대화의 맥락을 온전하게 전달하는 데 유리합니다.
* 일반 공유 링크는 링크를 가진 사람이 대화를 볼 수 있으므로 공개 가능한 내용만 공유해야 합니다.
* 공유 링크에는 일반적으로 링크를 만든 시점까지의 대화 스냅숏이 포함됩니다.
* 공유 후 원본 대화에 추가한 메시지는 기존 링크에 자동으로 반영되지 않습니다.
* 영상 촬영 당시와 달리 현재 일반 공유 링크에서는 대화를 직접 이어가는 기능이 중단되었습니다.
* 공유 링크와 ChatGPT 답변 안의 외부 생성 링크는 목적과 위험이 다릅니다.
* 공유 전에는 개인정보, 고객 정보, 계약 조건, 내부 전략 등의 포함 여부를 확인해야 합니다.
* 공유 링크는 설정의 데이터 제어 메뉴에서 확인하고 삭제할 수 있습니다.
* 공유 링크를 삭제해도 원래 ChatGPT 대화는 그대로 유지됩니다.
* 상대방이 이미 복사하거나 가져간 대화 내용까지 공유 링크 삭제로 제거되지는 않을 수 있습니다.
* Enterprise 공유 링크는 같은 워크스페이스 구성원으로 접근이 제한될 수 있습니다.
* 완성된 대화를 전달할 때는 공유 링크, 공동 논의에는 그룹 채팅, 장기 자료 관리에는 프로젝트가 적합합니다.
* 안전한 공유는 링크 생성으로 끝나는 것이 아니라 공유 전 점검, 공유 범위 관리, 링크 회수까지 포함합니다.
