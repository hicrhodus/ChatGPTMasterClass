# 05-2. 웹 검색 (Web Search)

## 1. 이 강의에서 배울 내용

실무에서는 계속해서 최신 정보를 확인해야 합니다.

경쟁사의 신제품 출시, 산업 규제 변화, 고객사 뉴스, 환율, 원자재 가격, 정책 발표, 보안 취약점, 제품 가격과 재고처럼 업무 판단에 영향을 주는 정보는 수시로 바뀝니다.

기존 검색 방식에서는 사용자가 검색어를 입력하고, 여러 링크를 열고, 필요한 내용을 복사한 뒤 서로 다른 출처를 비교해야 했습니다. 이후 다시 표나 보고서 형태로 정리하는 과정도 필요했습니다.

ChatGPT의 웹 검색 기능을 사용하면 검색과 정리 과정을 하나의 대화 안에서 진행할 수 있습니다.

다만 웹 검색을 사용했다고 해서 결과가 자동으로 정확해지는 것은 아닙니다.

출처가 실제로 존재하는지, 정보의 날짜가 맞는지, 제시된 숫자가 원문과 일치하는지 직접 확인해야 합니다.

이 강의를 통해 다음 내용을 익힐 수 있습니다.

* ChatGPT의 웹 검색 기능이 무엇인지 이해할 수 있습니다.
* 웹 검색이 필요한 질문과 필요하지 않은 질문을 구분할 수 있습니다.
* Knowledge Cutoff와 최신성 리스크를 이해할 수 있습니다.
* 자동 검색과 직접 검색의 차이를 이해할 수 있습니다.
* 역할, 맥락, 과제, 출력 형식, 검증 조건으로 검색 프롬프트를 작성할 수 있습니다.
* 기간, 지역, 대상, 출처 유형으로 검색 범위를 제한할 수 있습니다.
* 답변에 포함된 인라인 인용과 출처 목록을 확인할 수 있습니다.
* 출처의 날짜, 수치, 원문 맥락과 신뢰성을 검증할 수 있습니다.
* 확인된 사실, 추론, 확인 필요 사항을 구분할 수 있습니다.
* 검색 결과를 업무 보고용 리서치 메모로 변환할 수 있습니다.
* 검색 프롬프트에 회사의 민감한 정보를 입력하지 않는 기준을 이해할 수 있습니다.

## 2. ChatGPT 웹 검색이란 무엇인가

ChatGPT 웹 검색은 사용자의 질문과 관련된 최신 정보를 웹에서 찾고, 여러 자료의 내용을 읽은 뒤 하나의 답변으로 정리하는 기능입니다.

일반적인 검색 엔진은 검색어와 관련된 링크 목록을 주로 제공합니다.

사용자는 링크를 하나씩 열고, 필요한 내용을 찾고, 여러 자료를 직접 비교해야 합니다.

반면 ChatGPT 웹 검색은 다음 과정을 하나의 대화 안에서 수행할 수 있습니다.

1. 질문의 검색 의도를 파악합니다.
2. 필요한 검색어를 구성합니다.
3. 웹에서 관련 자료를 찾습니다.
4. 여러 출처의 내용을 읽습니다.
5. 핵심 내용을 종합합니다.
6. 표, 목록, 보고서 등의 형태로 정리합니다.
7. 사용한 출처를 답변에 연결합니다.

즉, 단순히 정보를 찾아주는 기능이 아니라 검색 결과를 사용자의 업무 목적에 맞게 구조화하는 기능입니다.

| 일반적인 검색 엔진       | ChatGPT 웹 검색         |
| ---------------- | -------------------- |
| 관련 링크 목록을 제공     | 링크 내용을 읽고 답변으로 종합    |
| 검색어를 사용자가 직접 설계  | 자연어로 업무 목적을 설명 가능    |
| 여러 페이지를 직접 비교    | 여러 출처의 내용을 비교해 정리    |
| 결과를 다시 문서로 작성    | 표와 보고서 초안으로 바로 변환    |
| 검색 결과의 판단은 사용자 몫 | 판단에 필요한 시사점과 질문까지 제안 |

그러나 최종 판단의 책임까지 ChatGPT가 대신하는 것은 아닙니다.

ChatGPT는 자료를 수집하고 정리하는 보조 도구이며, 중요한 정보의 검증과 업무 판단은 사용자가 수행해야 합니다.

## 3. 웹 검색이 필요한 질문 구분하기

모든 질문에 웹 검색이 필요한 것은 아닙니다.

웹 검색 여부를 판단할 때는 다음 질문을 사용하면 됩니다.

> 이 정보는 시간이 지나면서 바뀔 수 있는가?

가격, 정책, 일정, 인사, 제품 기능처럼 변경될 가능성이 있다면 웹 검색이 필요합니다.

반대로 일반적인 개념, 오래전에 확정된 역사적 사실, 사용자가 제공한 문서의 요약처럼 최신성이 중요하지 않은 작업은 반드시 검색할 필요가 없습니다.

| 질문 유형          | 웹 검색 필요성 | 이유               |
| -------------- | -------- | ---------------- |
| 오늘의 환율         | 매우 높음    | 실시간으로 변동         |
| 현재 주가          | 매우 높음    | 시장 상황에 따라 변동     |
| 최근 경쟁사 뉴스      | 높음       | 새로운 발표와 기사 확인 필요 |
| 최신 법률과 규제      | 높음       | 개정 여부 확인 필요      |
| 현재 제품 가격       | 높음       | 판매처와 시점에 따라 변동   |
| 소프트웨어의 최신 기능   | 높음       | 업데이트에 따라 변경      |
| 일반적인 마케팅 개념    | 낮음       | 비교적 안정적인 지식      |
| 과거 역사적 사건      | 낮음       | 사실이 크게 변하지 않음    |
| 사용자가 첨부한 문서 요약 | 낮음       | 제공된 자료를 기준으로 처리  |
| 글쓰기와 문장 수정     | 낮음       | 외부 최신 정보가 필수가 아님 |

실무에서는 다음 표현이 포함된 질문을 웹 검색 후보로 볼 수 있습니다.

* 오늘
* 현재
* 최근
* 최신
* 이번 주
* 이번 달
* 올해
* 지금 판매 중인
* 현재 시행 중인
* 최근 발표된
* 최신 버전의

이런 표현이 들어간 경우에는 답변이 언제 기준인지 확인해야 합니다.

## 4. Knowledge Cutoff와 최신성 리스크

AI 모델은 일정 시점까지 수집된 자료를 바탕으로 학습됩니다.

모델이 학습한 지식의 기준 시점을 일반적으로 **Knowledge Cutoff**라고 합니다.

Knowledge Cutoff 이후에 발생한 사건이나 변경 사항은 모델 내부 지식에 포함되지 않을 수 있습니다.

예를 들어 다음 정보는 계속 달라질 수 있습니다.

* 국가와 기업의 주요 인사
* 제품 가격과 재고
* 소프트웨어 기능과 메뉴
* 법률과 규정
* 기업의 투자와 인수합병
* 신제품 출시
* 정책과 지원 사업
* 스포츠 경기 결과
* 환율과 주가
* 여행지 운영 시간
* 행사 일정

따라서 모델이 매우 자신 있게 답변하더라도 최신 정보라고 단정할 수 없습니다.

웹 검색이 필요한 질문에서는 답변에 기준일을 포함하도록 요청하는 것이 좋습니다.

<pre><code>웹 검색을 사용하여 현재 기준의 정보를 확인해 주세요.

답변 첫 부분에 조사 기준일을 표시하고,
각 항목에는 자료의 발표일 또는 수정일을 함께 작성해 주세요.</code></pre>

이 요청을 사용하면 오래된 자료와 최근 자료가 섞이는 문제를 줄일 수 있습니다.

## 5. 자동 검색과 직접 검색

ChatGPT는 질문의 내용을 보고 웹 검색이 필요한지 판단할 수 있습니다.

예를 들어 `오늘`, `최근`, `현재`, `최신` 같은 표현이 들어가면 자동으로 검색을 수행할 가능성이 높습니다.

하지만 자동 판단이 항상 사용자의 업무 목적과 일치하는 것은 아닙니다.

최신 자료와 출처가 반드시 필요한 업무라면 웹 검색을 명시적으로 요청하는 것이 안전합니다.

### 5.1 자동 검색

다음과 같이 최신성이 명확한 질문을 입력하면 ChatGPT가 검색을 실행할 수 있습니다.

<pre><code>최근 한 달 동안 발표된 생성형 AI 관련 주요 뉴스를 정리해 주세요.</code></pre>

### 5.2 검색 기능 직접 선택

사용 환경에 따라 입력창 주변의 도구 메뉴에서 `웹 검색` 또는 `Search`를 선택할 수 있습니다.

또는 입력창에 `/`를 입력한 뒤 검색 기능을 선택할 수 있습니다.

화면 구성과 메뉴 위치는 업데이트에 따라 달라질 수 있습니다.

따라서 특정 버튼의 위치를 외우기보다 다음 기준으로 확인하는 것이 좋습니다.

* 검색 기능이 활성화되어 있는가?
* 답변에 웹 출처가 표시되는가?
* 조사 기준일이 최근인가?
* 요청한 기간과 지역이 반영되었는가?

### 5.3 프롬프트에서 직접 검색 요청

도구를 선택하지 않더라도 프롬프트에 명확히 지시할 수 있습니다.

<pre><code>웹 검색을 사용하여 조사해 주세요.

모든 핵심 사실에는 클릭 가능한 출처를 연결하고,
출처에서 직접 확인되지 않는 정보는 추정하지 마세요.</code></pre>

업무상 최신 정보가 반드시 필요하다면 도구 선택과 프롬프트 지시를 함께 사용하는 것이 좋습니다.

## 6. 웹 검색 프롬프트의 5가지 요소

좋은 검색 결과는 좋은 질문에서 나옵니다.

다음과 같이 요청하면 범위가 너무 넓습니다.

<pre><code>전기차 배터리 시장을 조사해 주세요.</code></pre>

이 요청에는 지역, 기간, 조사 목적, 결과 형식과 검증 조건이 없습니다.

따라서 일반적인 산업 설명이 나오거나, 오래된 정보와 최근 정보가 섞일 수 있습니다.

웹 검색 프롬프트에는 다음 다섯 가지 요소를 포함하는 것이 좋습니다.

| 요소    | 의미              | 작성 예시              |
| ----- | --------------- | ------------------ |
| 역할    | 어떤 관점으로 분석할지 지정 | 신사업 전략 분석가         |
| 맥락    | 사용자의 상황과 목적 설명  | 시장 진입 가능성 검토       |
| 과제    | 조사할 내용 지정       | 투자, 규제, 파트너십 조사    |
| 출력 형식 | 결과의 구조 지정       | 비교표와 핵심 요약         |
| 검증 조건 | 출처와 불확실성 처리 기준  | 공식 자료 우선, 확인 불가 표시 |

### 6.1 역할

역할은 ChatGPT가 어떤 관점으로 자료를 선택하고 해석할지 알려줍니다.

<pre><code>당신은 신사업 전략 분석가입니다.</code></pre>

같은 자료라도 다음 역할에 따라 관점이 달라질 수 있습니다.

* 투자 분석가
* 마케팅 전략가
* IT 보안 담당자
* 구매 담당자
* 법무 담당자
* 영업 기획자
* 인사 교육 담당자

### 6.2 맥락

맥락은 사용자가 현재 어떤 상황에 있는지 설명합니다.

<pre><code>저는 전기차 배터리 리사이클링 시장 진입 가능성을 검토하는
기획팀 실무자입니다.</code></pre>

맥락이 없으면 ChatGPT는 자료를 일반적으로 정리합니다.

맥락이 있으면 업무 판단에 필요한 정보에 우선순위를 둘 수 있습니다.

### 6.3 과제

과제는 웹에서 무엇을 조사할지 구체적으로 지정합니다.

<pre><code>북미 지역을 기준으로 최근 6개월 동안의
스타트업 투자 유치, 정책·규제 변화,
대형 기업의 파트너십 동향을 조사해 주세요.</code></pre>

### 6.4 출력 형식

출력 형식을 지정하면 검색 결과를 다시 정리하는 시간을 줄일 수 있습니다.

<pre><code>출력 형식은 다음 표로 작성해 주세요.

| 구분 | 대상 | 날짜 | 핵심 내용 | 확인된 수치 | 업무 영향 | 출처 | 검증 상태 |</code></pre>

### 6.5 검증 조건

검증 조건은 웹 검색 프롬프트에서 가장 중요한 요소입니다.

<pre><code>다음 조건을 지켜 주세요.

- 최근 6개월 자료를 우선 사용합니다.
- 날짜, 수치, 기업명은 출처에서 직접 확인되는 경우만 작성합니다.
- 확인되지 않는 내용은 추정하지 않습니다.
- 확인할 수 없는 항목은 "원천 출처 확인 불가"라고 표시합니다.
- 공식 발표, 공시, 정부 문서와 공식 제품 문서를 우선합니다.
- 모든 핵심 사실에는 클릭 가능한 출처를 연결합니다.</code></pre>

검증 조건이 없으면 ChatGPT가 여러 자료를 종합하는 과정에서 추론한 내용을 사실처럼 표현할 수 있습니다.

## 7. 검색 범위를 제한하는 조건

검색 범위가 넓으면 결과가 많아지지만 업무 관련성은 낮아질 수 있습니다.

검색 프롬프트에는 다음 네 가지 범위 조건을 넣는 것이 좋습니다.

### 7.1 기간

기간은 오래된 자료를 걸러냅니다.

예시는 다음과 같습니다.

* 최근 1개월
* 최근 3개월
* 최근 6개월
* 최근 1년
* 2026년 1월 이후
* 특정 사건 발생 이후

<pre><code>최근 6개월 동안 발표된 자료를 우선 사용해 주세요.</code></pre>

### 7.2 지역

지역은 시장과 제도 환경의 범위를 정합니다.

<pre><code>북미 시장을 기준으로 조사해 주세요.

미국과 캐나다를 구분하고,
지역이 확인되지 않는 자료는 제외해 주세요.</code></pre>

### 7.3 대상

조사할 기업, 제품, 기관 또는 규제의 범위를 제한합니다.

<pre><code>시장 점유율 상위 5개 기업만 비교해 주세요.</code></pre>

### 7.4 출처 유형

출처 유형은 정보의 신뢰성과 원천성을 통제합니다.

<pre><code>다음 출처를 우선 사용해 주세요.

1. 정부와 공공기관 문서
2. 기업 공식 발표와 공시
3. 공식 제품 및 가격 페이지
4. 원문 연구 자료
5. 신뢰도 높은 언론 보도

홍보성 블로그와 출처가 없는 요약 글은 핵심 근거로 사용하지 마세요.</code></pre>

범위를 제한하면 자료의 양은 줄어들지만 결과의 업무 활용성은 높아집니다.

## 8. 실습용 웹 검색 프롬프트

이번 실습에서는 북미 전기차 배터리 리사이클링 시장의 최근 동향을 조사합니다.

웹 검색 기능을 활성화한 뒤 아래 프롬프트를 입력합니다.

<pre><code>당신은 신사업 전략 분석가입니다.

저는 전기차 배터리 리사이클링 시장 진입 가능성을 검토하는
기획팀 실무자입니다.

북미 지역을 기준으로 최근 6개월 동안의
주요 스타트업 투자 유치,
정책·규제 변화,
대형 기업의 파트너십 동향을 조사해 주세요.

다음 조건을 지켜 주세요.

- 최근 6개월 자료를 우선 사용합니다.
- 기업명, 투자 금액, 날짜, 규제명은
  출처에서 직접 확인되는 경우만 작성합니다.
- 확인되지 않는 항목은 추정하지 말고
  "원천 출처 확인 불가"라고 표시합니다.
- 공식 발표, 공시, 정부 문서,
  신뢰도 높은 경제 매체를 우선합니다.
- 모든 핵심 사실에는 클릭 가능한 출처를 연결합니다.
- 서로 다른 출처의 내용이 충돌하면
  충돌 내용을 별도로 표시합니다.

출력 형식은 다음 표로 작성해 주세요.

| 구분 | 대상 | 날짜 | 핵심 내용 | 확인된 수치 | 업무 영향 | 출처 | 검증 상태 |

표 아래에는 다음 내용을 추가해 주세요.

1. 기회 요인 3개
2. 리스크 요인 3개
3. 추가 확인이 필요한 질문 3개</code></pre>

이 프롬프트에는 다음 요소가 모두 포함되어 있습니다.

* 역할: 신사업 전략 분석가
* 맥락: 시장 진입 가능성을 검토하는 기획팀
* 과제: 투자, 규제, 파트너십 조사
* 범위: 최근 6개월, 북미 지역
* 출력 형식: 출처와 검증 상태를 포함한 표
* 검증 조건: 공식 자료 우선, 확인 불가 처리

## 9. 웹 검색 결과 확인하기

검색이 실행되면 ChatGPT는 여러 출처를 조사한 뒤 답변을 생성합니다.

검색에 걸리는 시간은 질문의 범위, 자료의 양, 시스템 상태에 따라 달라질 수 있습니다.

검색 결과에서는 다음 항목을 확인합니다.

* 요청한 기간이 반영되었는가?
* 요청한 지역이 반영되었는가?
* 조사 대상이 과제와 일치하는가?
* 날짜와 수치가 포함되어 있는가?
* 출처가 연결되어 있는가?
* 검증 상태가 구분되어 있는가?
* 확인할 수 없는 정보가 추정되지 않았는가?
* 최종 판단과 원문 사실이 구분되어 있는가?

답변이 일반적인 설명에 그친다면 다음과 같이 재요청합니다.

<pre><code>현재 답변은 일반적인 배경 설명이 많습니다.

웹 검색 결과에서 직접 확인된 최근 6개월의 사건만 남겨 주세요.

각 항목에는 날짜, 원천 출처, 확인된 수치,
업무 영향을 포함해 표로 다시 작성해 주세요.</code></pre>

## 10. 인라인 인용과 출처 목록 확인하기

웹 검색을 사용한 답변에는 출처가 표시될 수 있습니다.

출처는 크게 두 가지 방식으로 확인할 수 있습니다.

### 10.1 인라인 인용

답변의 문장이나 숫자 옆에 출처 표시가 붙을 수 있습니다.

출처 표시를 선택하면 해당 내용에 사용된 웹페이지를 확인할 수 있습니다.

인라인 인용은 다음을 확인할 때 유용합니다.

* 특정 숫자의 근거
* 기업 발표 날짜
* 정책이나 규제의 원문
* 제품 가격
* 통계 자료의 출처

### 10.2 출처 목록

응답 아래의 `Sources` 또는 출처 메뉴를 선택하면 검색에 사용된 자료의 목록을 확인할 수 있습니다.

화면 크기와 사용 환경에 따라 출처 목록은 다음 위치에 표시될 수 있습니다.

* 답변 아래쪽
* 화면 중앙의 팝업 영역
* 화면 오른쪽 패널
* 모바일의 별도 출처 화면

버튼 위치보다 중요한 것은 어떤 자료가 실제로 사용되었는지 확인하는 것입니다.

## 11. 출처가 있다는 것과 검증되었다는 것은 다릅니다

답변에 링크가 있다는 이유만으로 해당 내용이 검증되었다고 판단하면 안 됩니다.

ChatGPT가 원문을 해석하는 과정에서 다음 문제가 발생할 수 있습니다.

* 숫자의 단위를 잘못 해석
* 오래된 자료를 최신 정보처럼 사용
* 발표 예정과 실제 시행을 혼동
* 계획과 확정 사항을 혼동
* 원문의 일부 표현을 과도하게 일반화
* 2차 기사를 원천 자료처럼 사용
* 동일한 보도자료를 인용한 여러 기사를 독립 출처로 판단
* 서로 다른 지역이나 제품 정보를 결합

따라서 중요한 사실은 출처를 직접 열어 확인해야 합니다.

> 출처는 검증의 시작점이며, 검증의 완료를 의미하지 않습니다.

## 12. 원문 대조를 위한 5가지 검증 기준

중요한 출처는 다음 다섯 가지 기준으로 확인합니다.

| 검증 기준  | 확인 질문                       |
| ------ | --------------------------- |
| 접근 가능성 | 링크가 실제로 열리는가?               |
| 최신성    | 자료의 날짜가 요청한 기간에 포함되는가?      |
| 수치 일치  | ChatGPT가 제시한 숫자가 원문과 같은가?   |
| 문맥 일치  | 원문이 실제로 같은 주장을 하는가?         |
| 원천성    | 공식 자료인가, 다른 글을 요약한 2차 자료인가? |

### 12.1 링크가 실제로 열리는가

페이지가 삭제되었거나 잘못된 주소일 수 있습니다.

링크가 열리지 않으면 해당 정보를 보고서의 근거로 사용하지 않습니다.

### 12.2 자료의 날짜가 맞는가

기사 게시일, 문서 작성일, 수정일을 확인합니다.

최근 자료를 요청했는데 오래된 문서가 포함되었다면 별도로 분리합니다.

### 12.3 숫자가 원문과 일치하는가

투자 금액, 시장 규모, 성장률, 가격과 날짜를 직접 비교합니다.

다음 항목도 함께 확인합니다.

* 통화 단위
* 세금 포함 여부
* 연간 또는 월간 기준
* 예상치 또는 확정치
* 전체 시장 또는 특정 지역 수치

### 12.4 원문의 문맥이 같은가

원문에서 가능성으로 언급한 내용을 ChatGPT가 확정된 사실처럼 표현하지 않았는지 확인합니다.

예를 들어 다음 표현은 의미가 다릅니다.

| 원문 표현   | 의미           |
| ------- | ------------ |
| 계획하고 있다 | 확정되지 않을 수 있음 |
| 검토하고 있다 | 의사결정 전 단계    |
| 발표했다    | 공식 공개        |
| 체결했다    | 실제 계약 완료     |
| 예상한다    | 전망 또는 추정     |
| 달성했다    | 실제 결과        |

### 12.5 원천 자료인가

가능하면 다음 순서로 출처를 평가합니다.

1. 정부와 공공기관의 원문
2. 기업 공시와 공식 발표
3. 공식 가격 및 제품 문서
4. 원문 연구 보고서
5. 신뢰도 높은 언론
6. 전문 기관의 분석
7. 일반 블로그와 커뮤니티

출처가 많다고 반드시 좋은 것은 아닙니다.

같은 보도자료를 반복해 인용한 기사 여러 개보다 하나의 공식 발표가 더 강한 근거가 될 수 있습니다.

## 13. 교차 검증하기

의사결정에 중요한 내용은 하나의 출처만 확인하지 않고 다른 독립 출처와 비교하는 것이 좋습니다.

예를 들어 기업의 투자 유치 정보를 확인한다면 다음 자료를 비교할 수 있습니다.

* 기업 공식 보도자료
* 투자사의 공식 발표
* 규제기관 또는 공시 자료
* 신뢰도 높은 경제 매체

교차 검증 요청은 다음과 같이 작성할 수 있습니다.

<pre><code>표에 포함된 핵심 사실을 교차 검증해 주세요.

각 항목마다 가능한 경우 서로 독립적인 출처 두 개를 제시해 주세요.

동일한 보도자료를 재인용한 기사들은
독립적인 출처로 계산하지 마세요.

출처가 하나뿐이라면 "단일 출처"라고 표시해 주세요.</code></pre>

교차 검증 결과는 다음처럼 정리할 수 있습니다.

| 항목    | 1차 출처 | 독립 출처  | 검증 결과 | 주의사항        |
| ----- | ----- | ------ | ----- | ----------- |
| 투자 유치 | 기업 발표 | 투자사 발표 | 확인 완료 | 금액 단위 확인    |
| 정책 시행 | 정부 문서 | 전문 매체  | 확인 완료 | 시행일 확인      |
| 파트너십  | 기업 발표 | 없음     | 단일 출처 | 실제 계약 범위 확인 |

## 14. 확인된 사실과 추론 분리하기

웹 검색 결과에는 원문에서 확인된 사실과 ChatGPT가 해석한 시사점이 함께 들어갈 수 있습니다.

두 내용을 분리하면 업무 보고서의 신뢰성을 높일 수 있습니다.

<pre><code>위 답변을 다음 세 가지로 분리해 주세요.

1. 출처에서 직접 확인된 사실
2. 확인된 사실을 기반으로 한 해석 또는 시사점
3. 추가 확인이 필요한 내용

출력 형식은 다음 표로 작성해 주세요.

| 내용 | 분류 | 근거 출처 | 검증 상태 | 업무 활용 시 주의사항 |</code></pre>

각 분류는 다음처럼 활용합니다.

| 분류        | 활용 기준                   |
| --------- | ----------------------- |
| 확인된 사실    | 출처를 직접 확인한 뒤 보고서 근거로 사용 |
| 해석 또는 시사점 | 분석 의견으로 표시              |
| 추가 확인 필요  | 결론에서 제외하고 리스크로 관리       |

특히 `업무 영향`, `기회 요인`, `리스크 요인`은 원문에 직접 쓰여 있는 사실이 아니라 분석 결과일 수 있습니다.

따라서 사실과 의견을 같은 수준으로 표현하지 않습니다.

## 15. 컨텍스트 표류와 가짜 출처 방지하기

긴 대화에서는 앞에서 사용한 가상 사례나 잘못된 전제가 이후 검색에 영향을 줄 수 있습니다.

이를 **컨텍스트 표류**라고 이해할 수 있습니다.

예를 들어 대화 초반에 가상의 회사명과 투자 사례를 만든 뒤, 같은 대화에서 실제 사례를 검색하도록 요청하면 ChatGPT가 가상 정보와 실제 정보를 혼동할 수 있습니다.

컨텍스트 표류를 줄이려면 다음 방법을 사용합니다.

### 15.1 새 대화에서 검색하기

실제 조사 작업은 가상의 시나리오를 만들었던 채팅과 분리합니다.

### 15.2 실재 여부 확인 요청하기

<pre><code>조사를 시작하기 전에 등장하는 기업, 기관과 규제가
실제로 존재하는지 먼저 확인해 주세요.

실재 여부를 확인할 수 없는 대상은 검색 결과에서 제외해 주세요.</code></pre>

### 15.3 가정과 실제 정보를 구분하기

<pre><code>이 대화에 포함된 가상의 예시와
웹에서 확인한 실제 정보를 명확히 구분해 주세요.

실제 출처가 없는 가상 정보는 결과표에 포함하지 마세요.</code></pre>

### 15.4 출처 없는 항목을 삭제하기

<pre><code>답변의 각 항목을 출처와 다시 연결해 주세요.

출처에서 직접 확인되지 않는 기업명, 날짜, 금액과 규제명은
삭제하거나 "원천 출처 확인 불가"라고 표시해 주세요.</code></pre>

## 16. 부족한 검색 결과 수정하기

### 16.1 출처가 부족한 경우

<pre><code>현재 답변에는 출처가 충분히 연결되어 있지 않습니다.

모든 핵심 문장과 수치에 출처를 다시 연결해 주세요.

출처에서 직접 확인되지 않는 내용은 삭제하거나
"원천 출처 확인 불가"라고 표시해 주세요.</code></pre>

### 16.2 오래된 자료가 섞인 경우

<pre><code>검색 결과에 자료의 기준일이 섞여 있습니다.

최근 6개월 이내의 자료만 본문에 남기고,
그보다 오래된 자료는 "과거 참고 자료"로 분리해 주세요.

각 항목에는 발표일 또는 수정일을 표시해 주세요.</code></pre>

### 16.3 결론이 과도하게 단정적인 경우

<pre><code>현재 결론은 검색 근거보다 단정적으로 표현되어 있습니다.

다음 세 가지로 구분해 다시 작성해 주세요.

1. 출처로 확인된 사실
2. 합리적인 해석
3. 추가 검증이 필요한 가설

확인된 사실만 최종 결론에 포함해 주세요.</code></pre>

### 16.4 출처가 서로 충돌하는 경우

<pre><code>서로 다른 출처에서 내용이 일치하지 않는 항목을 찾아 주세요.

각 항목을 다음 표로 정리해 주세요.

| 쟁점 | 출처 A의 주장 | 출처 B의 주장 | 날짜 차이 | 가능한 원인 | 추가 확인 방법 |

어느 한쪽을 임의로 정답으로 선택하지 마세요.</code></pre>

## 17. 검색 결과를 업무 자료로 변환하기

검색과 검증이 끝났다면 결과를 업무에 사용할 수 있는 형태로 바꿉니다.

검색 결과를 그대로 두면 다음 업무에서 다시 자료를 찾아야 합니다.

따라서 검색한 내용을 한 페이지짜리 리서치 메모로 정리하는 것이 좋습니다.

같은 대화에서 다음 프롬프트를 입력합니다.

<pre><code>지금까지 검증한 검색 결과를 바탕으로
업무 보고용 1페이지 리서치 메모를 작성해 주세요.

# [주제] 리서치 메모

## 1. 결론 요약
- 핵심 결론을 3~5개 bullet로 작성합니다.

## 2. 확인된 핵심 사실

| 핵심 사실 | 기준일 | 근거 출처 | 검증 상태 |

## 3. 업무 영향

| 영향 영역 | 설명 | 권장 조치 |

## 4. 리스크와 확인 필요 사항

| 리스크 | 이유 | 추가 확인 방법 |

## 5. 다음 액션

- 바로 실행할 일
- 추가 조사할 일
- 의사결정권자에게 보고할 일

주의사항:

- 출처가 확인되지 않은 정보는 결론에 넣지 마세요.
- 검증 상태가 "부분 확인" 또는 "출처 불충분"인 항목은
  리스크 섹션으로 분리하세요.
- 확인된 사실과 해석을 명확히 구분해 주세요.</code></pre>

이 구조에서 가장 중요한 점은 검증되지 않은 정보가 결론에 포함되지 않는다는 것입니다.

확인되지 않은 내용은 삭제하는 것이 아니라 리스크와 확인 필요 사항으로 분리합니다.

## 18. 웹 검색을 업무 판단에 활용하기

웹 검색의 가치는 정보를 찾는 데서 끝나지 않습니다.

찾은 정보를 바탕으로 어떤 판단을 해야 하는지 질문할 수 있습니다.

다음 세 가지 활용 수준을 비교해 보겠습니다.

| 활용 수준  | 요청 예시                          | 결과      |
| ------ | ------------------------------ | ------- |
| 정보 수집  | 최근 시장 뉴스를 찾아 주세요               | 자료 목록   |
| 산출물 작성 | 시장 조사 보고서를 작성해 주세요             | 보고서 초안  |
| 판단 지원  | 우리 회사가 진입할 때의 기회와 리스크를 평가해 주세요 | 의사결정 지원 |

단순 자료 수집 이후에는 다음 질문으로 발전시킬 수 있습니다.

<pre><code>검증된 검색 결과만 사용하여
우리 조직의 시장 진입 여부를 판단할 때 고려할 기준을 만들어 주세요.

다음 형식으로 작성해 주세요.

| 판단 기준 | 긍정 신호 | 부정 신호 | 현재 확인된 상태 | 추가 조사 항목 |

확인된 사실과 분석 의견을 구분해 주세요.</code></pre>

다만 ChatGPT의 판단은 의사결정 참고 자료입니다.

실제 투자, 계약, 법률, 인사와 같은 중요한 결정은 관련 담당자의 검토와 승인 절차를 거쳐야 합니다.

## 20. 웹 검색 사용 시 보안과 개인정보

웹 검색 프롬프트에는 검색 목적에 필요하지 않은 민감 정보를 입력하지 않는 것이 좋습니다.

다음 정보는 검색어에 그대로 포함하지 않습니다.

* 실제 고객의 개인정보
* 비공개 계약 금액
* 내부 프로젝트 코드명
* 미공개 제품명
* 내부 시스템 주소
* 직원 인사 정보
* 회사의 기밀 전략
* 계정과 인증 정보
* 공개되지 않은 장애 정보

회사 정보를 포함해야 한다면 다음처럼 익명화합니다.

| 실제 정보    | 검색용 표현         |
| -------- | -------------- |
| 실제 고객사명  | 국내 제조기업 A사     |
| 실제 제품명   | B2B 업무 자동화 솔루션 |
| 실제 계약 금액 | 수억 원 규모        |
| 실제 직원명   | 프로젝트 담당자       |
| 내부 프로젝트명 | 신규 시스템 구축 프로젝트 |

검색 결과가 필요하다는 이유로 외부에 공개되지 않은 정보를 프롬프트에 넣어서는 안 됩니다.

## 21. 흔한 오류와 해결 방법

| 오류 상황           | 원인                   | 해결 방법                       |
| --------------- | -------------------- | --------------------------- |

| 확인되지 않은 기업이 포함됨 | 컨텍스트 표류 또는 추론        | 실재 여부를 먼저 확인하도록 요청합니다.      |

| 정보가 너무 많음       | 조사 범위와 출력 형식이 넓음     | 기간, 지역, 대상 수를 줄입니다.         |
| 가격과 재고가 실제와 다름  | 빠르게 변하는 판매 정보        | 구매 직전에 공식 판매 페이지를 다시 확인합니다. |

## 23. 실습 완료 기준

이번 강의의 실습은 다음 기준으로 완료할 수 있습니다.

* 최신 정보가 필요한 실습 주제를 선정했다.
* 기간, 지역, 대상과 업무 목적을 구체적으로 작성했다.
* 웹 검색 기능을 사용해 자료를 조사했다.
* 역할, 맥락, 과제, 출력 형식과 검증 조건을 프롬프트에 포함했다.
* 답변에 출처와 검증 상태가 포함된 표를 만들었다.
* 주요 출처를 최소 세 개 이상 직접 열었다.
* 자료의 날짜가 요청 기간에 맞는지 확인했다.
* 주요 숫자가 원문과 일치하는지 확인했다.
* 원문과 ChatGPT 답변의 맥락이 같은지 확인했다.
* 공식 자료와 2차 요약 자료를 구분했다.
* 출처에서 확인되지 않는 내용을 별도로 분리했다.
* 확인된 사실과 해석을 구분했다.
* 검색 결과를 업무용 1페이지 리서치 메모로 변환했다.
* 검증되지 않은 정보가 결론이 아니라 리스크에 포함되었는지 확인했다.
* 검색 프롬프트에 민감한 내부 정보가 포함되지 않았는지 점검했다.

## 24. 핵심 정리

* ChatGPT 웹 검색은 최신 웹 자료를 찾아 읽고 사용자의 목적에 맞게 정리하는 기능입니다.
* 모든 질문에 웹 검색이 필요한 것은 아닙니다.
* 가격, 정책, 일정, 뉴스처럼 바뀌는 정보에는 웹 검색이 필요합니다.
* Knowledge Cutoff 이후의 정보는 모델 내부 지식에 없을 수 있습니다.
* 최신 자료가 반드시 필요한 업무에서는 웹 검색을 명확히 요청합니다.
* 검색 프롬프트는 역할, 맥락, 과제, 출력 형식과 검증 조건으로 구성합니다.
* 기간, 지역, 대상과 출처 유형을 제한하면 결과의 관련성이 높아집니다.
* 검증 조건에는 출처 연결과 확인 불가 처리 기준을 포함합니다.
* 답변에 출처가 표시되었다고 자동으로 검증된 것은 아닙니다.
* 출처의 접근 가능성, 날짜, 수치, 문맥과 원천성을 확인해야 합니다.
* 중요한 정보는 서로 독립적인 출처로 교차 검증하는 것이 좋습니다.
* 확인된 사실과 ChatGPT의 해석을 구분해야 합니다.
* 가상의 정보와 실제 검색 결과가 섞이지 않도록 컨텍스트 표류를 주의합니다.
* 검색 결과는 결론, 핵심 사실, 업무 영향, 리스크와 다음 액션으로 구조화할 수 있습니다.
* 확인되지 않은 정보는 결론에서 제외하고 리스크로 분리합니다.
* 웹 검색의 가치는 자료 수집에서 끝나지 않고 업무 판단을 지원하는 데 있습니다.
* 검색 프롬프트에는 고객 정보와 내부 기밀을 그대로 입력하지 않습니다.
* 검증 가능한 업무 자료는 검색, 출처 확인, 원문 대조와 업무 문서 변환의 순서로 완성됩니다.
