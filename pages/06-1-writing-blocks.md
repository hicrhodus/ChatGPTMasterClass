# 06-1. 글쓰기 블록: 답변을 수정 가능한 업무 문서로 다루기

## 1. 이 강의에서 배울 내용

ChatGPT에게 보고서나 이메일 초안을 요청하면 처음부터 완성도가 높은 결과를 받을 수 있습니다.

그러나 실제 업무에서는 첫 번째 결과보다 그다음 과정이 더 중요합니다.

* 문장을 더 간결하게 바꿉니다.
* 특정 문단의 논리를 보완합니다.
* 담당자와 일정을 수정합니다.
* 표현을 조직의 문체에 맞춥니다.
* 여러 번 수정한 내용을 최종본으로 정리합니다.
* 완성된 문서를 복사하거나 파일로 저장합니다.

긴 문서가 일반 채팅 답변으로만 표시되면 수정할 부분을 찾기 어렵고, 작은 부분을 고치기 위해 문서 전체를 다시 생성하게 되는 경우가 많습니다.

ChatGPT의 **글쓰기 블록(Writing Block)** 은 이메일, 보고서, 메시지, 게시글과 같은 초안을 별도의 편집 가능한 영역으로 표시해 이러한 수정 작업을 더 편리하게 만들어 줍니다.

이 강의를 통해 다음 내용을 익힐 수 있습니다.

* 글쓰기 블록이 무엇인지 이해할 수 있습니다.
* 일반 채팅 답변과 글쓰기 블록의 차이를 구분할 수 있습니다.
* 회의 메모를 글쓰기 블록의 보고서 초안으로 변환할 수 있습니다.
* 글쓰기 블록의 제목과 문서 구조를 확인할 수 있습니다.
* 수정할 문단만 선택해 ChatGPT에 부분 수정을 요청할 수 있습니다.
* AI가 제안한 변경 내용을 확인하고 반영할 수 있습니다.
* 글쓰기 블록 안의 문장을 직접 편집할 수 있습니다.
* 수정한 내용을 기준으로 후속 작업을 이어갈 수 있습니다.
* 완성한 문서를 복사하거나 Library에 저장할 수 있습니다.
* 사용 환경에서 제공되는 경우 문서를 파일로 내려받을 수 있습니다.
* Canvas와 글쓰기 블록의 관계를 이해할 수 있습니다.
* 글쓰기 블록을 보고서, 이메일, 회의록과 안내문 작성에 적용할 수 있습니다.

## 2. 첫 번째 답변보다 수정 과정이 중요합니다

ChatGPT를 활용하면 다음과 같은 문서의 초안을 빠르게 만들 수 있습니다.

* 업무 보고서
* 이메일
* 회의록
* 사내 공지
* 고객 안내문
* 제안서
* 업무 매뉴얼
* 교육 안내문
* 소셜 미디어 게시물

문서의 첫 초안은 비교적 빠르게 만들 수 있습니다.

그러나 실무에서 초안을 그대로 사용하는 경우는 많지 않습니다.

일반적으로 다음과 같은 추가 작업이 필요합니다.

1. 누락된 정보를 보완합니다.
2. 불필요한 설명을 줄입니다.
3. 표현을 더 명확하게 바꿉니다.
4. 수신자와 보고 대상에 맞춰 문체를 조정합니다.
5. 특정 문단의 논리를 강화합니다.
6. 담당자와 일정을 확정합니다.
7. 최종 내용을 검토하고 저장합니다.

일반 채팅 답변 안에 긴 문서가 그대로 표시되면 다음과 같은 문제가 발생할 수 있습니다.

* 수정할 문단을 찾기 어렵습니다.
* 일부 내용만 다시 작성시키기 어렵습니다.
* 문서 전체가 반복해서 생성됩니다.
* 수정할 때마다 표현이 다른 부분까지 바뀔 수 있습니다.
* 여러 버전 중 어떤 것이 최신본인지 헷갈립니다.
* 긴 대화창을 위아래로 계속 이동해야 합니다.
* AI가 만든 내용과 사람이 수정한 내용을 함께 관리하기 어렵습니다.

따라서 실무에서는 좋은 첫 답변을 받는 것만큼 **초안을 안정적으로 수정하고 최종본으로 완성하는 과정** 이 중요합니다.

## 3. 글쓰기 블록이란 무엇인가

글쓰기 블록은 ChatGPT가 사용자가 다시 수정하거나 재사용할 가능성이 높은 초안 텍스트를 별도의 편집 가능한 영역으로 표시하는 기능입니다.

OpenAI 공식 한국어 도움말에서는 이를 `작성 블록`으로 번역하기도 합니다. 이 강의에서는 기능의 성격을 더 직관적으로 설명하기 위해 `글쓰기 블록`이라는 용어를 사용합니다.

글쓰기 블록에서는 사용 환경과 문서 유형에 따라 다음과 같은 작업을 할 수 있습니다.

* 문서 내용을 직접 편집합니다.
* 전체 내용을 복사합니다.
* 특정 문장이나 문단만 선택합니다.
* 선택한 부분만 ChatGPT에 수정하도록 요청합니다.
* 문서 전체의 수정을 요청합니다.
* 전체 화면 편집 보기로 엽니다.
* 최근 AI 수정 내용을 실행 취소하거나 다시 실행합니다.
* 지원되는 문서를 Library에 저장합니다.
* 제공되는 경우 이메일 앱이나 다른 서비스로 연결합니다.

실제로 사용할 수 있는 버튼과 작업은 플랜, 기기, 모델, 워크스페이스 정책과 기능 배포 상태에 따라 다를 수 있습니다.

글쓰기 블록은 단순히 긴 답변을 보기 좋게 표시하는 상자가 아닙니다.

> ChatGPT가 만든 초안을 AI와 사람이 함께 수정하는 문서 작업 공간입니다.

## 4. 일반 답변과 글쓰기 블록의 차이

일반 답변과 글쓰기 블록은 사용 목적이 다릅니다.

| 구분       | 일반 채팅 답변      | 글쓰기 블록            |
| -------- | ------------- | ----------------- |
| 주요 목적    | 설명을 읽고 참고     | 문서 초안을 작성하고 수정    |
| 적합한 분량   | 짧은 답변과 대화     | 이메일, 보고서와 긴 문서    |
| 직접 편집    | 일반적으로 어려움     | 블록 안에서 직접 편집 가능   |
| 부분 수정    | 범위를 말로 지정해야 함 | 선택 영역을 기준으로 수정 가능 |
| 전체 화면 편집 | 일반적으로 없음      | 지원되는 경우 사용 가능     |
| 후속 작업    | 새 답변으로 계속 생성  | 같은 문서를 중심으로 수정    |
| 주요 결과    | 참고용 텍스트       | 계속 다듬을 업무 문서      |

다음과 같은 질문은 일반 답변으로 처리하기에 적합합니다.

* 이 용어의 뜻을 설명해 주세요.
* 이 문장의 문제점을 알려 주세요.
* 아이디어 세 개를 제안해 주세요.
* 두 개념의 차이를 알려 주세요.

다음과 같은 요청은 글쓰기 블록에 적합합니다.

* 팀장에게 보낼 이메일 초안을 작성해 주세요.
* 회의 메모를 1페이지 보고서로 만들어 주세요.
* 신입사원용 온보딩 문서를 작성해 주세요.
* 고객 안내문 초안을 작성해 주세요.
* 제안서의 실행 계획 부분을 작성해 주세요.

기준은 결과물을 나중에 다시 고치거나 재사용할 가능성이 있는가입니다.

## 5. 글쓰기 블록의 기본 업무 흐름

글쓰기 블록은 다음 순서로 활용할 수 있습니다.

> 원자료 입력 → 초안 생성 → 구조 검토 → 부분 수정 → 직접 편집 → 최종 검토 → 복사·저장·다운로드

| 단계        | 주요 작업          | 확인 사항             |
| --------- | -------------- | ----------------- |
| 1. 원자료 입력 | 메모와 작성 조건 제공   | 필요한 정보가 빠지지 않았는가? |
| 2. 초안 생성  | 편집 가능한 문서 요청   | 글쓰기 블록으로 표시되는가?   |
| 3. 구조 검토  | 제목과 섹션 확인      | 요청한 구조와 일치하는가?    |
| 4. 부분 수정  | 특정 문단만 AI로 수정  | 다른 부분은 유지되는가?     |
| 5. 직접 편집  | 이름, 일정과 표현 수정  | 실제 정보가 정확한가?      |
| 6. 최종 검토  | 전체 문서 읽기       | 최신 수정이 모두 반영되었는가? |
| 7. 결과 활용  | 복사, 저장 또는 다운로드 | 사용할 문서 형식이 적합한가?  |

이 흐름의 핵심은 문서를 매번 새로 생성하지 않는 것입니다.

같은 초안을 중심으로 필요한 부분만 수정하면서 최종 문서를 완성합니다.

## 6. 글쓰기 블록으로 업무 보고서 초안 만들기

이번 실습에서는 회의 메모를 팀장 보고용 1페이지 업무 보고서로 변환합니다.

다음 프롬프트를 새 채팅에 입력합니다.

<pre><code>다음 회의 메모를 바탕으로
팀장 보고용 1페이지 업무 보고서 초안을 작성해 주세요.

회의 메모:

- 문제: 고객이 첫 구매 단계에서 문의를 많이 남김
- 원인: 안내 메일이 길고 핵심 행동이 명확하지 않음
- 개선안: 첫 메일을 3단계 체크리스트로 단순화
- 담당자: CS팀 김대리, 마케팅팀 이과장
- 일정: 다음 주 수요일까지 개선안 초안 작성

문서 구조:
1. 배경
2. 주요 문제
3. 개선 방향
4. 실행 계획
5. 요청 사항

문체는 간결한 업무 보고체로 작성해 주세요.

완성된 결과는 직접 수정할 수 있는
글쓰기 블록 형태로 작성해 주세요.</code></pre>

마지막 문장의 `글쓰기 블록 형태로 작성해 주세요`는 반드시 필요한 명령은 아닙니다.

ChatGPT가 수정하거나 재사용할 가능성이 높은 문서라고 판단하면 자동으로 글쓰기 블록을 만들 수 있습니다.

다만 일반 답변으로만 표시된다면 편집 가능한 문서 초안을 원한다는 점을 명시하는 것이 도움이 됩니다.

## 7. 생성된 글쓰기 블록 확인하기

초안이 만들어지면 먼저 내용을 수정하지 말고 문서의 기본 구조를 점검합니다.

### 7.1 제목 확인

보고서의 목적을 알 수 있는 제목이 있는지 확인합니다.

예를 들어 다음과 같은 제목이 생성될 수 있습니다.

<pre><code>온라인몰 신규 회원 첫 구매 안내 개선 보고</code></pre>

제목이 너무 일반적이라면 다음과 같이 수정할 수 있습니다.

<pre><code>제목에 온라인몰과 첫 구매 안내 개선이라는
업무 목적이 드러나도록 수정해 주세요.</code></pre>

### 7.2 문서 구조 확인

요청한 다섯 개 섹션이 모두 포함되었는지 확인합니다.

1. 배경
2. 주요 문제
3. 개선 방향
4. 실행 계획
5. 요청 사항

### 7.3 원자료 반영 여부 확인

회의 메모의 다음 내용이 누락되지 않았는지 확인합니다.

* 고객 문의 증가
* 긴 안내 메일
* 불명확한 핵심 행동
* 3단계 체크리스트
* 담당자
* 일정

### 7.4 문체 확인

팀장 보고용 문서라면 다음 기준을 확인합니다.

* 결론이 명확합니다.
* 문장이 지나치게 길지 않습니다.
* 배경 설명이 과도하지 않습니다.
* 실행 계획이 구체적입니다.
* 요청 사항이 분명합니다.
* 모호한 표현보다 업무 용어를 사용합니다.

초안의 구조와 핵심 정보에 문제가 없다면 부분 수정 단계로 이동합니다.

## 8. 글쓰기 블록에서 사용할 수 있는 주요 작업

글쓰기 블록의 상단이나 전체 화면 편집기에는 사용 환경에 따라 다음 작업이 표시될 수 있습니다.

| 작업         | 주요 용도             |
| ---------- | ----------------- |
| 복사         | 전체 내용을 클립보드에 복사   |
| 전체 화면 편집   | 문서를 넓은 화면에서 수정    |
| Library 저장 | 문서를 다시 찾고 재사용     |
| 실행 취소      | 최근 AI 편집 이전으로 이동  |
| 다시 실행      | 취소한 AI 편집을 다시 적용  |
| 다운로드       | 지원되는 형식으로 파일 저장   |
| 직접 편집      | 문서 안의 내용을 사용자가 수정 |
| AI 수정 요청   | 선택 영역 또는 전체 문서 수정 |

모든 버튼이 모든 대화에서 동일하게 표시되는 것은 아닙니다.

기능이 보이지 않는다면 다음을 확인합니다.

* 다른 문서 유형으로 다시 요청합니다.
* 데스크톱 웹 환경에서 확인합니다.
* 전체 화면 편집 보기를 엽니다.
* 사용 중인 모델을 확인합니다.
* 조직 계정의 워크스페이스 정책을 확인합니다.
* 기능이 계정에 배포되었는지 확인합니다.

## 9. 전체 문서를 다시 만들지 않고 일부만 수정하기

글쓰기 블록의 가장 중요한 사용법은 **부분 수정** 입니다.

예를 들어 보고서의 `개선 방향`만 더 구체적으로 바꾸고 싶다고 가정하겠습니다.

문서 전체를 다시 작성하도록 요청하면 다음 문제가 발생할 수 있습니다.

* 잘 작성된 다른 문장까지 바뀝니다.
* 담당자나 일정이 달라질 수 있습니다.
* 이전에 수정한 표현이 사라질 수 있습니다.
* 문서의 전체 길이와 구조가 바뀔 수 있습니다.
* 수정 전후를 비교하기 어렵습니다.

따라서 고칠 문장이나 문단만 선택한 뒤 다음과 같이 요청합니다.

<pre><code>이 문단을 더 실행 중심으로 바꿔 주세요.</code></pre>

조금 더 구체적으로 요청할 수도 있습니다.

<pre><code>선택한 개선 방향 문단만 수정해 주세요.

다음 내용을 포함해 더 실행 중심으로 작성해 주세요.

- 첫 안내 메일을 3단계 체크리스트로 변경
- 각 단계에 고객이 해야 할 행동 명시
- 자주 묻는 질문은 별도 링크로 분리
- 문의량 변화 측정을 위한 기준 포함

나머지 문단은 수정하지 마세요.</code></pre>

수정 범위를 명확히 지정하면 이미 잘 작성된 부분을 유지하면서 필요한 부분만 개선할 수 있습니다.

## 10. AI가 제안한 변경 내용 확인하기

선택한 문단의 수정을 요청하면 사용 환경에 따라 수정 전후의 차이가 표시될 수 있습니다.

예를 들어 다음과 같이 보일 수 있습니다.

* 삭제할 내용은 취소선으로 표시됩니다.
* 새로 작성한 내용은 다른 색상으로 표시됩니다.
* 변경 내용을 수락하거나 취소하는 버튼이 나타납니다.
* 최근 AI 편집을 실행 취소하거나 다시 실행할 수 있습니다.

AI가 제안한 변경 내용을 바로 반영하기 전에 다음을 확인합니다.

* 요청한 문단만 변경되었는가?
* 원래 의미가 유지되었는가?
* 새로운 사실이 임의로 추가되지 않았는가?
* 실제로 실행 가능한 내용인가?
* 기존 담당자와 일정이 유지되었는가?
* 문서 전체의 문체와 어울리는가?

제안 내용이 적절하면 반영합니다.

일부만 마음에 들지 않는다면 다음과 같이 다시 요청합니다.

<pre><code>실행 항목은 유지하되,
고객 문의량을 30% 줄인다는 수치는 근거가 없으므로 삭제해 주세요.</code></pre>

AI의 제안은 초안입니다.

최종 반영 여부는 사용자가 결정해야 합니다.

## 11. 수정 범위를 명확히 지정하는 방법

문서를 부분 수정하는 방법은 크게 두 가지입니다.

### 11.1 문장을 직접 선택하기

글쓰기 블록이나 전체 화면 편집기에서 수정할 문장 또는 문단을 선택합니다.

그다음 다음과 같이 요청합니다.

<pre><code>이 부분을 팀장 보고용으로 더 간결하게 바꿔 주세요.</code></pre>

선택한 영역이 명확하면 `이 부분`이라는 표현을 사용할 수 있습니다.

### 11.2 문단의 위치나 제목을 지정하기

문장을 직접 선택하기 어렵다면 수정할 위치를 말로 지정합니다.

<pre><code>3번 개선 방향 문단만 수정해 주세요.

실행 주체, 실행 기한과 확인 지표가 드러나도록 작성해 주세요.

나머지 문단은 그대로 유지해 주세요.</code></pre>

또는 다음과 같이 지정할 수 있습니다.

* 첫 번째 문단만 수정해 주세요.
* `주요 문제` 섹션만 수정해 주세요.
* 실행 계획 표의 일정 열만 바꿔 주세요.
* 마지막 요청 사항 두 문장만 간결하게 바꿔 주세요.
* 제목과 배경은 유지하고 개선 방향만 다시 작성해 주세요.

수정 범위를 명확히 지정하는 습관은 불필요한 전체 재작성을 줄여 줍니다.

## 12. 사람이 글쓰기 블록을 직접 편집하기

모든 수정 작업을 ChatGPT에 요청할 필요는 없습니다.

이름, 날짜, 숫자와 짧은 표현은 사용자가 직접 수정하는 것이 더 빠르고 정확할 수 있습니다.

예를 들어 실행 계획에 다음 내용이 있다고 가정하겠습니다.

<pre><code>담당자: CS팀 김대리
일정: 다음 주 수요일</code></pre>

실제 담당자와 일정이 변경되었다면 블록 안에서 직접 수정합니다.

<pre><code>담당자: CS팀 최재완
일정: 다음 주 금요일</code></pre>

직접 편집이 적합한 항목은 다음과 같습니다.

* 담당자 이름
* 날짜
* 금액
* 부서명
* 제품명
* 프로젝트명
* 전화번호
* 이메일 주소
* 짧은 단어
* 오탈자

AI 수정이 적합한 작업은 다음과 같습니다.

* 문단 전체의 논리 변경
* 문체 변경
* 분량 축소
* 복잡한 내용 재구성
* 표 형식 변환
* 요약과 확장
* 독자 수준에 맞춘 표현 조정

좋은 작업 방식은 AI와 사람 중 하나만 선택하는 것이 아닙니다.

> AI는 초안과 재구성을 담당하고, 사람은 사실·판단·최종 표현을 책임집니다.

## 13. 직접 편집한 최신 문서로 후속 작업하기

지원되는 글쓰기 블록의 편집 내용은 잠시 후 대화와 함께 저장됩니다.

이후 후속 질문을 하면 ChatGPT가 최신 버전의 내용을 사용할 수 있습니다. 임시 채팅에서는 대화가 끝난 뒤 편집 내용이 유지되지 않을 수 있습니다.

직접 수정한 뒤 다음과 같이 후속 작업을 요청할 수 있습니다.

<pre><code>현재 수정된 최신 문서를 기준으로
팀장에게 보낼 이메일 본문을 작성해 주세요.</code></pre>

<pre><code>현재 최종본을 기준으로
회의에서 설명할 1분 발표문을 작성해 주세요.</code></pre>

<pre><code>현재 문서의 실행 계획을
담당자별 체크리스트로 변환해 주세요.</code></pre>

중요한 변경을 직접 입력한 직후에는 다음과 같이 확인할 수 있습니다.

<pre><code>현재 문서에 반영된 담당자와 일정을 알려 주세요.</code></pre>

ChatGPT의 답변과 직접 수정한 내용을 비교하면 최신 버전이 반영되었는지 확인할 수 있습니다.

## 14. 문서를 복사하고 저장하기

문서 작성이 끝나면 업무에 사용할 수 있는 형태로 보관합니다.

### 14.1 전체 내용 복사

복사 버튼을 사용해 문서 전체를 클립보드에 복사할 수 있습니다.

복사한 내용은 다음 위치에 붙여넣을 수 있습니다.

* Microsoft Word
* Outlook
* Gmail
* Google Docs
* Microsoft Teams
* Slack
* Notion
* 사내 그룹웨어
* 업무 보고 시스템

붙여넣은 뒤 제목, 글꼴, 표와 줄바꿈이 정상적으로 유지되는지 확인합니다.

### 14.2 Library에 저장

지원되는 문서 초안은 Library에 저장할 수 있습니다.

Library는 ChatGPT에서 업로드하거나 생성한 파일을 다시 찾고 활용하는 공간입니다. Library의 제공 범위와 동작은 플랜 및 워크스페이스에 따라 달라질 수 있습니다.

반복해서 사용할 문서는 다음과 같이 구분해 관리하는 것이 좋습니다.

* 주간 보고서 템플릿
* 고객 안내문 템플릿
* 회의록 템플릿
* 프로젝트 종료 보고서
* 교육 안내문
* 표준 이메일

### 14.3 파일로 내려받기

강의 영상에서 사용한 환경에서는 글쓰기 블록의 문서를 Word 파일로 내려받고, 완성된 문서의 제목·섹션·표가 적용된 결과를 확인했습니다.

Canvas의 공식 도움말에서는 일반 문서를 PDF, Markdown과 Word 형식으로 내보낼 수 있다고 안내합니다. 글쓰기 블록의 다운로드 메뉴와 지원 형식은 사용 중인 화면, 모델과 계정 상태에 따라 다르게 나타날 수 있습니다.

다운로드 버튼이 보이지 않는다면 다음 방법을 사용합니다.

1. 전체 내용을 복사합니다.
2. Word나 다른 문서 편집기에 붙여넣습니다.
3. 형식을 점검합니다.
4. 필요한 파일 형식으로 저장합니다.

## 15. 글쓰기 블록의 서식

글쓰기 블록은 일반적으로 다음과 같은 기본 문서 서식을 지원합니다.

* 제목
* 굵은 글씨
* 기울임꼴
* 링크
* 글머리 기호 목록
* 번호 목록
* 체크리스트

지원되는 서식은 일반적인 업무 문서 초안을 작성하는 데 적합합니다.

다음과 같이 문서 형식을 명시할 수 있습니다.

<pre><code>다음 서식을 적용해 주세요.

- 문서 제목은 1단계 제목
- 각 주요 항목은 2단계 제목
- 실행 항목은 번호 목록
- 확인 사항은 체크리스트
- 담당자와 일정은 표로 작성</code></pre>

다만 복잡한 문서 디자인이나 회사의 공식 서식은 Word, PowerPoint 또는 사내 문서 시스템에서 최종 편집하는 것이 적합합니다.

## 16. Canvas와 글쓰기 블록의 관계

Canvas는 긴 글쓰기와 코딩 작업을 위해 별도의 편집 영역을 제공하는 ChatGPT 인터페이스입니다.

글쓰기 블록과 Canvas에는 다음과 같은 공통점이 있습니다.

* 문서 내용을 직접 편집할 수 있습니다.
* 특정 부분을 선택해 수정할 수 있습니다.
* ChatGPT에 문서의 수정과 검토를 요청할 수 있습니다.
* 긴 문서를 하나의 작업 대상으로 다룰 수 있습니다.
* 결과를 복사하거나 내보낼 수 있습니다.

그러나 두 기능은 완전히 같은 것은 아닙니다.

| 구분     | 글쓰기 블록           | Canvas              |
| ------ | ---------------- | ------------------- |
| 기본 위치  | 채팅 응답 안          | 별도의 편집 화면           |
| 주요 목적  | 대화 흐름 안에서 초안 편집  | 긴 글쓰기·코딩 프로젝트       |
| 실행 방식  | 문서 요청 시 자동 생성 가능 | 별도로 열거나 자동 실행       |
| 수정 방식  | 직접 편집과 선택 영역 수정  | 직접 편집, 선택 수정과 전용 도구 |
| 모델 호환성 | 모델과 배포 상태에 따라 다름 | 지원 모델이 제한될 수 있음     |

현재 OpenAI는 글쓰기와 코딩 기능을 채팅 응답 안의 글쓰기 블록과 코드 블록으로도 직접 제공하고 있습니다. Canvas는 지원되는 모델과 환경에서 별도의 편집 인터페이스로 제공될 수 있습니다.

따라서 특정 기능의 이름보다 다음 관점을 기억하는 것이 중요합니다.

> ChatGPT가 만든 결과물을 일회성 답변이 아니라 직접 수정할 수 있는 업무 문서로 다룹니다.

## 17. 글쓰기 블록을 적용하기 좋은 업무

배경, 현황, 문제, 대응 계획과 요청 사항으로 구분하고,

공지 제목, 핵심 안내, 상세 내용과 문의처로 구분해 주세요.</code></pre>

## 18. 부분 수정에 사용할 수 있는 프롬프트

### 18.1 분량 줄이기

<pre><code>선택한 문단만 절반 분량으로 줄여 주세요.

핵심 사실과 결론은 유지하고,
반복되는 설명을 삭제해 주세요.</code></pre>

### 18.2 실행 중심으로 바꾸기

<pre><code>선택한 문단을 실행 중심으로 바꿔 주세요.

담당자, 할 일, 기한과 완료 기준이 드러나게 작성해 주세요.</code></pre>

### 18.3 문체 바꾸기

<pre><code>선택한 문단을 임원 보고용 문체로 바꿔 주세요.

결론을 먼저 제시하고,
문장을 짧고 명확하게 작성해 주세요.</code></pre>

### 18.4 표현을 부드럽게 바꾸기

<pre><code>선택한 문단의 핵심 메시지는 유지하되,
상대방이 방어적으로 받아들이지 않도록
공손하고 중립적인 표현으로 수정해 주세요.</code></pre>

### 18.5 근거와 의견 구분하기

<pre><code>선택한 문단에서
확인된 사실과 작성자의 판단을 분리해 주세요.

확인되지 않은 내용은
"추가 확인 필요"라고 표시해 주세요.</code></pre>

### 18.6 다른 내용은 유지하기

<pre><code>2번 주요 문제 문단만 수정해 주세요.

제목, 배경, 개선 방향, 실행 계획과 요청 사항은

## 19. 문서 품질 검토하기

글쓰기 블록에서 초안을 완성한 뒤 다음 항목을 검토합니다.

### 19.1 사실

* 이름이 정확한가?
* 일정이 정확한가?
* 수치가 원자료와 일치하는가?
* 부서명과 제품명이 정확한가?
* AI가 임의로 추가한 정보가 없는가?

### 19.2 구조

* 결론이 먼저 제시되었는가?
* 제목만 읽어도 문서의 흐름을 알 수 있는가?
* 중복되는 섹션이 없는가?
* 요청 사항이 분명한가?
* 실행 계획을 확인할 수 있는가?

### 19.3 문체

* 보고 대상에 적합한가?
* 지나치게 장황하지 않은가?
* 모호한 표현이 없는가?
* 책임 소재를 과도하게 단정하지 않는가?
* 조직에서 사용하는 용어와 일치하는가?

### 19.4 실행 가능성

* 담당자가 지정되어 있는가?
* 기한이 있는가?
* 완료 기준이 있는가?
* 추가 확인할 정보가 구분되어 있는가?
* 의사결정이 필요한 사항이 드러나는가?

다음 프롬프트로 최종 검토를 요청할 수 있습니다.

<pre><code>현재 문서를 최종 검토해 주세요.

다음 기준으로 문제점을 찾아 주세요.

1. 원자료와 다른 사실
2. 모호한 표현
3. 중복된 설명
4. 실행 주체와 기한이 없는 항목
5. 팀장의 의사결정이 필요한 사항

문서 전체를 바로 다시 쓰지 말고,
먼저 수정 제안 목록만 작성해 주세요.</code></pre>

## 20. 버전 관리와 최종본 보관

글쓰기 블록 안에서 계속 수정하면 현재 문서가 최신본 역할을 할 수 있습니다.

그러나 중요한 업무 문서는 ChatGPT 대화에만 보관하지 않는 것이 좋습니다.

다음과 같은 방식으로 최종본을 별도로 관리합니다.

* 회사의 문서 관리 시스템에 저장합니다.
* Word 또는 PDF 파일로 보관합니다.
* SharePoint나 Google Drive에 저장합니다.
* 파일명에 버전과 날짜를 기록합니다.
* 승인 전 초안과 최종 승인본을 구분합니다.

파일명 예시는 다음과 같습니다.

<pre><code>온라인몰_첫구매안내_개선보고_v0.1_2026-07-24.docx
온라인몰_첫구매안내_개선보고_v1.0_최종.docx</code></pre>

ChatGPT의 글쓰기 블록은 문서를 만드는 작업 공간입니다.

조직의 공식 승인과 문서 관리 절차를 대체하는 시스템은 아닙니다.

## 21. 보안과 개인정보 주의사항

업무 문서를 작성할 때는 입력한 메모와 생성된 초안에 민감한 정보가 포함될 수 있습니다.

다음 정보는 조직의 승인 없이 입력하지 않습니다.

* 고객 개인정보
* 임직원 인사 정보
* 비공개 계약 조건
* 제품 원가와 마진
* 내부 시스템 주소
* 계정과 인증 정보
* 미공개 제품과 전략
* 법률상 보호가 필요한 정보

실제 정보가 필요하지 않은 실습에서는 다음처럼 익명화합니다.

| 실제 정보    | 실습용 표현     |
| -------- | ---------- |
| 실제 고객사명  | 고객사 A      |
| 실제 담당자   | CS팀 김대리    |
| 실제 제품명   | 온라인몰 서비스   |
| 실제 매출액   | 가상의 금액     |
| 내부 프로젝트명 | 신규 개선 프로젝트 |

AI가 만든 문서에는 잘못된 사실이나 과장된 표현이 들어갈 수 있습니다.

최종 문서의 사실, 수치, 책임 표현과 의사결정 내용은 반드시 사람이 검토해야 합니다.

## 23. 실습하기

이번 강의에서는 하나의 보고서 초안을 글쓰기 블록에서 계속 다듬습니다.

### 23.1 초안 생성

아래 프롬프트를 입력합니다.

<pre><code>다음 회의 메모를 바탕으로
팀장 보고용 1페이지 업무 보고서 초안을 작성해 주세요.

회의 메모:
- 주제: 온라인몰 신규 회원 첫 구매 안내 개선
- 문제: 고객이 첫 구매 단계에서 문의를 많이 남김
- 원인: 안내 메일이 길고 핵심 행동이 명확하지 않음
- 개선안: 첫 메일을 3단계 체크리스트로 단순화
- 담당자: CS팀 김대리, 마케팅팀 이과장
- 일정: 다음 주 수요일까지 개선안 초안 작성

문서 구조:
1. 배경
2. 주요 문제
3. 개선 방향
4. 실행 계획

<pre><code>이 문단을 더 실행 중심으로 바꿔 주세요.</code></pre>

변경 전후 내용을 비교하고 적절한 수정안을 반영합니다.

### 23.4 직접 편집

`CS팀 김대리`를 자신의 이름 또는 가상의 담당자로 변경합니다.

일정을 `다음 주 금요일`로 변경합니다.

### 23.5 최신 버전 확인

<pre><code>현재 문서에 반영된 담당자와 일정을 알려 주세요.

문서를 다시 작성하지 말고
현재 내용만 확인해 주세요.</code></pre>

### 23.6 최종 품질 점검

<pre><code>현재 문서를 다음 기준으로 최종 점검해 주세요.

- 사실과 일정의 일치
- 중복 표현
- 모호한 문장

* Library 저장

## 24. 실습 완료 기준

이번 강의의 실습은 다음 기준으로 완료할 수 있습니다.

* 회의 메모를 입력해 업무 보고서 초안을 만들었다.
* 결과가 글쓰기 블록으로 표시되는지 확인했다.
* 문서의 제목과 다섯 개 섹션을 확인했다.
* 원자료의 핵심 내용이 누락되지 않았는지 확인했다.
* `개선 방향` 문단만 선택해 부분 수정을 요청했다.
* AI가 제안한 변경 전후 내용을 검토했다.
* 적절한 변경 내용을 반영했다.
* 담당자와 일정을 직접 수정했다.
* ChatGPT가 최신 수정 내용을 사용하는지 확인했다.
* 최종 문서의 사실과 표현을 검토했다.
* 완성한 문서를 복사하거나 저장했다.
* 일반 답변과 글쓰기 블록의 차이를 설명할 수 있다.
* Canvas와 글쓰기 블록의 관계를 설명할 수 있다.
* 전체 재생성보다 부분 수정이 유리한 이유를 설명할 수 있다.

## 25. 핵심 정리

* 글쓰기 블록은 이메일, 보고서와 문서 초안을 편집할 수 있는 작업 영역입니다.
* 글쓰기 블록은 긴 답변을 보기 좋게 표시하는 상자가 아니라 직접 고쳐 쓰는 문서 공간입니다.
* 일반 답변은 읽고 참고하는 데 적합하고, 글쓰기 블록은 수정과 재사용이 필요한 문서에 적합합니다.
* 실무에서는 첫 번째 답변보다 내용을 수정하고 최종본으로 완성하는 과정이 더 중요합니다.
* 문서 전체를 반복해서 생성하면 이미 잘 작성된 부분까지 바뀔 수 있습니다.
* 수정할 문단을 선택하거나 제목과 위치를 지정하면 필요한 부분만 바꿀 수 있습니다.
* 부분 수정 요청에는 `나머지 문단은 유지해 주세요`라는 조건을 추가할 수 있습니다.
* AI가 제안한 수정은 반영하기 전에 사실, 의미와 실행 가능성을 검토해야 합니다.
* 이름, 일정과 숫자처럼 짧고 정확해야 하는 정보는 사람이 직접 편집하는 것이 효율적입니다.
* AI는 초안과 재구성을 담당하고, 사람은 사실과 최종 판단을 책임집니다.
* 지원되는 글쓰기 블록의 편집 내용은 잠시 후 대화에 저장되며 후속 작업에 활용할 수 있습니다.
* 임시 채팅에서는 편집 내용이 대화 종료 후 유지되지 않을 수 있습니다.
* 완성된 문서는 복사하거나, 지원되는 경우 Library에 저장할 수 있습니다.
* 다운로드 버튼과 지원 형식은 사용 환경에 따라 달라질 수 있습니다.
* Canvas와 글쓰기 블록은 모두 문서 편집을 지원하지만 표시 위치와 지원 모델이 다를 수 있습니다.
* 중요한 것은 기능 이름보다 ChatGPT의 답변을 수정 가능한 업무 문서로 다루는 관점입니다.
* 중요한 최종 문서는 조직의 공식 문서 관리 시스템에 별도로 저장해야 합니다.
* 글쓰기 블록의 기본 흐름은 `초안 생성 → 부분 수정 → 직접 편집 → 최종 검토 → 저장`입니다.
