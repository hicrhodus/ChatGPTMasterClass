# 04-3. 임시 채팅: 기록 없이 ChatGPT 사용하기

## 1. 이 강의에서 배울 내용

이번 강의에서는 ChatGPT 의 **임시 채팅** 기능을 다룹니다.

ChatGPT 를 업무에 사용하다 보면 기록으로 남기고 싶지 않은 대화가 생깁니다. 고객 이메일 초안을 잠깐 다듬거나, 실험적인 프롬프트를 테스트하거나, 공용 PC 에서 일회성으로 질문해야 할 때가 있습니다.

이럴 때 사용할 수 있는 기능이 임시 채팅입니다.

이 강의를 통해 다음 내용을 익힐 수 있습니다.

* 임시 채팅이 무엇인지 이해할 수 있습니다.
* 일반 채팅과 임시 채팅의 차이를 구분할 수 있습니다.
* 임시 채팅이 차단하는 세 가지 요소를 이해할 수 있습니다.
* 임시 채팅을 켜고 테스트 대화를 실행할 수 있습니다.
* 임시 채팅의 30일 보존 정책과 한계를 이해할 수 있습니다.
* 임시 채팅에서 좋은 결과를 얻었을 때 결과물을 안전하게 백업할 수 있습니다.
* 업무에서 임시 채팅을 언제 사용해야 하는지 판단할 수 있습니다.

## 2. 임시 채팅이 필요한 이유

ChatGPT 는 사용자와 나눈 대화를 여러 방식으로 활용합니다.

일반 채팅에서는 대화가 사이드바에 기록으로 남습니다. 이전 대화를 다시 열어 이어서 작업할 수 있고, 진행 중인 문서나 아이디어를 계속 발전시킬 수 있습니다. 또한 사용자의 설정에 따라 메모리나 개인화 맥락이 답변에 영향을 줄 수 있습니다.

업무에 꾸준히 사용할 때는 이런 기능이 유용합니다.

하지만 모든 대화를 기록으로 남길 필요는 없습니다.

예를 들어 다음과 같은 상황을 생각해 볼 수 있습니다.

* 민감한 이메일 초안을 잠깐 다듬고 싶다.
* 테스트용으로 이상한 프롬프트를 써보고 싶다.
* 평소 업무 스타일과 전혀 다른 역할극 프롬프트를 실험하고 싶다.
* 공용 PC 에서 ChatGPT 를 사용해야 한다.
* 화면 공유 중 개인 대화 기록이 보이는 것이 부담스럽다.
* 나중에 다시 볼 필요가 없는 일회성 질문을 하고 싶다.
* 기존 메모리에 영향을 주지 않고 실험하고 싶다.

이럴 때 임시 채팅을 사용할 수 있습니다.

임시 채팅은 웹 브라우저의 시크릿 모드와 비슷하게 이해하면 됩니다. 일반 대화 기록과 분리된 일회성 대화 공간을 만드는 기능입니다.

다만 임시 채팅이 완전한 보안 금고라는 뜻은 아닙니다. 이 점은 뒤에서 다시 다룹니다.

## 3. 임시 채팅의 세 가지 핵심 원칙

임시 채팅은 크게 세 가지를 차단합니다.

| 구분       | 의미                               |
| -------- | -------------------------------- |
| 히스토리 차단  | 대화가 사이드바의 채팅 기록에 남지 않습니다.        |
| 메모리 차단   | 기존 메모리를 읽지 않고, 새 메모리도 만들지 않습니다.  |
| 모델 학습 차단 | 해당 대화가 모델 개선용 학습 데이터로 사용되지 않습니다. |

이 세 가지가 임시 채팅의 핵심입니다.

정리하면 다음과 같습니다.

> 임시 채팅은 기록, 메모리, 모델 학습에서 분리된 1회성 대화 모드입니다.

하지만 여기서 오해하면 안 되는 부분이 있습니다.

임시 채팅은 대화가 사용자 화면의 기록에 남지 않도록 해주는 기능입니다. 그렇다고 해서 입력한 데이터가 즉시 완전히 삭제된다는 의미는 아닙니다. 안전과 악용 방지를 위해 일정 기간 보존될 수 있습니다.

따라서 임시 채팅은 **프라이버시와 기록 분리를 위한 기능** 으로 이해해야 합니다.

## 4. 일반 채팅과 임시 채팅 비교

일반 채팅과 임시 채팅은 사용 목적이 다릅니다.

| 구분         | 일반 채팅                 | 임시 채팅              |
| ---------- | --------------------- | ------------------ |
| 사이드바 기록    | 남음                    | 남지 않음              |
| 나중에 다시 열기  | 가능                    | 불가능                |
| 같은 대화 이어가기 | 가능                    | 창을 닫기 전까지만 가능      |
| 메모리 사용     | 설정에 따라 사용 가능          | 사용하지 않음            |
| 새 메모리 생성   | 설정에 따라 가능             | 생성하지 않음            |
| 모델 학습 사용   | 데이터 설정에 따라 가능         | 사용되지 않음            |
| 적합한 상황     | 장기 작업, 정기 업무, 다시 볼 대화 | 일회성 질문, 테스트, 기록 분리 |
| 주의점        | 대화가 쌓이면 관리 필요         | 결과물을 닫기 전에 복사해야 함  |

일반 채팅은 업무 기록을 남기고 계속 이어가는 데 적합합니다.

임시 채팅은 기록과 메모리를 남기지 않고 일회성으로 처리하는 데 적합합니다.

따라서 어떤 대화를 시작하기 전에 먼저 판단해야 합니다.

> 이 대화는 나중에 다시 열어봐야 하는가?
> 아니면 지금만 쓰고 사라져도 되는가?

나중에 다시 필요하다면 일반 채팅을 사용합니다.

지금만 쓰고 사라져도 된다면 임시 채팅을 사용할 수 있습니다.

## 5. 임시 채팅 사용법

### 5.1 임시 채팅 켜기

ChatGPT 화면을 엽니다.

새로운 채팅 화면에서 임시 채팅을 켤 수 있는 메뉴를 찾습니다. 화면 환경에 따라 다음과 같이 표시될 수 있습니다.

* `임시 채팅`
* `Temporary Chat`
* 점선 말풍선 아이콘
* 모델명 또는 새 채팅 관련 메뉴 안의 임시 채팅 토글

임시 채팅을 켜면 일반 채팅과 다른 화면 표시가 나타납니다. 일부 환경에서는 화면 배경이나 상단 표시가 달라지고, 중앙에 `임시 채팅` 이라는 안내가 나타날 수 있습니다.

임시 채팅 화면에는 다음 의미의 안내 문구가 표시될 수 있습니다.

<pre><code>이 채팅은 기록에 표시되거나 모델을 훈련하는 데 사용되지 않습니다.
안전상의 이유로 최대 30일 동안 사본을 보관할 수 있습니다.</code></pre>

이 안내 문구에서 핵심은 두 가지입니다.

* 기록에 표시되지 않습니다.
* 모델 훈련에 사용되지 않습니다.
* 단, 안전 목적상 최대 30일 동안 보관될 수 있습니다.

### 5.2 대화 입력하기

임시 채팅 상태에서도 사용 방법은 일반 채팅과 거의 같습니다.

입력창에 프롬프트를 입력하고 전송하면 됩니다.

실습을 위해 아래 문장을 입력해 보겠습니다.

<pre><code>내 직업은 비밀 요원이야.</code></pre>

이 문장은 실제 정보를 입력하는 것이 아니라, 임시 채팅이 기록과 메모리에 남지 않는지 확인하기 위한 테스트 문장입니다.

ChatGPT 는 이 문장을 진짜 정보로 확정하지 않고, 대화 맥락에 따라 농담이나 가정으로 받아들일 수 있습니다. 이 실습에서 중요한 것은 답변 내용이 아닙니다.

중요한 것은 이 대화가 왼쪽 사이드바에 기록으로 남지 않는다는 점입니다.

### 5.3 임시 채팅 빠져나가기

이제 임시 채팅에서 빠져나와 보겠습니다.

새 채팅을 열거나, 다른 일반 채팅으로 이동합니다. 단축키를 사용한다면 다음 단축키를 사용할 수 있습니다.

<pre><code>Ctrl + Shift + O</code></pre>

새 대화로 이동한 뒤 사이드바를 확인합니다.

방금 임시 채팅에서 나눈 대화가 사이드바에 남아 있지 않아야 합니다.

즉, 사용자는 해당 대화를 다시 열 수 없습니다.

이것이 임시 채팅의 핵심 동작입니다.

## 6. 메모리 차단 확인하기

임시 채팅은 사이드바 기록에 남지 않을 뿐 아니라 메모리에도 저장되지 않습니다.

이를 확인하려면 임시 채팅을 닫은 뒤 일반 새 채팅을 열고 아래처럼 질문해 볼 수 있습니다.

<pre><code>내 직업이 뭐지?</code></pre>

또는 조금 더 정확하게 테스트하려면 고유한 실습용 키워드를 사용하는 것이 좋습니다.

예를 들어 임시 채팅에서 아래 문장을 입력합니다.

<pre><code>실습용 임시 키워드: 청록색 고래 프로젝트

이 문장은 ChatGPT 임시 채팅 실습을 위한 가상 정보입니다.
이 키워드를 기억해 두었다가, 나중에 내가 물어보면 알려주세요.</code></pre>

임시 채팅 안에서는 ChatGPT 가 같은 세션 안의 맥락을 기억할 수 있습니다.

이어서 아래처럼 물어보면 답할 수 있습니다.

<pre><code>방금 입력한 실습용 임시 키워드를 한 문장으로 다시 말해 주세요.</code></pre>

정상적으로는 다음과 비슷하게 답할 수 있습니다.

<pre><code>실습용 임시 키워드는 "청록색 고래 프로젝트"입니다.</code></pre>

이것은 임시 채팅 안에서의 대화 맥락이 유지된다는 뜻입니다.

이제 임시 채팅을 닫고 일반 새 채팅을 엽니다.

그리고 아래처럼 질문합니다.

<pre><code>내가 방금 임시 채팅에서 입력한 실습용 임시 키워드가 무엇이었나요?</code></pre>

정상적인 경우 ChatGPT 는 해당 임시 채팅 내용을 알 수 없다고 답해야 합니다.

이 실습을 통해 두 가지를 확인할 수 있습니다.

1. 임시 채팅 안에서는 같은 세션의 맥락이 유지됩니다.
2. 임시 채팅을 닫으면 일반 채팅에서 그 내용을 기억하지 못합니다.

## 7. 히스토리와 메모리의 차이

임시 채팅을 이해하려면 히스토리와 메모리를 구분해야 합니다.

처음에는 두 개념이 비슷해 보일 수 있지만, 실제로는 다릅니다.

| 구분   | 의미                              | 예시                  |
| ---- | ------------------------------- | ------------------- |
| 히스토리 | 사이드바에 보이는 채팅 기록                 | 이전 대화를 다시 열어보기      |
| 메모리  | ChatGPT 가 사용자에 대해 지속적으로 기억하는 정보 | 사용자의 직업, 선호 형식, 관심사 |

히스토리는 눈에 보이는 대화 로그입니다.

메모리는 ChatGPT 가 사용자에 대해 기억하는 장기 정보에 가깝습니다.

일반 채팅에서는 대화가 히스토리에 남고, 설정에 따라 메모리에 영향을 줄 수 있습니다.

임시 채팅은 이 두 가지를 모두 차단합니다.

* 사이드바 기록에 남지 않습니다.
* 기존 메모리를 사용하지 않습니다.
* 새 메모리를 만들지 않습니다.

따라서 임시 채팅은 메모리 오염을 막는 샌드박스처럼 사용할 수 있습니다.

예를 들어 평소에는 업무용 답변을 받도록 ChatGPT 를 사용하고 있는데, 일시적으로 전혀 다른 역할극 프롬프트를 실험하고 싶다면 임시 채팅이 적합합니다.

## 8. 맞춤 지침은 예외일 수 있습니다

임시 채팅이 모든 개인화 요소를 완전히 끄는 것은 아닙니다.

중요한 예외가 있습니다.

**맞춤 지침** 은 임시 채팅에서도 적용될 수 있습니다.

맞춤 지침은 사용자가 ChatGPT 에게 전역으로 설정해 둔 응답 방식입니다. 예를 들어 다음과 같은 지침이 있을 수 있습니다.

* 항상 한국어로 답변해 주세요.
* 표를 적극적으로 사용해 주세요.
* 전문적인 비즈니스 문체로 답변해 주세요.
* 답변 마지막에 후속 질문을 제안해 주세요.

임시 채팅은 메모리를 차단하지만, 맞춤 지침까지 자동으로 꺼지는 것은 아닐 수 있습니다.

따라서 완전히 백지 상태의 모델 반응을 테스트하고 싶다면 임시 채팅만 켜는 것으로는 부족할 수 있습니다. 이 경우 맞춤 지침도 별도로 확인해야 합니다.

정리하면 다음과 같습니다.

| 항목     | 임시 채팅에서의 동작 |
| ------ | ----------- |
| 채팅 기록  | 남지 않음       |
| 기존 메모리 | 사용하지 않음     |
| 새 메모리  | 생성하지 않음     |
| 모델 학습  | 사용되지 않음     |
| 맞춤 지침  | 적용될 수 있음    |

## 9. 30일 보존 정책의 의미

임시 채팅은 기록이 사라지는 것처럼 보입니다.

하지만 사용자 화면에서 사라지는 것과 OpenAI 시스템에서 즉시 완전히 삭제되는 것은 같은 의미가 아닙니다.

임시 채팅에서 나눈 대화는 사용자 UI 에서는 저장되지 않습니다. 즉, 사이드바에서 다시 열 수 없습니다.

그러나 안전과 악용 방지 목적상 최대 30일 동안 사본이 보관될 수 있습니다.

따라서 임시 채팅은 다음과 같이 이해해야 합니다.

<pre><code>임시 채팅 = 기록 미저장 + 메모리 차단 + 모델 학습 배제

임시 채팅 ≠ 즉시 완전 삭제
임시 채팅 ≠ 절대 보안 공간
임시 채팅 ≠ 기업 1급 기밀 입력 허용</code></pre>

이 기준이 매우 중요합니다.

임시 채팅은 편의성과 프라이버시를 위한 도구입니다. 하지만 기업의 핵심 기밀, 개인정보 원문, 미공개 전략, 보안 등급이 높은 문서를 입력해도 되는 공간은 아닙니다.

## 10. 임시 채팅으로 충분한 상황과 부족한 상황

실무에서는 임시 채팅을 사용할 수 있는 상황과 사용하면 안 되는 상황을 구분해야 합니다.

| 상황                            | 판단                   |
| ----------------------------- | -------------------- |
| 민감도가 낮은 이메일 초안 다듬기            | 임시 채팅 사용 가능          |
| 실제 고객명을 A사, B사로 바꾼 클레임 이메일 작성 | 임시 채팅 사용 가능          |
| 실험적인 프롬프트 테스트                 | 임시 채팅 적합             |
| 기존 메모리를 오염시키고 싶지 않은 역할극 테스트   | 임시 채팅 적합             |
| 공용 PC 에서 일회성 브레인스토밍           | 임시 채팅 적합             |
| 정기 보고서처럼 나중에 다시 열어야 하는 작업     | 일반 채팅 권장             |
| 회사 미공개 소스코드 전체 입력             | 입력 금지 또는 별도 보안 환경 필요 |
| 고객 개인정보 원문 입력                 | 입력 금지                |
| 핵심 기밀 문서 입력                   | 입력 금지                |
| 계약 금액, 인사 정보, 내부 전략 원문 입력     | 비식별화하거나 입력 금지        |

업무 기준으로는 이렇게 판단하면 됩니다.

* 공개되어도 문제가 거의 없는 정보는 일반 채팅 또는 임시 채팅을 사용할 수 있습니다.
* 내부용이지만 비식별화 가능한 정보는 비식별화 후 임시 채팅을 사용할 수 있습니다.
* 고객명, 계약 금액, 인사 정보, 미공개 전략이 포함된 정보는 원문 입력을 피해야 합니다.
* 영업비밀, 개인정보 원문, 보안 등급이 높은 문서는 입력하지 않습니다.

## 11. 데이터 제어 설정과 함께 사용하기

임시 채팅과 함께 확인하면 좋은 설정이 있습니다.

바로 데이터 제어 설정입니다.

일반적으로 ChatGPT 설정의 데이터 제어 영역에서는 내 대화가 모델 개선에 사용될지 여부를 관리할 수 있습니다.

임시 채팅은 기본적으로 모델 학습에 사용되지 않습니다. 하지만 실무에서는 실수로 임시 채팅을 켜지 않고 일반 채팅에 민감한 내용을 입력할 수도 있습니다.

이런 상황에 대비하려면 데이터 제어 설정도 함께 확인하는 것이 좋습니다.

기본 흐름은 다음과 같습니다.

1. 화면 좌측 하단의 계정 아이콘을 클릭합니다.
2. `설정` 으로 이동합니다.
3. `데이터 제어` 항목을 선택합니다.
4. `모두를 위한 모델 개선` 또는 유사한 항목을 확인합니다.
5. 필요에 따라 비활성화합니다.

이 설정을 꺼두면 일반 채팅에서도 모델 개선 학습 사용을 제한할 수 있습니다.

정리하면 실무에서는 다음처럼 이해하면 됩니다.

| 기능        | 역할                       |
| --------- | ------------------------ |
| 임시 채팅     | 특정 대화를 기록과 메모리에서 분리      |
| 데이터 제어 설정 | 계정 차원의 모델 개선 학습 사용 여부 관리 |

두 기능은 목적이 다릅니다.

따라서 더 안전한 업무 사용을 위해서는 임시 채팅과 데이터 제어 설정을 함께 이해해야 합니다.

## 12. 창을 닫기 전 결과물 백업하기

임시 채팅에서 가장 자주 발생하는 실수가 있습니다.

좋은 답변을 받았는데 창을 닫아 버리는 것입니다.

임시 채팅은 일반 채팅처럼 나중에 다시 열 수 없습니다.

창을 닫거나 다른 대화로 이동하면 사용자는 그 대화를 다시 복구할 수 없습니다.

따라서 임시 채팅에서 좋은 결과를 얻었다면 반드시 창을 닫기 전에 복사해야 합니다.

복사해 둘 수 있는 곳은 다음과 같습니다.

* 메모장
* Word
* Notion
* OneNote
* 사내 문서 시스템
* 이메일 초안
* 프로젝트 문서

임시 채팅을 사용할 때는 아래 순서를 습관화하는 것이 좋습니다.

1. 임시 채팅에서 질문합니다.
2. 결과를 확인합니다.
3. 필요한 결과물을 복사합니다.
4. 업무 문서에 붙여넣습니다.
5. 문서에 저장되었는지 확인합니다.
6. 그다음 임시 채팅을 닫습니다.

핵심은 간단합니다.

> 임시 채팅 결과물은 창을 닫기 전에 반드시 복사합니다.

## 13. 업무 예시 1: 고객 클레임 이메일 다듬기

### 13.1 상황

해외 고객에게 클레임 대응 이메일을 보내야 한다고 가정하겠습니다.

이 작업에는 고객 커뮤니케이션이라는 민감한 업무 맥락이 포함됩니다. 실제 고객명과 계약 조건을 그대로 입력하지 않고, 비식별화해서 처리해야 합니다.

### 13.2 프롬프트 예시

<pre><code>나는 영업팀 담당자입니다.
고객에게 보낼 이메일 초안을 작성하려고 합니다.

아래 정보는 모두 비식별화되어 있습니다.

상황:
- 우리 회사: A사
- 고객사: B사
- 이슈: 납품 일정 3일 지연
- 목적: 정중한 사과와 후속 조치 안내

작성 조건:
- 고객과의 관계를 해치지 않는 정중한 표현
- 과도한 법적 책임을 인정하지 않는 표현
- 재발 방지 계획 포함
- 한국어와 영어 버전 모두 작성

출력 형식:
1. 한국어 이메일
2. 영어 이메일
3. 표현상 주의할 점</code></pre>

### 13.3 왜 임시 채팅을 쓰는가

이 작업은 나중에 대화 기록으로 남길 필요가 낮을 수 있습니다.

또한 고객 클레임이라는 업무 맥락이 포함되기 때문에, 실제 정보는 비식별화하고 임시 채팅에서 처리하는 방식이 적합합니다.

단, 실제 고객명, 계약 금액, 제품명, 내부 책임 소재 등은 그대로 입력하지 않습니다.

## 14. 업무 예시 2: 페르소나 프롬프트 테스트

### 14.1 상황

신제품 아이디어를 검토하기 위해 20대 대학생 소비자 페르소나의 반응을 테스트한다고 가정하겠습니다.

이런 실험적 대화는 기존 업무용 메모리에 영향을 주지 않는 것이 좋습니다.

### 14.2 프롬프트 예시

<pre><code>지금부터 당신은 20대 대학생 소비자 페르소나로 응답해 주세요.

상황:

- 새로운 개인 생산성 앱의 콘셉트를 테스트하고 있습니다.

일반 채팅에서 반복하면 기존 업무용 메모리나 대화 기록이 불필요하게 오염될 수 있습니다.

이럴 때 임시 채팅을 사용하면 실험을 안전하게 분리할 수 있습니다.

## 15. 업무 예시 3: 공용 PC 에서 회의 브레인스토밍하기

임시 채팅을 사용하면 해당 회의에서 만든 대화가 기록으로 남지 않습니다.

임시 채팅은 해당 대화 기록을 남기지 않는 기능이지, 공용 PC 의 계정 노출 문제를 모두 해결하는 기능은 아닙니다.

## 16. 임시 채팅 사용 체크리스트

임시 채팅을 사용하기 전에 아래 항목을 확인합니다.

| 체크 항목    | 확인 질문                            |
| -------- | -------------------------------- |
| 기록 필요성   | 이 대화는 나중에 다시 열 필요가 없는가?          |
| 메모리 영향   | 이 대화가 기존 메모리에 영향을 주면 안 되는가?      |
| 정보 민감도   | 입력 정보가 비식별화되어 있는가?               |
| 보존 가능성   | 최대 30일간 보관될 수 있어도 문제가 없는가?       |
| 결과 백업    | 창을 닫기 전에 결과물을 복사할 준비가 되어 있는가?    |
| 외부 도구    | 외부 GPTs 나 액션을 사용하지 않는가?          |
| 입력 금지 정보 | 개인정보 원문이나 기업 1급 기밀이 포함되어 있지 않은가? |

이 체크리스트에서 하나라도 불안한 항목이 있다면 입력 내용을 줄이거나 비식별화해야 합니다.

민감도가 높은 정보라면 임시 채팅이 아니라 별도의 승인된 업무 환경을 사용해야 합니다.

## 18. 실습하기

이번 강의에서는 임시 채팅을 직접 열고, 기록과 메모리에 남지 않는지 확인해 보겠습니다.

### 18.1 준비하기

먼저 결과물을 복사해 둘 문서를 하나 열어 둡니다.

사용할 수 있는 도구는 다음과 같습니다.

* 메모장
* Word
* Notion
* OneNote
* 사내 문서 도구

임시 채팅은 창을 닫으면 복구할 수 없으므로, 실습 결과를 복사해 둘 공간이 필요합니다.

### 18.2 임시 채팅 켜기

ChatGPT 에 접속합니다.

임시 채팅 또는 Temporary Chat 을 켭니다.

화면에 임시 채팅 표시가 나타나는지 확인합니다.

### 18.3 테스트 문장 입력하기

임시 채팅에 아래 문장을 입력합니다.

<pre><code>실습용 임시 키워드: 청록색 고래 프로젝트

이 문장은 ChatGPT 임시 채팅 실습을 위한 가상 정보입니다.
이 키워드를 기억해 두었다가, 나중에 내가 물어보면 알려주세요.</code></pre>

이어서 아래 요청을 입력합니다.

<pre><code>방금 입력한 실습용 임시 키워드를 한 문장으로 다시 말해 주세요.</code></pre>

임시 채팅 안에서는 해당 키워드를 다시 말할 수 있어야 합니다.

### 18.4 체크리스트 만들기

이제 임시 채팅에서 아래 요청을 입력합니다.

<pre><code>방금 실습한 내용을 바탕으로,
임시 채팅을 사용할 때 기억해야 할 핵심 주의사항 5가지를 초보자용 체크리스트로 작성해 주세요.</code></pre>

생성된 결과를 복사해 준비해 둔 문서에 붙여넣습니다.

### 18.5 임시 채팅 종료하기

임시 채팅 창을 닫거나 일반 새 채팅으로 이동합니다.

사이드바에 방금 사용한 임시 채팅이 남아 있는지 확인합니다.

정상이라면 기록이 남지 않아야 합니다.

### 18.6 일반 채팅에서 기억 여부 확인하기

일반 새 채팅을 열고 아래 문장을 입력합니다.

<pre><code>내가 방금 임시 채팅에서 입력한 실습용 임시 키워드가 무엇이었나요?</code></pre>

ChatGPT 가 해당 내용을 알 수 없다고 답하면 정상입니다.

## 19. 실습 완료 기준

이번 강의의 실습은 다음 기준으로 완료할 수 있습니다.

* 임시 채팅을 켜는 위치를 확인했다.
* 임시 채팅 표시가 화면에 나타나는 것을 확인했다.
* 임시 채팅 안에서 테스트 키워드를 입력했다.
* 같은 임시 세션 안에서는 테스트 키워드를 다시 확인했다.
* 임시 채팅에서 생성한 체크리스트를 별도 문서에 복사했다.
* 임시 채팅을 닫은 뒤 사이드바에 기록이 남지 않는 것을 확인했다.
* 일반 새 채팅에서 임시 키워드를 기억하지 못하는 것을 확인했다.
* 임시 채팅은 복구할 수 없다는 점을 이해했다.
* 30일 보존 가능성과 보안상 한계를 이해했다.

## 20. 핵심 정리

* 임시 채팅은 기록을 남기지 않고 1회성으로 ChatGPT 를 사용할 수 있는 대화 모드입니다.
* 임시 채팅은 사이드바의 대화 기록에 남지 않습니다.
* 임시 채팅은 기존 메모리를 읽지 않고, 새 메모리도 만들지 않습니다.
* 임시 채팅의 대화는 모델 개선용 학습 데이터로 사용되지 않습니다.
* 같은 임시 채팅 세션 안에서는 대화 맥락이 유지됩니다.
* 임시 채팅을 닫으면 사용자는 해당 대화를 다시 열 수 없습니다.
* 임시 채팅에서도 맞춤 지침은 적용될 수 있습니다.
* 임시 채팅은 안전 목적상 최대 30일간 보관될 수 있으므로 즉시 완전 삭제나 절대 보안 공간으로 이해하면 안 됩니다.
* 중요한 결과물은 창을 닫기 전에 반드시 복사해야 합니다.
* 고객명, 계약 금액, 개인정보, 내부 기밀, 보안 등급 높은 문서는 임시 채팅에도 원문으로 입력하지 않습니다.
* 임시 채팅은 완전 보안 금고가 아니라, 기록과 메모리 오염을 줄이는 프라이버시 샌드박스입니다.
