# 05-1. 사진 및 파일 추가 — 파일 분석·결과 검증·보안 점검

## 1. 이 강의에서 배울 내용

업무 자료는 대부분 ChatGPT 입력창에 바로 입력할 수 있는 텍스트 형태로만 존재하지 않습니다.

보고서는 PDF로 전달되고, 매출 자료는 Excel이나 CSV 파일로 관리되며, 회의 내용은 화이트보드 사진이나 화면 캡처로 남아 있는 경우가 많습니다.

이런 자료를 ChatGPT에 활용하려면 과거에는 사람이 먼저 자료를 읽고, 필요한 내용을 골라 복사한 뒤, 다시 질문으로 정리해야 했습니다. 이 과정은 시간이 오래 걸리고 일부 정보가 빠지기 쉽습니다.

ChatGPT의 사진 및 파일 추가 기능을 사용하면 문서, 이미지, 데이터를 직접 대화에 첨부하고 그 내용을 기반으로 분석을 요청할 수 있습니다.

이 강의를 통해 다음 내용을 익힐 수 있습니다.

* 사진 및 파일 추가 기능의 의미를 이해할 수 있습니다.
* 파일 추가 버튼, 드래그 앤 드롭, 복사·붙여넣기로 자료를 첨부할 수 있습니다.
* PDF 문서를 구조 파악, 목적 지정 추출, 근거 검증 순서로 분석할 수 있습니다.
* 화이트보드 사진이나 화면 캡처를 표와 실행 과제로 정리할 수 있습니다.
* CSV 또는 Excel 데이터를 점검, 분석, 시각화 순서로 처리할 수 있습니다.
* ChatGPT가 생성한 내용에서 파일 근거와 추론을 구분할 수 있습니다.
* 숫자와 계산 결과를 검증하는 프롬프트를 사용할 수 있습니다.
* 파일을 업로드하기 전에 개인정보와 기업 기밀을 점검할 수 있습니다.
* 개인용 계정과 조직용 업무 환경의 데이터 처리 차이를 이해할 수 있습니다.

## 2. 파일을 직접 추가한다는 것의 의미

사진 및 파일 추가는 단순히 이메일에 첨부 파일을 넣는 것과 같은 기능이 아닙니다.

사용자가 가지고 있는 자료를 ChatGPT가 읽고 분석할 수 있는 **입력값**으로 바꾸는 기능입니다.

예를 들어 다음과 같은 작업을 할 수 있습니다.

* 긴 PDF에서 특정 주제와 관련된 내용만 추출
* 여러 문서의 주장과 근거 비교
* 화이트보드 사진을 회의록으로 변환
* 오류 화면 캡처에서 오류 메시지와 대응 방법 정리
* Excel이나 CSV의 결측치와 중복값 확인
* 월별 매출 추이와 이상 징후 분석
* 분석 결과를 표와 차트로 시각화
* 파일 내용을 보고서, 이메일, 체크리스트로 변환

기존 방식과 파일 추가 방식을 비교하면 다음과 같습니다.

| 기존 방식            | 사진 및 파일 추가 방식          |
| ---------------- | ---------------------- |
| 사람이 문서를 먼저 읽고 요약 | ChatGPT가 문서를 먼저 읽고 구조화 |
| 필요한 부분을 복사해 입력   | 원본 파일을 직접 첨부           |
| 사진의 내용을 직접 옮겨 적음 | 사진 속 글자와 구조를 분석        |
| Excel에서 수작업으로 집계 | 데이터 파일을 점검하고 계산        |
| 결과를 별도 문서로 다시 정리 | 분석 결과를 원하는 산출물로 변환     |

그러나 파일만 올린다고 좋은 결과가 자동으로 만들어지는 것은 아닙니다.

이번 강의 전체를 관통하는 기본 흐름은 다음과 같습니다.

> 파일을 추가하고 → 목적과 출력 형식을 정해 요청하고 → 결과를 검증합니다.

PDF, 이미지, 데이터처럼 자료의 형태는 달라도 이 흐름은 같습니다.

## 3. ChatGPT에 파일을 추가하는 방법

ChatGPT에는 여러 가지 방법으로 파일을 추가할 수 있습니다.

화면 구성은 웹, 모바일 앱, 계정 및 업데이트 상태에 따라 달라질 수 있으므로 버튼의 모양보다 기능의 목적을 기준으로 확인하는 것이 좋습니다.

### 3.1 파일 추가 버튼 사용하기

입력창 주변의 더하기 버튼 또는 클립 모양 버튼을 선택합니다.

메뉴에서 `사진 및 파일 첨부` 또는 이에 해당하는 항목을 선택한 뒤 파일을 찾습니다.

파일이 입력창에 표시되면 프롬프트를 함께 입력하고 전송합니다.

### 3.2 드래그 앤 드롭으로 추가하기

Windows 파일 탐색기나 바탕화면에서 파일을 선택한 뒤 ChatGPT 입력창으로 끌어다 놓을 수 있습니다.

여러 파일을 비교하거나 분석할 때는 필요한 파일을 함께 선택해 드래그할 수도 있습니다.

다만 파일을 너무 많이 한 번에 추가하면 분석 범위가 넓어지고 결과가 산만해질 수 있습니다.

파일마다 목적이 다르다면 대화를 구분하거나, 파일별 분석 후 마지막에 통합하도록 요청하는 것이 좋습니다.

### 3.3 이미지를 복사해서 붙여넣기

이미지는 파일로 저장하지 않고 복사한 뒤 ChatGPT 입력창에 바로 붙여넣을 수도 있습니다.

예를 들어 다음 방식으로 활용할 수 있습니다.

* 화면 캡처 후 `Ctrl + V`
* 웹페이지 이미지 복사 후 붙여넣기
* Excel 표 영역을 그림으로 복사해 붙여넣기
* 오류 메시지 창 캡처 후 붙여넣기

이 방법은 화면에 나타난 내용을 빠르게 질문할 때 편리합니다.

### 3.4 모바일에서 사진 촬영하기

모바일 ChatGPT 앱에서는 기존 사진을 선택하거나 카메라로 직접 촬영해 첨부할 수 있습니다.

다음과 같은 상황에서 유용합니다.

* 회의가 끝난 직후 화이트보드 촬영
* 현장에서 장비나 제품 상태 촬영
* 종이 문서의 특정 부분 촬영
* 손글씨 메모 촬영
* 오류 화면 촬영

단, 사진 배경에 다른 문서나 사람의 개인정보가 함께 찍히지 않았는지 확인해야 합니다.

### 3.5 `@` 메뉴 활용하기

사용 환경에 따라 입력창에서 `@` 기호를 입력하면 파일 첨부나 연결된 자료를 선택할 수 있는 메뉴가 나타날 수 있습니다.

메뉴의 구성은 계정과 기능 업데이트에 따라 달라질 수 있으므로, 표시되는 항목을 확인해 사용합니다.

## 4. 지원 파일과 업로드 제한 이해하기

ChatGPT는 일반적인 문서, 스프레드시트, 프레젠테이션 및 이미지 형식을 지원합니다.

대표적으로 다음과 같은 파일을 활용할 수 있습니다.

| 자료 유형  | 대표 파일          | 주요 활용               |
| ------ | -------------- | ------------------- |
| 문서     | PDF, DOCX, TXT | 요약, 추출, 비교, 검토      |
| 스프레드시트 | XLSX, CSV      | 품질 점검, 계산, 분석, 시각화  |
| 프레젠테이션 | PPTX           | 슬라이드 구조 분석, 내용 요약   |
| 이미지    | PNG, JPG, JPEG | 화면 분석, 글자 추출, 구조 해석 |

2026년 7월 현재 OpenAI 공식 안내에는 다음과 같은 업로드 제한이 명시되어 있습니다.

| 구분         | 공식 안내 기준        |
| ---------- | --------------- |
| 파일당 최대 크기  | 512MB           |
| 텍스트·문서 파일  | 파일당 최대 200만 토큰  |
| CSV·스프레드시트 | 행 크기에 따라 약 50MB |
| 이미지        | 이미지당 20MB       |

업로드 가능한 파일 수와 사용량은 플랜, 기능, 조직 정책 및 시점에 따라 달라질 수 있습니다.

기술적으로 업로드할 수 있는 크기라고 해서 한 번에 분석하는 것이 항상 좋은 것은 아닙니다.

다음과 같은 방법으로 분석 범위를 줄이면 더 안정적인 결과를 얻을 수 있습니다.

* 긴 문서는 장이나 주제별로 분할합니다.
* 불필요한 페이지와 부록을 제거합니다.
* 스프레드시트의 불필요한 열과 행을 삭제합니다.
* 여러 문서는 먼저 개별 분석한 뒤 통합 비교합니다.
* 이미지가 흐리다면 필요한 영역만 잘라 다시 첨부합니다.

## 5. 좋은 파일 분석 요청의 기본 구조

파일을 첨부한 뒤 다음처럼 요청하는 경우가 많습니다.

<pre><code>이 파일 요약해 줘.</code></pre>

이 요청도 결과는 만들 수 있지만, 일반적인 요약에 그칠 가능성이 높습니다.

업무에 필요한 결과를 얻으려면 다음 요소를 포함하는 것이 좋습니다.

| 요소      | 설명                  |
| ------- | ------------------- |
| 역할      | 내가 어떤 업무를 담당하는지     |
| 목적      | 파일을 분석하는 이유         |
| 범위      | 무엇을 찾거나 제외할지        |
| 기준      | 어떤 관점으로 분석할지        |
| 출력 형식   | 표, 목록, 보고서 등 원하는 형태 |
| 불확실성 처리 | 근거가 없을 때 어떻게 표시할지   |

기본 구조는 다음과 같습니다.

<pre><code>나는 [역할]입니다.

첨부한 파일을 [업무 목적]을 위해 분석하려고 합니다.

다음 기준으로 확인해 주세요.

1. [분석 기준 1]
2. [분석 기준 2]
3. [분석 기준 3]

출력 형식은 [원하는 결과물]로 작성해 주세요.

파일에서 근거를 찾기 어려운 내용은 추측하지 말고
"근거 확인 필요"라고 표시해 주세요.</code></pre>

이 구조는 PDF, 이미지, Excel, CSV 등 대부분의 파일 분석 요청에 적용할 수 있습니다.

## 6. PDF 분석의 3단계 원칙

긴 PDF를 분석할 때는 한 번의 요청으로 모든 결과를 얻으려고 하지 않는 것이 좋습니다.

이번 강의에서는 다음 3단계로 진행합니다.

1. 문서 구조 파악
2. 목적에 맞는 내용 추출
3. 근거와 추론 검증

이 세 단계를 사용하면 긴 문서를 처음부터 모두 읽지 않더라도 필요한 내용을 체계적으로 찾을 수 있습니다.

## 7. 1단계: PDF 전체 구조 파악하기

실무에서 200페이지가 넘는 자료를 전달받았다고 가정해 보겠습니다.

바로 세부 내용을 찾기 전에 먼저 문서의 지도를 만드는 것이 좋습니다.

강의에서는 공개 배포된 212페이지 분량의 AI 정부 서비스 사례집을 사용합니다.

파일을 첨부한 뒤 다음 프롬프트를 입력합니다.

<pre><code>첨부한 PDF는 공개 배포된 AI 정부 서비스 사례집입니다.

본문을 읽기 전에 전체 구조를 먼저 파악하려고 합니다.

사례집에 수록된 사례 전체를 다음 표로 정리해 주세요.

| 분야 | 기관 | 서비스명 | 핵심 기능 한 줄 요약 |</code></pre>

이 요청의 목적은 문서 전체를 자세히 요약하는 것이 아닙니다.

다음 내용을 빠르게 확인하는 것입니다.

* 문서에 어떤 분야가 있는지
* 각 분야에 어떤 사례가 있는지
* 사례를 담당한 기관은 어디인지
* 각 사례의 핵심 기능은 무엇인지

이렇게 전체 구조를 먼저 파악하면 다음 질문에서 분석할 범위를 구체적으로 지정할 수 있습니다.

### 7.1 긴 문서에서 일부가 누락되는 경우

문서가 길거나 구조가 복잡하면 일부 사례가 빠질 수 있습니다.

이것을 바로 분석 실패로 판단할 필요는 없습니다.

다음처럼 후속 질문을 할 수 있습니다.

<pre><code>앞서 정리한 사례 목록에서 누락된 사례가 없는지
PDF의 목차와 비교해 다시 확인해 주세요.

누락된 사례가 있다면 기존 표에 추가해 주세요.</code></pre>

또는 분야별로 나누어 요청할 수 있습니다.

<pre><code>이번에는 사회복지 분야의 사례만 다시 확인해 주세요.

각 사례의 기관, 서비스명, 핵심 기능을
PDF에 제시된 순서대로 정리해 주세요.</code></pre>

## 8. 2단계: 분석 목적을 지정해 필요한 내용만 추출하기

전체 구조를 확인했다면 다음에는 자신의 업무 목적에 필요한 사례만 골라냅니다.

<pre><code>나는 우리 조직의 AI 서비스 도입 기획을 맡은 실무자입니다.

첨부한 사례집에서 민원 처리나 행정 효율화와 관련된 사례만 골라 주세요.

각 사례를 다음 기준으로 정리해 주세요.

1. 도입 배경
2. 핵심 성과: 문서에 제시된 수치 포함
3. 우리 조직에 적용할 때 확인해야 할 점
4. 담당자 인터뷰에서 언급된 실무 애로사항

출력 형식은 다음 표로 작성해 주세요.

| 사례 | 기관 | 도입 배경 | 핵심 성과 | 적용 시 확인할 점 | 실무 애로사항 |

문서에서 근거를 찾기 어려운 내용은 추측하지 말고
"근거 확인 필요"라고 표시해 주세요.</code></pre>

이 프롬프트에는 다음 요소가 포함되어 있습니다.

* 역할: AI 서비스 도입 기획 담당자
* 목적: 조직의 AI 서비스 도입 기획
* 범위: 민원 처리와 행정 효율화 사례
* 분석 기준: 배경, 성과, 적용 조건, 애로사항
* 출력 형식: 비교표
* 불확실성 처리: 근거 확인 필요

같은 PDF라도 단순 요약과 목적 중심 분석의 결과는 크게 달라집니다.

단순 요약은 문서가 무엇을 말하는지 알려줍니다.

목적 중심 분석은 그 문서를 내 업무에 어떻게 사용할지 알려줍니다.

## 9. 3단계: 문서 근거와 ChatGPT의 추론 분리하기

ChatGPT의 분석 결과에는 다음 두 종류의 내용이 섞일 수 있습니다.

1. 첨부 문서에서 직접 확인할 수 있는 내용
2. 문서 내용을 바탕으로 ChatGPT가 해석하거나 추론한 내용

추론 자체가 항상 잘못된 것은 아닙니다.

문제는 추론을 문서의 직접적인 사실이나 수치처럼 사용하는 것입니다.

특히 다음 항목은 반드시 검증해야 합니다.

* 매출과 비용
* 도입 성과 수치
* 기간과 일정
* 인원과 비율
* 법률과 규정
* 담당자의 직접 발언
* 계약과 책임 범위
* 원인과 결과의 관계

다음 프롬프트로 근거와 추론을 분리합니다.

<pre><code>위 답변 중 첨부 문서에서 직접 확인 가능한 내용과
ChatGPT가 추론하거나 해석한 내용을 분리해 주세요.

출력 형식은 다음 표로 작성해 주세요.

| 항목 | 문서에서 직접 확인 가능 | 추론 또는 해석 | 확인 필요 여부 |</code></pre>

결과를 검토할 때는 다음 기준을 적용합니다.

| 분류            | 활용 방법                    |
| ------------- | ------------------------ |
| 문서에서 직접 확인 가능 | 원문의 위치와 표현을 확인한 뒤 근거로 활용 |
| 부분적으로 확인 가능   | 원문과 해석을 분리해 작성           |
| 추론 또는 해석      | 아이디어나 가설로만 활용            |
| 확인 필요         | 원문 또는 다른 자료를 추가 확인       |

중요한 의사결정에 사용하는 내용은 ChatGPT의 자기 검토 결과만으로 확정하지 않습니다.

최종적으로 원문과 수치를 직접 확인해야 합니다.

## 10. 이미지 분석: 사진 한 장을 업무 자료로 바꾸기

이미지에는 글자만 있는 것이 아닙니다.

다음과 같은 시각 정보가 함께 포함될 수 있습니다.

* 표와 목록
* 화살표
* 도형과 박스
* 위치 관계
* 강조 표시
* 손글씨
* 색상 구분
* 화면의 경고 문구
* 차트와 그래프

ChatGPT는 이미지 속 텍스트뿐 아니라 이러한 구조를 바탕으로 내용을 설명하고 정리할 수 있습니다.

대표적인 활용 예시는 다음과 같습니다.

* 화이트보드 사진을 회의록으로 변환
* 손글씨 메모를 할 일 목록으로 변환
* 화면 캡처에서 오류 내용 추출
* 조직도나 프로세스 도식 설명
* 그래프의 변화와 이상 구간 설명
* 고객이 보낸 오류 이미지로 답변 초안 작성

## 11. 화이트보드 사진을 회의 정리표로 변환하기

이번 실습에서는 프로젝트 기획 회의에서 작성한 화이트보드 사진을 사용합니다.

목표는 사진의 내용을 단순히 옮겨 적는 것이 아닙니다.

다음 상태를 구분한 회의 정리표를 만드는 것입니다.

* 확정된 내용

* 읽기 어렵거나 확인이 필요한 내용
* 다음 회의 전에 확인할 질문

이미지를 첨부하고 다음 프롬프트를 입력합니다.

<pre><code>첨부한 이미지는 프로젝트 기획 회의에서 작성한 화이트보드 사진입니다.

사진 속 텍스트와 구조를 바탕으로 회의 내용을 정리해 주세요.

읽기 어려운 글자는 추측하지 말고

2. 확정된 일정
3. 검토 중이거나 미확정인 항목
4. 오른쪽 다이어그램이 나타내는 구조 설명
5. 다음 회의 전에 확인해야 할 질문

출력 형식은 다음 표로 작성해 주세요.

| 구분 | 내용 | 일정 | 상태(확정/검토중/확인 필요) |</code></pre>

이 프롬프트에서 중요한 부분은 다음과 같습니다.

* 사진의 텍스트와 구조를 함께 분석하도록 요청합니다.
* 읽기 어려운 손글씨를 함부로 추측하지 않게 합니다.
* 확정과 미확정을 구분합니다.
* 화살표와 박스의 관계도 설명하게 합니다.
* 결과를 업무에 활용할 수 있는 표로 만듭니다.

<pre><code>이미지에서 "교육 일정"으로 인식한 부분은
실제로 "촬영 일정"입니다.

이 수정 사항을 반영해 전체 표를 다시 작성해 주세요.</code></pre>

이처럼 결과를 확인하고 정정하면서 정확도를 높입니다.

<pre><code>위 내용을 프로젝트 회의록 초안으로 다시 작성해 주세요.

다음 구조를 사용해 주세요.

## 5. 확인 필요 사항

파일 분석 결과는 최종 산출물이 아니라 다른 업무 문서의 입력값이 될 수 있습니다.

## 12. 화면 캡처 활용하기

업무 시스템이나 Excel에서 오류가 발생했을 때 오류 문구를 직접 옮겨 적는 대신 화면을 캡처해 질문할 수 있습니다.

예를 들어 다음 자료를 첨부할 수 있습니다.

* Excel 오류 창
* Power BI 새로 고침 오류
* ERP 입력 오류
* 웹사이트 접근 오류
* 프로그램 설정 화면
* 고객이 전달한 오류 화면

다음과 같이 요청할 수 있습니다.

<pre><code>첨부한 이미지는 업무 시스템에서 발생한 오류 화면입니다.

화면에서 직접 확인할 수 있는 정보만 기준으로
다음 내용을 정리해 주세요.

1. 오류 메시지
2. 발생 가능한 원인 후보
3. 사용자가 먼저 확인할 항목
4. 시스템 관리자에게 전달할 정보
5. 추가 확인이 필요한 질문

확인되지 않은 원인을 단정하지 말고
"가능성" 또는 "추가 확인 필요"로 구분해 주세요.</code></pre>

고객지원 업무라면 답변 초안까지 만들 수 있습니다.

<pre><code>위 오류 분석을 바탕으로 고객에게 보낼 1차 답변을 작성해 주세요.

조건:
- 확인되지 않은 원인을 확정적으로 말하지 않습니다.
- 고객이 직접 확인할 수 있는 항목을 단계별로 안내합니다.
- 추가로 필요한 화면이나 정보를 요청합니다.
- 공손하고 간결한 고객지원 문체를 사용합니다.</code></pre>

화면 캡처에는 고객명, 이메일, 계정 정보, 주소, 매출 등의 정보가 함께 보일 수 있으므로 첨부 전에 필요한 부분만 자르거나 가립니다.

## 13. 데이터 파일 분석은 점검부터 시작합니다

PDF와 이미지는 주로 읽고 정리하는 작업입니다.

CSV와 Excel은 실제 숫자를 정리하고 계산하는 작업이 포함됩니다.

따라서 데이터 파일은 분석보다 품질 점검이 먼저입니다.

다음과 같은 데이터 문제는 계산 결과를 왜곡할 수 있습니다.

* 결측치
* 중복 행
* 숫자 열에 포함된 문자
* 쉼표가 포함된 텍스트형 숫자
* 날짜 형식 불일치
* 단위가 다른 값
* 열 이름의 불일치
* 음수나 비정상적으로 큰 값
* 집계에서 제외해야 하는 합계 행

기본 흐름은 다음과 같습니다.

> 데이터 품질 점검 → 문제 정리 → 분석 → 시각화 → 계산 검증

## 14. 1단계: 데이터 품질 점검하기

CSV 또는 Excel 파일을 첨부한 뒤 다음 프롬프트를 입력합니다.

<pre><code>첨부한 CSV 파일을 분석하기 전에
데이터 품질을 먼저 점검해 주세요.

다음 항목을 확인해 주세요.

1. 전체 행 수와 열 수
2. 각 열의 데이터 형식
3. 결측치가 있는 열
4. 중복 행 여부
5. 숫자 열에 문자 값이 섞여 있는지 여부
6. 분석 전에 정리해야 할 문제

출력 형식은 다음 표로 작성해 주세요.

| 점검 항목 | 결과 | 조치 필요 여부 | 권장 조치 |</code></pre>

점검 결과를 통해 다음 내용을 확인할 수 있습니다.

* 어떤 열을 분석에 사용할 수 있는지
* 숫자로 변환해야 하는 열이 있는지
* 빈 값을 제외하거나 대체해야 하는지
* 중복 행을 제거해야 하는지
* 날짜를 월 단위로 변환해야 하는지
* 계산 전에 정제할 문제가 있는지

데이터를 첨부한 즉시 매출 합계부터 요청하면 잘못된 형식의 값이 계산에서 빠지거나 잘못 처리될 수 있습니다.

## 15. 2단계: 데이터 문제를 정리한 뒤 분석하기

품질 점검이 끝나면 발견된 문제를 정리한 상태에서 분석을 요청합니다.

<pre><code>이제 이 매출 데이터를 분석해 주세요.

방금 품질 점검에서 발견된 문제는 정리한 뒤 계산해 주세요.

분석 목적은 월별 매출 추이와 이상 징후를 찾는 것입니다.

다음 분석을 수행해 주세요.

1. 월별 전체 매출 추이
2. 전월 대비 매출이 20% 이상 감소한 지역 또는 제품군
3. 감소 원인으로 의심되는 패턴

출력 형식은 다음 순서로 작성해 주세요.

1. 핵심 지표 표
2. 주요 인사이트
3. 추가 확인 질문</code></pre>

여기서 `감소 원인으로 의심되는 패턴`은 데이터에서 확인 가능한 상관관계나 징후입니다.

이를 실제 원인으로 확정해서는 안 됩니다.

예를 들어 특정 지역의 매출 감소가 확인되더라도 다음과 같은 추가 정보가 필요할 수 있습니다.

* 영업일수 변화
* 거래처 이탈
* 제품 단종
* 가격 인상
* 재고 부족
* 판매 채널 변경
* 일회성 대량 주문
* 데이터 누락

따라서 분석 결과에는 다음 질문도 함께 포함하도록 하는 것이 좋습니다.

<pre><code>데이터만으로 원인을 확정하기 어려운 항목은
가능한 원인 후보와 추가로 필요한 데이터를 구분해 주세요.</code></pre>

## 16. 3단계: 분석 결과 시각화하기

표로 분석한 내용을 차트로 만들면 변화의 시점과 크기를 더 쉽게 확인할 수 있습니다.

<pre><code>월별 전체 매출 추이를 라인 차트로 그려 주세요.

전월 대비 매출이 20% 이상 감소한 지역은
별도의 막대 차트로 비교해 주세요.</code></pre>

자료의 목적에 따라 다음 차트를 선택할 수 있습니다.

| 분석 목적      | 권장 차트          |
| ---------- | -------------- |
| 시간에 따른 변화  | 라인 차트          |
| 항목 간 크기 비교 | 막대 차트          |
| 구성 비중 비교   | 누적 막대 또는 원형 차트 |
| 두 수치의 관계   | 산점도            |
| 값의 분포 확인   | 히스토그램          |
| 이상치 확인     | 박스 플롯          |

차트는 보기 좋게 만드는 것이 목적이 아닙니다.

어떤 의사결정을 돕기 위한 차트인지 함께 요청해야 합니다.

<pre><code>각 차트 아래에 다음 내용을 함께 설명해 주세요.

1. 차트에서 가장 먼저 확인해야 할 변화
2. 의사결정에 중요한 구간
3. 추가 확인이 필요한 이상 징후
4. 이 차트만으로 단정할 수 없는 내용</code></pre>

## 17. 4단계: 계산 과정을 검증하기

매출, 비용, 인원, 비율처럼 의사결정에 사용되는 숫자는 계산 기준을 다시 확인해야 합니다.

다음 프롬프트를 사용합니다.

<pre><code>방금 수행한 분석의 계산 과정을 검증해 주세요.

다음 항목을 확인해 주세요.

1. 계산에 사용한 열 이름
2. 숫자 형식으로 변환한 열
3. 결측치 처리 방식
4. 중복 행 처리 방식
5. 집계 기준
6. 계산에서 제외한 행
7. 결과가 달라질 수 있는 가정

출력 형식은 다음 표로 작성해 주세요.

| 검증 항목 | 적용 방식 | 결과에 미치는 영향 | 추가 확인 필요 여부 |</code></pre>

데이터 분석에서 중요한 것은 그럴듯한 결과가 아니라 **재현 가능한 계산 기준**입니다.

가능하다면 다음 사항도 직접 확인합니다.

* 원본 데이터의 합계
* Excel 피벗 테이블 결과
* 특정 월이나 지역의 표본 행
* 제외된 값
* 결측치 처리 전후의 차이

## 18. 파일 분석 결과를 다른 업무 산출물로 변환하기

분석 결과는 다시 여러 업무 문서로 바꿀 수 있습니다.

| 분석 결과    | 변환 가능한 산출물               |
| -------- | ------------------------ |
| PDF 분석표  | 임원 보고서, 검토 의견서, 리스크 목록   |
| 회의 사진 정리 | 회의록, 액션 아이템, 일정표         |

## 19. 파일을 올리기 전 보안 점검

파일 추가는 강력한 기능이지만, 자료 자체가 외부 서비스에 입력된다는 점을 먼저 고려해야 합니다.

업무 파일에는 다음과 같은 정보가 포함될 수 있습니다.

* 고객 개인정보
* 임직원 인사 정보
* 계약서와 단가
* 매출, 원가, 마진
* 계좌와 결제 정보
* 제품 설계와 BOM
* 소스 코드
* 미공개 전략
* 인수합병 자료
* 의료 정보
* 시스템 접속 정보

기능을 사용할 수 있다는 것과 해당 파일을 업로드해도 된다는 것은 다른 문제입니다.

파일을 올리기 전에 다음 순서로 점검합니다.

### 19.1 개인정보 확인

다음 정보가 포함되어 있는지 확인합니다.

* 이름
* 전화번호
* 이메일 주소
* 주소
* 주민등록번호
* 계좌번호
* 고객 식별 번호
* 직원 번호

필요한 경우 삭제하거나 가명으로 바꿉니다.

### 19.2 기업 기밀 확인

다음 정보가 포함되어 있는지 확인합니다.

* 실제 고객명
* 계약 금액
* 원가와 마진
* 내부 평가 자료
* 제품 출시 일정
* 영업 전략
* 생산 공정
* 기술 문서
* 인증 정보

분석 목적에 필요하지 않은 정보는 제거합니다.

### 19.3 이미지 배경 확인

이미지는 중심 내용 외에도 배경 정보를 포함할 수 있습니다.

* 다른 문서
* 브라우저 탭
* 이메일 주소
* 사용자 계정명
* 시스템 주소
* 고객 이름
* 알림 메시지

필요한 영역만 잘라 첨부하거나 민감한 부분을 가립니다.

### 19.4 계정과 워크스페이스 확인

개인용 계정에서는 설정의 `데이터 제어`에서 `모두를 위한 모델 개선`을 끌 수 있습니다. 이 설정을 끄면 이후 대화는 기록에 남더라도 모델 개선에 사용되지 않습니다.

OpenAI는 ChatGPT Business, Enterprise, Edu 등 조직용 서비스의 입력과 출력을 기본적으로 모델 학습에 사용하지 않는다고 안내합니다. 다만 조직의 자체 보안 정책과 승인 절차가 우선입니다.

### 19.5 업로드 판단 기준

| 자료 상태        | 권장 판단                 |
| ------------ | --------------------- |
| 공개 자료        | 분석 가능                 |
| 가상·샘플 데이터    | 실습에 적합                |
| 비식별화된 내부 자료  | 조직 정책 확인 후 사용         |
| 고객 개인정보 포함   | 제거 전 업로드 금지           |
| 회사 기밀 포함     | 승인된 환경과 절차 없이는 업로드 금지 |
| 보안 등급이 높은 자료 | 일반 ChatGPT 업로드 금지     |

## 21. 흔한 오류와 해결 방법

| 오류 상황             | 원인              | 해결 방법                           |
| ----------------- | --------------- | ------------------------------- |
| PDF 결과가 너무 일반적임   | `요약해 줘`라고만 요청함  | 역할, 목적, 범위, 출력 형식을 추가합니다.       |
| 긴 문서의 일부 내용이 빠짐   | 문서가 길거나 구조가 복잡함 | 장·분야별로 나누어 다시 분석합니다.            |
| 문서에 없는 내용이 포함됨    | 추론과 문서 근거가 섞임   | 근거와 추론을 분리하도록 요청합니다.            |
| 이미지의 글자를 잘못 읽음    | 흐림, 기울어짐, 잘림    | 더 선명한 이미지나 필요한 영역만 다시 첨부합니다.    |
| 손글씨를 임의로 해석함      | 불확실성 처리 지시가 없음  | 읽기 어려운 부분은 `확인 필요`로 표시하게 합니다.   |
| 숫자 합계가 맞지 않음      | 숫자 열이 문자로 처리됨   | 품질 점검과 형식 변환을 먼저 요청합니다.         |
| 데이터 인사이트가 단정적임    | 패턴과 원인을 구분하지 않음 | 원인 후보와 추가 필요 데이터를 분리합니다.        |
| 차트가 실무에 도움이 되지 않음 | 차트 목적을 지정하지 않음  | 추이, 비교, 관계 등 목적을 먼저 지정합니다.      |
| 실제 기밀 파일을 업로드함    | 업로드 전 보안 검토 누락  | 즉시 대화를 삭제하고 조직의 보안 절차를 따릅니다.    |
| 파일 업로드가 되지 않음     | 크기·사용량·지원 형식 제한 | 파일을 분할하거나 크기를 줄이고 지원 형식을 확인합니다. |

* 이미지 파일을 첨부하거나 붙여넣었다.
* 읽기 어려운 부분을 `확인 필요`로 구분했다.

* CSV 또는 Excel 파일의 품질을 먼저 점검했다.
* 데이터 문제를 정리한 뒤 분석을 실행했다.
* 분석 결과를 차트로 시각화했다.

## 24. 핵심 정리

* 사진 및 파일 추가는 업무 자료를 ChatGPT의 분석 입력값으로 바꾸는 기능입니다.
* PDF, 이미지, Excel, CSV 등 자료 유형에 따라 다른 분석 방법을 사용할 수 있습니다.
* 파일을 첨부한 뒤 `요약해 줘`라고만 요청하기보다 역할, 목적, 기준과 출력 형식을 함께 제시해야 합니다.
* 긴 PDF는 구조 파악, 목적 지정 추출, 근거 검증 순서로 분석하는 것이 좋습니다.
* 문서에서 직접 확인할 수 있는 내용과 ChatGPT가 추론한 내용을 분리해야 합니다.
* 이미지 분석에서는 읽기 어려운 글자를 임의로 추측하지 않도록 지시해야 합니다.
* 화면 캡처에는 의도하지 않은 개인정보와 내부 정보가 함께 포함될 수 있습니다.
* 데이터 파일은 분석보다 결측치, 중복, 형식 등의 품질 점검이 먼저입니다.
* 데이터 분석은 점검, 분석, 시각화, 계산 검증 순서로 진행합니다.
* 데이터에서 발견한 패턴과 실제 원인을 구분해야 합니다.
* 중요한 숫자는 계산에 사용한 열, 제외된 행과 처리 기준을 확인해야 합니다.
* 파일 분석 결과는 보고서, 회의록, 이메일과 실행 계획으로 다시 변환할 수 있습니다.
* 업로드할 수 있는 파일과 업로드해도 되는 파일은 다릅니다.
* 개인 계정에는 고객 정보와 회사 기밀을 올리지 않습니다.
* 업무 자료는 조직에서 승인한 환경에서 최소한의 정보만 사용합니다.
* 좋은 파일 분석은 파일을 올리는 것으로 끝나지 않고, 목적이 분명한 요청과 결과 검증으로 완성됩니다.
