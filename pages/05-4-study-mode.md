# 05-4. 공부 모드 (Study Mode)

## 1. 이 강의에서 배울 내용

ChatGPT를 사용하다 보면 모르는 내용을 질문하고 바로 답을 얻는 방식에 익숙해집니다.

답변을 복사해 업무에 사용하면 당장의 문제는 빠르게 해결할 수 있습니다. 하지만 며칠 뒤 비슷한 문제가 생겼을 때 원리나 판단 기준이 기억나지 않아 다시 ChatGPT에 질문하게 되는 경우도 많습니다.

답은 받았지만 그 내용이 자신의 지식과 역량으로 남지 않은 것입니다.

ChatGPT의 **공부 모드(Study Mode)** 는 이런 문제를 줄이기 위한 학습 방식입니다.

공부 모드는 단순히 정답을 제공하는 데 그치지 않고, 질문과 힌트, 단계별 설명과 이해 점검을 통해 사용자가 직접 생각하도록 돕습니다.

이 강의를 통해 다음 내용을 익힐 수 있습니다.

* ChatGPT 공부 모드가 무엇인지 이해할 수 있습니다.
* 일반 채팅과 공부 모드의 차이를 구분할 수 있습니다.
* 공부 모드의 세 가지 핵심 원리를 설명할 수 있습니다.
* 공부 모드를 실행하고 종료할 수 있습니다.
* 학습 주제, 현재 수준, 학습 목표와 업무 적용 상황을 설정할 수 있습니다.
* 정답 요청형 프롬프트를 학습 요청형 프롬프트로 바꿀 수 있습니다.
* ChatGPT가 답을 바로 제공하지 않고 질문과 힌트로 진행하도록 요청할 수 있습니다.
* 배운 내용을 자신의 언어로 다시 설명하고 피드백을 받을 수 있습니다.
* 파일과 이미지를 활용해 사내 문서나 업무 자료를 학습할 수 있습니다.
* 메모리를 활용해 학습 경험을 개인화할 수 있습니다.
* Excel, Power Query, Power BI와 사내 규정 학습에 공부 모드를 적용할 수 있습니다.
* 공부 모드의 결과를 검증하고 업무에 적용하는 방법을 이해할 수 있습니다.

## 2. 공부 모드가 필요한 이유

일반적인 ChatGPT 사용 방식은 질문을 입력하고 완성된 답변을 받는 것입니다.

이 방식은 다음과 같은 업무에 효과적입니다.

* 이메일 초안 작성
* 문장 수정
* 보고서 요약
* 아이디어 생성
* 간단한 사실 확인
* 일정과 체크리스트 작성

당장 처리해야 할 업무가 있다면 완성된 결과를 빠르게 받는 것이 효율적입니다.

그러나 새로운 개념이나 도구를 학습해야 할 때는 완성된 답변만 받는 방식에 한계가 있습니다.

예를 들어 Excel에서 새로운 함수를 배워야 한다고 가정해 보겠습니다.

다음과 같이 질문할 수 있습니다.

<pre><code>XLOOKUP 함수의 사용 방법을 알려 주세요.</code></pre>

ChatGPT는 함수의 구문과 예시를 바로 설명할 수 있습니다.

사용자는 제공된 수식을 복사해 현재 파일에 적용할 수 있습니다. 하지만 데이터 구조나 조건이 조금만 달라지면 다음과 같은 문제가 발생할 수 있습니다.

* 어떤 값을 찾는 것인지 설명하지 못합니다.
* 조회 범위를 왜 선택했는지 이해하지 못합니다.
* 수식의 각 인수가 어떤 역할을 하는지 모릅니다.
* 오류가 발생했을 때 원인을 찾지 못합니다.
* 다른 업무 상황에 함수를 적용하지 못합니다.
* 비슷한 문제가 생길 때마다 다시 질문합니다.

여기에서 중요한 점은 다음과 같습니다.

> 당장의 업무 속도가 빨라지는 것과 자신의 실력이 향상되는 것은 서로 다른 문제입니다.

정답을 복사하면 현재 업무는 빨리 끝낼 수 있습니다.

하지만 원리와 판단 과정을 이해하지 못하면 사용자의 문제 해결 능력은 충분히 향상되지 않습니다.

공부 모드는 정답을 얻는 속도보다 **정답에 도달하는 사고 과정** 을 학습하도록 돕는 기능입니다.

## 3. 공부 모드란 무엇인가

공부 모드는 사용자가 주제를 더 깊게 이해할 수 있도록 설계된 ChatGPT의 학습 경험입니다.

일반적인 답변처럼 최종 결과만 제공하는 대신 다음과 같은 방식으로 학습을 진행할 수 있습니다.

* 사용자가 이미 알고 있는 내용을 질문합니다.
* 현재 이해 수준을 확인합니다.
* 문제를 작은 단계로 나눕니다.
* 단계별로 힌트를 제공합니다.
* 사용자가 직접 답하게 합니다.
* 틀린 답에서도 올바른 사고 방향을 찾도록 돕습니다.
* 이해한 내용을 자신의 말로 설명하게 합니다.
* 퀴즈와 연습 문제로 이해도를 점검합니다.
* 다른 상황에 개념을 적용하게 합니다.

OpenAI는 공부 모드를 사용자가 목표와 수준에 맞춰 단계별로 학습하고, 질문과 피드백을 통해 이해를 심화할 수 있는 학습 경험으로 설명합니다. 공부 모드는 새로운 개념 학습, 연습 문제 풀이, 시험 준비, 수업 자료 복습, 퀴즈와 플래시카드 생성 등에 활용할 수 있습니다.

공부 모드의 목적은 답을 숨기거나 사용자를 시험하는 것이 아닙니다.

정답에 도달하기 위해 필요한 사고 과정을 사용자가 직접 경험하도록 돕는 것이 목적입니다.

## 4. 일반 채팅과 공부 모드의 차이

일반 채팅과 공부 모드는 서로 우열을 가리는 기능이 아닙니다.

사용 목적에 따라 선택해야 합니다.

| 구분     | 일반 채팅            | 공부 모드           |
| ------ | ---------------- | --------------- |
| 주요 목적  | 빠른 답변과 결과물 생성    | 개념과 사고 과정 학습    |
| 진행 방식  | 질문 후 완성된 답변 제공   | 질문, 힌트와 단계별 설명  |
| 사용자 역할 | 답변을 읽고 활용        | 직접 답하고 판단       |
| 설명 구조  | 한 번에 전체 설명 가능    | 작은 단위로 나누어 설명   |
| 이해도 확인 | 사용자가 별도로 요청      | 중간 질문과 퀴즈로 점검   |
| 주요 결과  | 문서, 수식, 요약, 아이디어 | 자기 설명, 문제 해결 능력 |
| 적합한 상황 | 즉시 처리할 업무        | 장기적으로 익혀야 할 지식  |
| 대표 예시  | 이메일 초안 작성        | Excel 함수의 원리 학습 |

다음과 같은 상황에서는 일반 채팅이 적합합니다.

* 지금 바로 사용할 문장이 필요합니다.
* 이미 원리를 알고 있으며 형식만 바꾸면 됩니다.
* 반복적인 문서 작업을 빠르게 처리해야 합니다.
* 학습보다 산출물 완성이 더 중요합니다.

다음과 같은 상황에서는 공부 모드가 적합합니다.

* 새로운 도구나 기능을 처음 배웁니다.
* 답은 알지만 원리를 이해하지 못합니다.
* 비슷한 문제를 스스로 해결하고 싶습니다.
* 자신의 판단 과정에서 부족한 부분을 확인하고 싶습니다.
* 사내 규정이나 업무 매뉴얼을 실제 상황에 적용하고 싶습니다.
* 시험, 자격증 또는 직무 교육을 준비합니다.

## 5. 공부 모드의 세 가지 핵심 원리

공부 모드는 크게 세 가지 원리로 이해할 수 있습니다.

### 5.1 질문으로 사고를 끌어내기

첫 번째 원리는 사용자가 직접 생각하도록 질문하는 것입니다.

이를 **소크라테스식 문답** 이라고 설명할 수 있습니다.

ChatGPT가 정답을 바로 말하는 대신 다음과 같이 질문합니다.

* 이 함수는 어떤 문제를 해결한다고 생각합니까?
* 두 표를 연결하려면 어떤 값이 공통으로 있어야 합니까?
* 이 결과가 잘못되었다면 어느 단계부터 확인해야 합니까?
* 현재 알고 있는 내용은 무엇입니까?
* 다음 단계에서 무엇을 시도해 보겠습니까?

이러한 질문은 사용자가 기존 지식과 새로운 개념을 연결하도록 돕습니다.

잘못된 답을 하더라도 즉시 실패로 처리하지 않습니다.

사용자가 맞게 이해한 부분과 부족한 부분을 구분한 뒤 다음 힌트를 제공할 수 있습니다.

### 5.2 작은 단위로 나누어 설명하기

두 번째 원리는 복잡한 내용을 작은 단계로 나누는 것입니다.

이를 **스캐폴딩(Scaffolding)** 이라고 합니다.

예를 들어 Power BI 데이터 모델을 처음 배우는 사람에게 관계, 기본 키, 외래 키, 필터 방향과 스타 스키마를 한 번에 설명하면 학습 부담이 커질 수 있습니다.

공부 모드는 다음 순서로 설명할 수 있습니다.

1. 표와 표를 연결하는 이유를 이해합니다.
2. 두 표에 공통으로 있는 열을 찾습니다.
3. 중복되지 않는 기준값의 역할을 이해합니다.
4. 관계 방향을 확인합니다.
5. 필터가 전달되는 방식을 이해합니다.
6. 팩트 테이블과 차원 테이블로 확장합니다.

이처럼 현재 이해할 수 있는 수준에서 시작해 점진적으로 난이도를 높입니다.

### 5.3 이해했는지 확인하기

세 번째 원리는 사용자가 실제로 이해했는지 확인하는 것입니다.

단순히 `이해하셨습니까?`라고 묻는 것은 충분하지 않습니다.

사용자는 자신이 이해했다고 생각하더라도 실제로 설명하거나 적용하지 못할 수 있습니다.

공부 모드에서는 다음과 같은 방법으로 이해도를 점검할 수 있습니다.

* 개념을 자신의 말로 설명합니다.
* 간단한 퀴즈에 답합니다.
* 틀린 예시를 찾습니다.
* 새로운 상황에 개념을 적용합니다.
* 두 개념의 차이를 비교합니다.
* 실제 업무 사례의 해결 방법을 제안합니다.
* 다른 사람에게 설명하듯 정리합니다.

정리하면 공부 모드는 다음 흐름으로 진행됩니다.

> 질문 → 단계별 설명 → 사용자 답변 → 피드백 → 이해 점검 → 업무 적용

## 6. 공부 모드에서 기대할 수 있는 학습 경험

공부 모드는 다음과 같은 학습 활동에 활용할 수 있습니다.

| 학습 목적     | 활용 방법                           |
| --------- | ------------------------------- |
| 새로운 개념 학습 | 기초 개념부터 단계별로 설명                 |
| 연습 문제 풀이  | 답 대신 힌트와 다음 단계 제공               |
| 시험 준비     | 예상 문제, 퀴즈와 오답 피드백               |
| 수업 자료 복습  | 노트, 슬라이드와 교재 내용을 점검             |
| 업무 도구 학습  | Excel, Power Query와 Power BI 실습 |
| 문서 학습     | 규정과 매뉴얼 기반 상황 판단 퀴즈             |
| 기획 역량 강화  | 가정과 논리를 질문으로 검토                 |
| 설명 능력 향상  | 자기 언어로 설명한 내용 피드백               |
| 면접 준비     | 질문과 답변을 반복하며 보완                 |
| 언어 학습     | 수준에 맞는 대화와 교정                   |

공부 모드는 단순히 학생이나 시험 준비자만을 위한 기능이 아닙니다.

기업 실무자도 새로운 기능과 제도, 도구, 산업 지식을 학습하는 데 사용할 수 있습니다.

## 7. 공부 모드의 이용 범위와 제한 사항

현재 공부 모드는 전 세계의 ChatGPT 플랜에서 웹, iOS와 Android를 통해 사용할 수 있으며, 사용자가 이용할 수 있는 모든 ChatGPT 모델과 함께 사용할 수 있습니다.

공부 모드는 일반 ChatGPT 대화와 임시 채팅에서 사용할 수 있습니다.

다만 현재 공식 안내에 따르면 GPT와 프로젝트 안의 대화에서는 공부 모드를 사용할 수 없습니다. 계정과 앱에 따라 퀴즈와 같은 별도의 학습 바로가기가 표시될 수 있지만, 별도 메뉴가 없더라도 공부 모드에서 퀴즈나 연습 문제를 요청할 수 있습니다.

공부 모드를 사용할 때는 다음 한계를 이해해야 합니다.

* 항상 질문만 제공하는 것은 아닙니다.
* 상황에 따라 정답을 직접 설명할 수 있습니다.
* 설명과 답변에 오류가 있을 수 있습니다.
* 파일이나 이미지의 내용을 잘못 인식할 수 있습니다.
* 사용량과 파일 업로드는 사용자의 플랜과 모델 제한을 따릅니다.
* 학교나 조직의 AI 활용 규칙을 대신하지 않습니다.
* 중요한 정책, 법률, 재무와 업무 수치는 별도로 검증해야 합니다.

공부 모드가 정답을 바로 제공한다면 다음처럼 요청합니다.

<pre><code>정답을 바로 말하지 말고,
제가 먼저 생각할 수 있도록 힌트부터 제공해 주세요.</code></pre>

## 8. 공부 모드 시작하기

공부 모드는 새로운 대화 또는 일반 ChatGPT 대화에서 시작할 수 있습니다.

### 8.1 웹에서 시작하기

현재 웹에서는 다음 방법을 사용할 수 있습니다.

1. 프롬프트 입력창에서 `@study`를 입력합니다.
2. 표시되는 추천 항목에서 `Study` 또는 `공부하기`를 선택합니다.

또는 다음 방법을 사용할 수 있습니다.

1. 입력창의 더하기 버튼을 선택합니다.
2. 검색창에 `study` 또는 `공부`를 입력합니다.
3. 공부 모드를 선택합니다.

로그인한 상태에서 공부 모드 전용 화면으로 직접 이동할 수도 있습니다. 메뉴 구성과 명칭은 업데이트나 언어 설정에 따라 달라질 수 있습니다.

### 8.2 모바일에서 시작하기

iOS에서는 첨부 버튼을 열어 공부 모드를 선택하거나 `@study`를 입력할 수 있습니다.

Android에서는 더하기 버튼의 도구 목록에서 공부 모드를 검색하거나 `@study`를 입력할 수 있습니다.

화면 구성은 앱 버전과 계정에 따라 달라질 수 있습니다.

### 8.3 공부 모드 종료하기

입력창에 선택된 공부 모드 표시를 제거하면 일반 대화 방식으로 돌아갈 수 있습니다.

웹 환경에서는 공부 모드 표시 옆에 커서를 두고 `Backspace` 또는 `Delete` 키를 눌러 제거할 수도 있습니다.

## 9. 좋은 학습은 학습 맥락 설정에서 시작합니다

공부 모드를 선택한 뒤 바로 질문을 입력할 수도 있습니다.

하지만 자신의 수준과 목적을 먼저 설명하면 더 적합한 학습 경험을 만들 수 있습니다.

다음 네 가지를 먼저 입력하는 것이 좋습니다.

| 항목       | 설명           | 작성 예시              |
| -------- | ------------ | ------------------ |
| 학습 주제    | 무엇을 배울 것인지   | Excel XLOOKUP 함수   |
| 현재 수준    | 이미 알고 있는 내용  | VLOOKUP은 가끔 사용     |
| 학습 목표    | 무엇을 이해하고 싶은지 | XLOOKUP의 원리와 사용 상황 |
| 업무 적용 상황 | 어디에 활용할 것인지  | 두 표에서 코드로 값 조회     |

기본 템플릿은 다음과 같습니다.

<pre><code>학습 주제:
[배우고 싶은 주제]

현재 수준:
[현재 알고 있는 내용과 어려운 부분]

학습 목표:
[학습을 마친 뒤 할 수 있어야 하는 일]

업무 적용 상황:
[이 개념을 사용할 실제 업무]</code></pre>

이 네 가지 정보는 ChatGPT가 다음 내용을 조정하는 데 도움이 됩니다.

* 설명의 난이도
* 사용하는 용어
* 예시의 종류
* 질문의 깊이
* 학습 속도
* 필요한 선수 지식
* 업무 적용 방식

학습 맥락이 없다면 설명이 지나치게 쉽거나 어려울 수 있습니다.

## 10. XLOOKUP 함수 학습 실습

이번 강의에서는 Excel의 XLOOKUP 함수를 공부 모드로 학습합니다.

공부 모드를 선택한 뒤 아래 프롬프트를 입력합니다.

<pre><code>학습 주제:
Excel XLOOKUP 함수

현재 수준:
VLOOKUP은 가끔 사용하지만
수식이 작동하는 원리는 정확히 모릅니다.

학습 목표:
XLOOKUP이 VLOOKUP보다 편리한 이유와
어떤 상황에서 사용하는지 이해하고 싶습니다.

업무 적용 상황:
두 개의 표에서 같은 상품코드를 기준으로
상품명이나 단가를 찾아 정리하는 업무입니다.</code></pre>

이 프롬프트를 입력하면 ChatGPT는 사용자의 목표를 다음과 같이 해석할 수 있습니다.

* 두 개의 표를 연결하는 상황을 이해합니다.
* VLOOKUP에 대한 기존 경험을 활용합니다.
* XLOOKUP과 VLOOKUP의 차이를 학습합니다.
* 상품코드를 기준으로 값을 찾는 실무 상황을 사용합니다.

중요한 점은 ChatGPT가 바로 XLOOKUP 수식만 제공하지 않을 수 있다는 것입니다.

먼저 다음과 같은 질문을 할 수 있습니다.

* 두 표에 공통으로 있는 값은 무엇입니까?
* 어떤 값을 기준으로 찾아야 합니까?
* 어느 표에서 어떤 값을 가져와야 합니까?
* VLOOKUP을 사용한다면 몇 번째 열을 지정해야 합니까?
* 찾는 열과 반환하는 열의 위치가 바뀌면 어떻게 됩니까?

사용자는 완벽한 답변을 작성할 필요가 없습니다.

현재 알고 있는 부분과 모르는 부분을 솔직하게 설명하면 됩니다.

<pre><code>두 표에 모두 있는 상품코드를 기준으로 찾아야 할 것 같습니다.

상품 마스터 표에서 상품명을 가져와야 합니다.

다만 XLOOKUP의 각 범위를 어떤 순서로 지정하는지는 잘 모르겠습니다.</code></pre>

이 답변을 바탕으로 ChatGPT는 사용자가 이해한 부분과 부족한 부분을 구분해 다음 단계로 진행할 수 있습니다.

## 11. 공부 모드의 대화 흐름 이해하기

XLOOKUP 실습은 다음과 같은 흐름으로 진행될 수 있습니다.

### 11.1 기존 지식 확인

사용자가 VLOOKUP을 가끔 사용한다고 입력했기 때문에 ChatGPT는 VLOOKUP에 대한 질문부터 시작할 수 있습니다.

<pre><code>VLOOKUP을 사용한다면
두 표에서 무엇을 기준으로 찾아야 하고,
어떤 값을 가져와야 할까요?</code></pre>

사용자는 다음과 같이 답할 수 있습니다.

<pre><code>상품코드를 기준으로 찾고,
상품 마스터의 상품명을 가져와야 합니다.</code></pre>

### 11.2 기존 방식의 한계 발견

다음 질문으로 VLOOKUP의 열 번호 문제를 확인할 수 있습니다.

<pre><code>상품코드가 첫 번째 열이고 상품명이 두 번째 열이라면
VLOOKUP의 열 번호에는 어떤 숫자를 입력해야 할까요?</code></pre>

사용자가 `2`라고 답하면 ChatGPT는 VLOOKUP이 반환 열의 위치를 숫자로 지정한다는 점을 설명할 수 있습니다.

### 11.3 새로운 개념 연결

그다음 XLOOKUP의 구조를 비교합니다.

VLOOKUP에서는 반환할 값의 위치를 열 번호로 지정합니다.

XLOOKUP에서는 다음 범위를 직접 지정합니다.

* 찾을 값
* 찾을 범위
* 반환할 범위

즉, `두 번째 열에서 가져오라`고 지정하는 대신 `상품명 열에서 가져오라`고 직접 지정할 수 있습니다.

### 11.4 이해가 어려운 부분 확인

ChatGPT는 다음과 같이 질문할 수 있습니다.

<pre><code>상품마스터[상품명]이라는 표현에서
어떤 부분이 가장 생소하거나 헷갈립니까?</code></pre>

사용자는 다음처럼 답할 수 있습니다.

<pre><code>대괄호 안에 열 이름을 쓰는 방식이 생소합니다.</code></pre>

ChatGPT는 이 답변에 맞춰 구조적 참조나 표 이름을 추가로 설명할 수 있습니다.

이 과정에서 ChatGPT는 사용자의 답변에 따라 학습 순서를 조정합니다.

정해진 설명을 일방적으로 제공하는 것이 아니라 사용자의 이해 상태에 따라 다음 질문과 설명이 달라지는 것입니다.

## 12. 정답 요청을 학습 요청으로 바꾸기

공부 모드를 사용하더라도 ChatGPT가 답을 바로 설명하는 경우가 있을 수 있습니다.

또한 공부 모드를 켜지 않은 일반 대화에서도 질문 중심의 학습 방식을 사용할 수 있습니다.

핵심은 **정답을 요청하는 것이 아니라 학습 과정을 요청하는 것** 입니다.

### 12.1 정답 요청형 프롬프트

<pre><code>Power Query 병합 방법을 알려 주세요.</code></pre>

이 요청은 빠른 기능 설명에는 적합합니다.

하지만 사용자가 병합의 원리와 조인 종류를 이해했는지 확인하기는 어렵습니다.

### 12.2 학습 요청형 프롬프트

<pre><code>Power Query 병합을 원리부터 배우고 싶습니다.

정답과 전체 절차를 바로 설명하지 말고,
제가 먼저 생각할 수 있도록 질문부터 해 주세요.

한 번에 하나의 질문만 해 주세요.

제가 답하면 맞게 이해한 부분과 부족한 부분을 구분한 뒤,
다음 힌트나 설명을 제공해 주세요.</code></pre>

### 12.3 공통 학습 제어 프롬프트

아래 프롬프트는 여러 학습 주제에 공통으로 사용할 수 있습니다.

<pre><code>정답을 바로 설명하지 말고,
제가 먼저 생각해 볼 수 있도록 질문부터 해 주세요.

한 번에 하나씩 진행해 주세요.

제가 답하면 그 답변을 기준으로
다음 힌트나 설명을 제공해 주세요.

제가 틀리더라도 바로 정답만 말하지 말고,
어느 부분까지 맞게 생각했는지 먼저 알려 주세요.</code></pre>

이 프롬프트는 일반 채팅에서도 사용할 수 있습니다.

다만 공부 모드는 질문, 단계별 설명, 이해 점검과 개인화를 학습 경험으로 제공한다는 점에서 차이가 있습니다.

## 13. 설명의 난이도와 속도 조정하기

공부 모드의 설명이 현재 수준과 맞지 않을 경우 추가 요청으로 조정할 수 있습니다.

### 13.1 설명이 너무 어려운 경우

<pre><code>설명이 현재 제 수준보다 어렵습니다.

전문 용어를 줄이고,
비전공자도 이해할 수 있는 비유와 작은 예시부터 설명해 주세요.</code></pre>

### 13.2 설명이 너무 쉬운 경우

<pre><code>기본 개념은 이해했습니다.

실무에서 발생할 수 있는 예외 상황과
자주 발생하는 오류를 포함해 난이도를 높여 주세요.</code></pre>

### 13.3 질문이 너무 많은 경우

<pre><code>질문이 너무 많아 학습 흐름을 따라가기 어렵습니다.

가장 중요한 질문 하나만 제시하고,
제가 답한 뒤 다음 단계로 이동해 주세요.</code></pre>

### 13.4 학습이 길어지는 경우

<pre><code>지금까지 배운 내용을 5줄로 정리해 주세요.

제가 이해한 내용과 아직 부족한 내용을 구분하고,
다음에 학습할 단계 하나만 제안해 주세요.</code></pre>

### 13.5 힌트를 더 받고 싶은 경우

<pre><code>아직 정답은 말하지 말고,
다음에 무엇을 확인해야 하는지 힌트 하나만 주세요.</code></pre>

## 14. 자신의 언어로 다시 설명하기

공부 모드에서 가장 중요한 단계 중 하나는 배운 내용을 자신의 언어로 다시 설명하는 것입니다.

설명을 읽고 이해했다고 느끼는 것과 직접 설명할 수 있는 것은 다릅니다.

자신의 언어로 설명하면 다음 내용을 확인할 수 있습니다.

* 핵심 개념을 이해했는가?
* 용어만 외운 것은 아닌가?
* 원인과 결과를 구분할 수 있는가?
* 예외 상황을 알고 있는가?
* 다른 업무 상황에도 적용할 수 있는가?
* 빠진 개념이 무엇인가?

다음 프롬프트를 사용합니다.

<pre><code>이제 제가 이해한 내용을 제 말로 설명해 보겠습니다.

제가 설명한 내용을 다음 세 가지로 구분해 피드백해 주세요.

1. 맞게 이해한 부분
2. 빠졌거나 부족한 부분
3. 더 정확히 표현해야 할 부분

바로 완성된 정답으로 바꾸지 말고,
제가 다시 수정할 수 있도록 질문과 힌트도 제공해 주세요.</code></pre>

프롬프트 뒤에 자신의 설명을 붙입니다.

<pre><code>XLOOKUP 함수는 한 범위에서 값을 찾고
다른 범위에서 대응하는 값을 가져오는 함수입니다.

VLOOKUP처럼 반환 열의 위치를 숫자로 지정하지 않고,
찾을 범위와 가져올 범위를 직접 지정합니다.

그래서 찾을 열과 반환할 열의 위치가 바뀌어도
수식을 이해하고 수정하기가 더 쉽습니다.</code></pre>

ChatGPT는 다음과 같이 구분해 피드백할 수 있습니다.

* 맞게 이해한 개념
* 표현이 모호한 부분
* 빠진 인수와 옵션
* VLOOKUP과의 차이
* 추가로 주의해야 할 오류
* 실제 업무 적용 예시

이 과정을 **메타인지** 관점에서 이해할 수도 있습니다.

사용자가 자신이 무엇을 알고 있고 무엇을 모르는지 확인하는 과정입니다.

학습이 끝났다고 생각되면 퀴즈나 적용 문제로 이해도를 확인합니다.

제 설명을 검토하고
초보자가 이해하기 어려운 부분을 질문해 주세요.</code></pre>

## 16. 업로드 파일과 이미지를 활용한 학습

공부 모드는 사용자가 업로드한 파일과 이미지를 학습 자료로 활용할 수 있습니다.

PDF, 이미지, 슬라이드, 수업 자료와 업무 문서를 업로드한 뒤 특정 페이지나 항목을 기준으로 학습을 요청할 수 있습니다. 파일 내용이 제대로 인식되지 않는 경우 더 선명한 파일을 업로드하거나, 중요한 내용을 직접 붙여넣거나, 학습할 페이지와 질문을 구체적으로 지정하는 것이 좋습니다.

### 16.1 사내 규정 학습

다음과 같이 요청할 수 있습니다.

<pre><code>첨부한 사내 정보보안 규정을
공부 모드 방식으로 학습하고 싶습니다.

목표는 규정을 단순히 요약하거나 암기하는 것이 아니라,
실제 업무 상황에서 올바른 행동을 판단하는 것입니다.

## 17. 메모리를 활용한 개인화

메모리 기능이 활성화되어 있으면 ChatGPT는 저장된 메모리와 과거 대화를 활용해 공부 모드를 개인화할 수 있습니다.

예를 들어 다음 정보를 반영할 수 있습니다.

* 사용자의 학습 목표
* 익숙한 도구와 업무
* 선호하는 설명 방식
* 이전에 학습한 주제
* 자주 어려워하는 부분
* 원하는 난이도

메모리를 사용하지 않아도 공부 모드는 정상적으로 작동합니다. 다만 메모리가 활성화되어 있으면 이전 학습 맥락을 바탕으로 더 개인화된 설명을 제공할 수 있습니다.

메모리를 활용하고 싶지 않다면 설정의 개인 맞춤 설정에서 메모리를 끌 수 있습니다.

중요한 개인정보나 민감한 업무 정보를 학습 편의를 위해 메모리에 저장하도록 요청해서는 안 됩니다.

## 24. 핵심 정리

* 공부 모드는 답을 더 빨리 받는 기능이 아니라 정답에 도달하는 사고 과정을 학습하는 기능입니다.
* 일반 채팅은 빠른 결과물 작성에 적합하고, 공부 모드는 장기적인 지식과 판단력 향상에 적합합니다.
* 공부 모드는 질문, 단계별 설명과 이해 점검을 통해 능동적인 학습을 유도합니다.
* 소크라테스식 문답은 정답보다 질문으로 사용자의 생각을 끌어냅니다.
* 스캐폴딩은 복잡한 개념을 작은 단계로 나누어 설명합니다.
* 이해 점검에서는 퀴즈, 자기 설명과 업무 적용 문제를 사용할 수 있습니다.
* 공부 모드를 시작할 때는 학습 주제, 현재 수준, 학습 목표와 업무 적용 상황을 입력하는 것이 좋습니다.
* 정답 요청을 학습 요청으로 바꾸면 일반 채팅에서도 질문 중심 학습을 진행할 수 있습니다.
* ChatGPT가 답을 바로 제공하면 힌트와 질문부터 달라고 다시 요청할 수 있습니다.
* 설명이 너무 쉽거나 어려우면 현재 수준과 원하는 난이도를 구체적으로 알려야 합니다.
* 배운 내용을 자신의 언어로 설명해야 이해가 부족한 부분을 발견할 수 있습니다.
* 파일과 이미지를 활용해 강의자료, 매뉴얼과 사내 규정을 학습할 수 있습니다.
* 메모리가 활성화되어 있으면 과거 학습 목표와 선호 방식을 반영할 수 있습니다.
* Excel, Power Query, Power BI와 기획 논리 학습 등 다양한 실무 분야에 적용할 수 있습니다.
* 공부 모드도 오류가 발생할 수 있으므로 중요한 정보는 공식 자료와 대조해야 합니다.
* 공부 모드를 잘 활용하는 사람은 AI에게 일을 대신 시키는 데서 끝나지 않고, AI를 통해 자신의 판단력과 문제 해결 능력을 강화합니다.
