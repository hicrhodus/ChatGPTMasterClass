# 04-4. 음성 입력

## 1. 이 강의에서 배울 내용

이번 강의에서는 ChatGPT 의 음성 입력 기능을 사용해 긴 업무 프롬프트를 빠르게 작성하고, 전송 전에 검수하는 방법을 학습합니다.

이 강의를 마치면 다음을 할 수 있습니다.

* 음성 입력과 음성 모드의 차이를 구분할 수 있습니다.
* 긴 업무 프롬프트를 음성으로 빠르게 작성할 수 있습니다.
* 전송 전에 숫자, 조건, 출력 형식, 민감 정보를 검수할 수 있습니다.
* 전사 정확도를 높이는 말하기 방법을 이해할 수 있습니다.
* 업무에서 음성 입력을 사용할 때 필요한 보안 기준을 점검할 수 있습니다.

## 2. 언제 쓰는 기능인가

ChatGPT 를 쓰다 보면 머릿속에는 할 말이 많은데, 막상 입력창 앞에서는 짧게 쓰게 되는 경우가 많습니다. 특히 모바일에서 사용할 때 이런 일이 자주 생깁니다.

프로젝트 배경, 조건, 예외 상황, 원하는 문체, 출력 형식까지 모두 적으려고 하면 시간이 오래 걸립니다. 쓰는 도중 중요한 조건을 빠뜨리기도 합니다. 그러면 결과물도 그만큼 두루뭉술해집니다.

음성 입력은 이 문제를 줄여주는 입력 방식입니다. 말로 설명하면 긴 프롬프트도 빠르게 만들 수 있고, 전송하기 전에 눈으로 확인하고 고칠 수도 있습니다.

핵심은 단순히 “빠르게 입력한다” 가 아닙니다. 음성 입력의 진짜 목적은 **머릿속 업무 맥락을 손실 없이 ChatGPT 입력창으로 옮기는 것** 입니다.

## 3. 먼저 알아둘 개념

### 3.1 음성 입력이란 무엇인가

음성 입력은 사용자의 말을 텍스트로 변환해 ChatGPT 입력창에 넣어 주는 기능입니다.

중요한 점은 전송 전에 텍스트를 확인하고 수정할 수 있다는 것입니다. 이 때문에 음성 입력은 숫자, 금액, 일정, 조건처럼 정확성이 중요한 업무 프롬프트에 적합합니다.

예를 들어 다음과 같은 상황에서 유용합니다.

* 프로젝트 기획 배경을 길게 설명해야 할 때
* 보고서 조건을 여러 개 전달해야 할 때
* 코드 오류 상황을 한국어로 길게 설명한 뒤 실제 코드는 붙여넣어야 할 때
* 회의 전 생각나는 내용을 빠르게 프롬프트 초안으로 만들고 싶을 때
* 모바일에서 긴 프롬프트를 작성해야 할 때

### 3.2 음성 입력, 음성 모드, 레코드 모드의 차이

ChatGPT 에는 말과 관련된 기능이 여러 가지 있습니다. 이름이 비슷해서 헷갈릴 수 있지만, 목적은 다릅니다.

| 구분      | 음성 입력                      | 음성 모드                   | 레코드 모드            |
| ------- | -------------------------- | ----------------------- | ----------------- |
| 목적      | 말을 텍스트 프롬프트로 변환            | AI 와 실시간 음성 대화          | 장시간 회의·대화 기록 및 요약 |
| 전송 전 수정 | 가능                         | 어려움                     | 녹음 후 요약 중심        |
| 적합한 업무  | 긴 프롬프트 작성, 조건 전달, 문서 초안 요청 | 아이디어 대화, 영어 회화, 빠른 질의응답 | 회의록, 인터뷰, 장시간 기록  |
| 통제력     | 높음                         | 낮음~중간                   | 중간                |
| 핵심 장점   | 전송 전 검수 가능                 | 즉각적인 대화                 | 긴 오디오 처리          |

실무 프롬프트 작성에는 음성 모드보다 음성 입력이 더 적합합니다. 이유는 간단합니다. 음성 입력은 말한 내용이 입력창에 텍스트로 들어오기 때문에, 전송 전에 숫자와 조건을 확인할 수 있습니다.

> **핵심 기준**
> 정교한 업무 요청은 음성 입력, 실시간 대화는 음성 모드, 긴 회의 기록은 레코드 모드가 적합합니다.

## 4. 따라 해보기

이번 실습의 목표는 마이크 아이콘을 눌러보는 것이 아닙니다. 목표는 **업무 맥락이 담긴 긴 프롬프트를 음성으로 만들고, 전송 전에 검수까지 해보는 것** 입니다.

상황은 이렇게 잡겠습니다. 다음 주 임원 보고에 쓸 월간 영업 실적 요약을 만든다고 가정합니다. 매출은 전월 대비 늘었지만, 신규 고객은 목표보다 부족한 상황입니다.

### 4.1 음성 입력 시작하기

ChatGPT 입력창에서 마이크 아이콘을 선택합니다.

이때 단어를 하나씩 끊어 말하지 말고, 문장 단위로 자연스럽게 말합니다. 음성 인식은 단어만 듣는 것이 아니라 앞뒤 문맥을 함께 고려해 문장을 전사하기 때문입니다.

### 4.2 업무 상황을 문장으로 말하기

아래 상황을 머릿속에 두고 말합니다.

* 나는 영업기획팀 담당자입니다.
* 다음 주 임원 보고에 사용할 월간 영업 실적 요약이 필요합니다.
* 이번 달 매출은 전월 대비 8% 증가했습니다.
* 신규 고객 수는 목표 대비 15% 부족했습니다.
* 보고서에는 매출 요약, 목표 대비 차이, 주요 원인, 다음 달 대응 계획이 필요합니다.
* 출력 형식은 임원 보고용 1페이지 요약문입니다.

## 5. 실습 프롬프트

아래 프롬프트를 음성으로 말해 보세요.

```text
나는 영업기획팀 담당자입니다. 다음 주 임원 보고에서 쓸 월간 영업 실적 요약 자료를 만들려고 합니다. 이번 달 매출은 전월 대비 8% 증가했지만, 신규 고객 수는 목표 대비 15% 부족했습니다. 보고서에는 매출 요약, 목표 대비 차이, 주요 원인, 다음 달 대응 계획을 포함해 주세요. 출력 형식은 임원 보고용 1페이지 요약문으로 작성해 주세요.
```

입력이 끝나면 확인 버튼을 누릅니다. 그러면 방금 말한 내용이 ChatGPT 입력창에 텍스트로 들어옵니다.

### 5.1 전송 전 검수하기

이제 바로 전송하지 않습니다. 전송 전에 눈으로 확인하는 단계가 음성 입력의 핵심입니다.

다음 항목을 확인합니다.

* `8%` 가 정확히 입력되었는가?
* `15%` 가 정확히 입력되었는가?
* “임원 보고용 1페이지 요약문” 이라는 출력 형식이 들어갔는가?
* 매출 요약, 목표 대비 차이, 주요 원인, 다음 달 대응 계획이 빠지지 않았는가?
* 실제 회사명, 고객명, 개인명 같은 민감 정보가 들어가지 않았는가?

틀린 부분이 있다면 그 부분만 키보드로 고칩니다. 음성 입력은 최종본이 아니라 **빠른 초안** 입니다. 말로 초안을 만들고, 눈으로 보정한 뒤 전송하는 흐름으로 사용해야 합니다.

## 6. 결과 확인하기

검수가 끝난 프롬프트를 전송한 뒤에는 결과물을 확인합니다.

좋은 결과물은 다음 조건을 충족합니다.

* 요청한 형식대로 임원 보고용 1페이지 요약문이 작성되었다.
* 매출 증가와 신규 고객 부족이 모두 반영되었다.
* 목표 대비 차이와 주요 원인이 구분되어 있다.
* 다음 달 대응 계획이 포함되어 있다.
* 답변이 너무 일반적이면 추가 질문으로 보완할 수 있다.

결과가 기대보다 약하다면 대부분은 ChatGPT 의 성능 문제가 아니라 입력한 맥락이 부족한 경우입니다. 이때는 다시 음성 입력으로 조건을 추가해 보완하면 됩니다.

## 7. 더 깊이 이해하기

### 7.1 음성 입력은 “속도” 보다 “맥락” 이 중요합니다

음성 입력을 빠른 타이핑 대체 기능으로만 보면 활용도가 제한됩니다. 실무에서 더 중요한 가치는 **맥락 손실을 줄이는 것** 입니다.

실무자는 대개 머릿속에 많은 정보를 가지고 있습니다. 하지만 키보드로 입력하는 과정에서 다음 문제가 생깁니다.

1. 타이핑 속도가 생각 속도를 따라가지 못한다.
2. 긴 배경 설명을 쓰다가 핵심 조건을 빠뜨린다.
3. 귀찮아서 프롬프트를 짧게 쓴다.
4. 짧게 쓴 만큼 낮은 품질의 답변을 받는다.
5. 결국 ChatGPT 가 별로라고 느낀다.

음성 입력은 이 흐름을 바꿔 줍니다. 말하듯이 업무 배경, 조건, 제약사항, 원하는 결과물 형식을 한 번에 전달할 수 있기 때문입니다.

### 7.2 문장으로 말해야 전사가 더 정확합니다

음성 인식은 단어를 하나씩 받아쓰는 방식이 아닙니다. 앞뒤 문맥을 보고 문장을 만듭니다.

좋지 않은 방식:

```text
신규... 프로젝트... 예산... 1억... 기간... 3개월...
```

좋은 방식:

```text
신규 프로젝트 기획안을 작성하려고 합니다. 전체 예산은 1억 원이고, 실행 기간은 3개월입니다. 단, 외부 협력사는 2곳 이하로 제한하고, 기존 시스템과 연동 가능해야 합니다.
```

단어를 뚝뚝 끊어 말하면 문맥 정보가 사라져 전사 품질이 떨어질 수 있습니다. 반대로 완전한 문장으로 말하면 시스템이 앞뒤 맥락을 활용하기 쉬워집니다.

### 7.3 길게 한 덩어리로 들어왔을 때

말한 내용이 한 덩어리 문단으로 들어왔다면, 전송 전에 문단을 나누세요.

예를 들어 다음 기준으로 정리할 수 있습니다.

```text
역할:
상황:
목적:
조건:
출력 형식:
```

필요하면 `Shift + Enter` 로 줄을 나누고, 조건을 목록 형태로 정리합니다. 조건이 목록으로 정리되면 빠진 항목을 찾기도 쉽고, ChatGPT 도 요구사항을 더 정확하게 반영합니다.

## 8. 전문 용어가 자주 틀릴 때

음성 입력은 전문 용어, 영문 약어, 코드 관련 용어를 한국어 음차로 전사할 수 있습니다.

예를 들어 다음과 같은 문제가 생길 수 있습니다.

| 말한 표현  | 잘못 전사될 수 있는 표현 | 원하는 표현      |
| ------ | -------------- | ----------- |
| 파워 쿼리  | 파워쿼리           | Power Query |
| 파워 피벗  | 파워피벗           | Power Pivot |
| 디액스    | 디엑스            | DAX         |
| 파이테스트  | 파이 테스트         | pytest      |
| 리액트 훅스 | 리액트 훅스         | React hooks |

이런 표현이 반복적으로 틀린다면 맞춤 설정에 교정 규칙을 넣어 둘 수 있습니다.

예시:

```text
내가 음성 입력을 사용해 질문할 때, 일부 기술 용어가 한국어 음차로 전사될 수 있습니다.

답변을 생성하기 전에 문맥을 고려하여 아래 표현을 자동으로 교정해 주세요.

- "파워 쿼리" 또는 "파워쿼리"는 "Power Query" 로 해석해 주세요.
- "파워 피벗" 또는 "파워피벗"은 "Power Pivot" 으로 해석해 주세요.
- "디액스" 또는 "디엑스"는 "DAX" 로 해석해 주세요.
- "파이테스트" 또는 "파이 테스트"는 "pytest" 로 해석해 주세요.
- "리액트 훅스"는 "React hooks" 로 해석해 주세요.

단, 실제 사용자가 한국어 표현을 의도한 경우에는 문맥을 우선해 주세요.
```

## 9. 실무에서 사용할 때의 주의사항

음성 입력은 편리하지만, 업무에서 사용할 때는 보안 기준이 중요합니다. 말로 설명하다 보면 실제 회사명, 고객명, 사람 이름, 내부 시스템명을 무심코 말하기 쉽습니다.

음성 입력을 켜기 전에 익명화 기준을 먼저 정하세요.

| 민감 정보   | 안전한 표현   |
| ------- | -------- |
| 실제 회사명  | A사       |
| 실제 고객명  | 주요 고객사   |
| 실제 부서명  | B부서      |
| 개인 이름   | 담당자 C    |
| 내부 시스템명 | 내부 시스템   |
| 정확한 예산  | 수억 원 규모  |
| 계약 조건   | 주요 계약 조건 |

전송 전에는 다음 항목을 다시 확인합니다.

* 실제 회사명이나 고객명이 들어가지 않았는가?
* 개인 이름이 그대로 들어가지 않았는가?
* 내부 시스템명이나 프로젝트명이 노출되지 않았는가?
* 정확한 예산, 계약 조건, 민감한 일정이 들어가지 않았는가?
* 회사의 AI 사용 정책에 어긋나는 정보가 포함되지 않았는가?

음성 입력이 편하다고 해서, 넣으면 안 되는 정보까지 편하게 들어가면 안 됩니다.

## 10. 자주 발생하는 문제

| 문제 상황              | 원인                              | 해결 방법                                        |
| ------------------ | ------------------------------- | -------------------------------------------- |
| 음성이 엉뚱한 외국어로 변환됨   | 자동 언어 감지가 주변 소음이나 짧은 발화를 잘못 해석함 | 첫 문장을 완전한 한국어 문장으로 말하고, 주 사용 언어를 한국어로 고정합니다. |
| 전사 결과에 오타가 많음      | 마이크가 멀거나 주변 소음이 큼               | 헤드셋 마이크를 사용하고 조용한 환경에서 다시 시도합니다.             |
| 숫자나 금액이 틀리게 전사됨    | 발화가 빠르거나 단위가 불명확함               | 숫자는 천천히 말하고, 전송 전 반드시 눈으로 확인합니다.             |
| 전문 용어가 한글 음차로 전사됨  | 발음 기반으로 일반 단어처럼 인식됨             | 자주 쓰는 용어는 맞춤 설정에 교정 규칙으로 등록합니다.              |
| 말한 내용이 너무 장황하게 들어감 | 말한 내용을 그대로 입력했지만 구조화하지 않음       | 전송 전 문단을 나누고 조건을 목록화합니다.                     |
| 회사 기밀이 그대로 입력됨     | 말로 설명하면서 실제 명칭을 무심코 말함          | 음성 입력 전 익명화 기준을 정하고, 전송 전 텍스트를 검수합니다.        |

## 11. 업무에 적용하기

음성 입력은 다음 업무에 바로 적용할 수 있습니다.

### 11.1 보고서 초안 작성

```text
나는 [부서/역할]입니다.

[보고서 목적]을 위해 보고서 초안을 작성하려고 합니다.

현재 상황은 [업무 배경]입니다.

반드시 포함할 내용은 다음과 같습니다.

첫째, [핵심 내용 1].
둘째, [핵심 내용 2].
셋째, [핵심 내용 3].
넷째, [주의할 조건].
다섯째, [원하는 결론 방향].

출력은 [보고서 형식]으로 작성해 주세요.
```

### 11.2 회의 후 정리

```text
나는 회의 진행자입니다.

방금 진행한 회의 내용을 기억나는 대로 말하겠습니다.

이 내용을 바탕으로 회의록 초안을 작성해 주세요.

구성은 다음과 같이 정리해 주세요.

- 회의 목적
- 주요 논의 내용
- 결정 사항
- 미결 이슈
- 담당자별 액션 아이템
- 후속 확인 일정

단, 실제 개인명은 담당자 A, 담당자 B 로 익명화해 주세요.
```

### 11.3 데이터 분석 요구사항 정리

```text
나는 데이터 분석 담당자입니다.

현업 부서에서 요청한 대시보드 요구사항을 정리하려고 합니다.

요청 배경은 [업무 배경]입니다.

분석 대상 데이터는 [데이터 종류]이고, 주요 분석 축은 [기간, 부서, 상품, 고객 등]입니다.

보고 싶은 지표는 다음과 같습니다.

첫째, [지표 1].
둘째, [지표 2].
셋째, [지표 3].

이 내용을 바탕으로 Power BI 대시보드 요구사항 정의서를 작성해 주세요.

출력 형식은 분석 목적, 사용자, 주요 지표, 필터 조건, 화면 구성, 데이터 준비사항으로 나누어 주세요.
```

## 12. 핵심 정리

* 음성 입력은 말한 내용을 텍스트 프롬프트로 바꾸는 기능입니다.
* 음성 입력은 전송 전에 검수할 수 있기 때문에 숫자, 조건, 보안 표현이 중요한 업무 프롬프트에 적합합니다.
* 핵심 절차는 **말로 초안, 눈으로 보정** 입니다.
* 전사 정확도를 높이려면 단어를 끊어 말하지 말고 완전한 문장으로 말해야 합니다.
* 전문 용어가 자주 틀리면 맞춤 설정에 교정 규칙을 넣을 수 있습니다.
* 업무에서는 음성 입력 전 익명화 기준을 정하고, 전송 전 민감 정보를 반드시 확인해야 합니다.
* 음성 입력의 목적은 빠른 입력이 아니라, 더 풍부한 맥락을 제공해 더 나은 업무 산출물을 만드는 것입니다.
