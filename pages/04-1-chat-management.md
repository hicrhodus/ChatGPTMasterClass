# 04-1. 대화 관리: 사이드바를 업무 도구함으로 만들기

## 1. 이 강의에서 배울 내용

이번 강의에서는 ChatGPT 의 사이드바에 쌓이는 대화를 체계적으로 관리하는 방법을 다룹니다.

ChatGPT 를 업무에 자주 사용하다 보면 대화가 빠르게 늘어납니다. 처음에는 몇 개 없어서 괜찮지만, 시간이 지나면 필요한 대화를 다시 찾는 데 시간이 걸리기 시작합니다.

예를 들어 이런 상황이 생깁니다.

* 어제 작성한 보고서 프롬프트가 어디 있었는지 찾기 어렵다.
* 지난주에 번역했던 이메일을 다시 찾고 싶다.
* 자주 쓰는 회의록 프롬프트를 매번 새로 작성한다.
* 일회성 질문과 중요한 업무 대화가 한곳에 섞인다.

이 강의를 통해 다음 내용을 익힐 수 있습니다.

* ChatGPT 대화를 업무 자산처럼 관리해야 하는 이유를 이해할 수 있습니다.
* 대화 이름을 업무 목적에 맞게 바꿀 수 있습니다.
* 아카이브에 보관과 삭제의 차이를 구분할 수 있습니다.
* 삭제와 데이터 제어 설정의 차이를 이해할 수 있습니다.
* 반복 업무에 사용하는 대화를 고정해 개인 프롬프트 라이브러리처럼 운영할 수 있습니다.
* ChatGPT 사이드바를 개인 업무 도구함으로 정리할 수 있습니다.

## 2. 왜 대화 관리가 필요한가

ChatGPT 를 처음 사용할 때는 입력창에 질문하고 답변을 받는 것만으로 충분합니다.

하지만 시간이 지나면 상황이 달라집니다.

업무 보고서, 이메일 초안, 회의록 정리, 번역, 아이디어 회의, 데이터 분석 질문, 강의 기획 대화가 계속 쌓입니다. 그러면 사이드바는 단순한 대화 기록 목록이 아니라, 내가 만들어 온 업무 산출물과 프롬프트가 모여 있는 공간이 됩니다.

문제는 이 공간을 관리하지 않으면 금방 찾기 어려운 상태가 된다는 것입니다.

이것은 ChatGPT 의 문제가 아닙니다.

핵심은 **사이드바를 어떻게 관리하느냐** 입니다.

ChatGPT 를 업무 도구로 사용하려면 좋은 질문을 하는 것만으로는 부족합니다. 좋은 대화를 다시 찾고, 좋은 프롬프트를 재사용하고, 필요 없는 대화는 정리할 수 있어야 합니다.

이번 강의에서는 네 가지 기능을 중심으로 살펴보겠습니다.

| 기능       | 핵심 목적                      |
| -------- | -------------------------- |
| 이름 바꾸기   | 나중에 찾기 쉽게 제목 정리            |
| 아카이브에 보관 | 지금은 숨기되 나중에 다시 볼 수 있게 정리   |
| 삭제       | 다시 볼 필요가 없거나 리스크가 있는 대화 제거 |
| 채팅 고정    | 반복 업무 프롬프트를 빠르게 불러오기       |

이 네 가지 기능을 구분해서 사용하면 ChatGPT 사이드바를 단순한 기록 목록이 아니라 **개인 업무 도구함** 으로 운영할 수 있습니다.

## 3. 이름 바꾸기: 대화를 나중에 찾을 수 있게 만들기

### 3.1 자동 제목의 문제

ChatGPT 는 대화가 시작되면 첫 질문이나 대화 내용을 기준으로 제목을 자동 생성합니다.

처음에는 이 기능이 편리해 보입니다. 사용자가 직접 제목을 만들지 않아도 대화가 목록에 저장되기 때문입니다.

하지만 업무에 계속 사용하다 보면 자동 제목의 한계가 드러납니다.

예를 들어 사이드바에 이런 제목이 생길 수 있습니다.

* 보고서 작성
* 이메일 수정
* 번역 도움
* 아이디어 정리
* 회의록 요약
* 신규 직원 환영 행사
* 배송 지연 안내

문제는 제목만 보고는 어떤 대화인지 정확히 알기 어렵다는 점입니다.

`보고서 작성` 이라는 제목이 있다고 해도, 어떤 보고서인지 알 수 없습니다. 월간 실적 보고서인지, 고객사 제안서인지, 임원 보고용 요약인지 알 수 없습니다.

`이메일 수정` 도 마찬가지입니다. 고객 사과 메일인지, 내부 공지 메일인지, 영문 비즈니스 메일인지 제목만으로 구분하기 어렵습니다.

결국 나중에 필요한 대화를 찾으려면 하나씩 클릭해서 내용을 확인해야 합니다.

이렇게 되면 ChatGPT 에서 시간을 절약하려고 만든 대화를 다시 찾느라 시간을 쓰게 됩니다.

### 3.2 이름 바꾸기의 역할

이름 바꾸기는 대화 제목을 사용자가 직접 수정하는 기능입니다.

중요한 점은 이름을 바꾼다고 해서 ChatGPT 의 답변 품질이 바뀌지는 않는다는 것입니다.

이름 바꾸기는 답변 품질을 높이는 기능이 아니라 **대화 관리 기능** 입니다.

하지만 업무 활용에서는 매우 중요합니다.

대화 제목이 잘 정리되어 있으면 다음 효과가 있습니다.

* 필요한 대화를 빠르게 찾을 수 있습니다.
* 비슷한 업무 대화를 구분할 수 있습니다.
* 자주 쓰는 프롬프트를 재사용하기 쉬워집니다.
* 대화 목록이 업무별로 정리됩니다.
* 사이드바가 개인 업무 도구함처럼 작동합니다.

### 3.3 좋은 제목의 기준

대화 제목은 그때그때 감으로 붙이기보다 하나의 규칙을 정해 두는 것이 좋습니다.

추천하는 형식은 다음과 같습니다.

<pre><code>[업무영역] 구체적 목적 + 산출물</code></pre>

이 형식은 세 부분으로 구성됩니다.

| 구성 요소  | 의미                | 예시                           |
| ------ | ----------------- | ---------------------------- |
| 업무영역   | 어떤 업무 범주인지 표시     | 보고서, 이메일, 회의, 강의, 리서치        |
| 구체적 목적 | 무엇을 하기 위한 대화인지 표시 | 월간 실적 요약, 납기 지연 사과, 액션아이템 정리 |
| 산출물    | 결과물의 형태 표시        | 초안, 템플릿, 체크리스트, 요약표          |

예를 들어 다음처럼 제목을 바꿀 수 있습니다.

<pre><code>[보고서] 2026년 6월 월간 실적 요약 초안
[이메일] 해외 고객 납기 지연 사과문 템플릿
[회의] 주간 팀 회의 액션아이템 정리
[강의] ChatGPT 입문 교육 커리큘럼 초안
[리서치] AI 업무 활용 사례 조사 요약</code></pre>

이렇게 제목을 정리하면 나중에 사이드바에서 제목만 보고도 어떤 대화인지 바로 파악할 수 있습니다.

### 3.4 이름 바꾸기 실습

이제 직접 이름을 바꿔 보겠습니다.

1. 사이드바에서 이름을 바꿀 대화를 선택합니다.
2. 대화 제목 옆의 더보기 메뉴를 클릭합니다.
3. `이름 바꾸기` 를 선택합니다.
4. 제목 입력창에 새 제목을 입력합니다.
5. Enter 를 눌러 저장합니다.

예를 들어 자동 제목이 `배송 지연 및 사과 안내` 라면 아래처럼 바꿀 수 있습니다.

<pre><code>[이메일] 고객 배송 지연 사과 메일 템플릿</code></pre>

자동 제목이 `신규 직원 환영 행사` 라면 아래처럼 바꿀 수 있습니다.

<pre><code>[인사] 신규 직원 환영 행사 기획안 초안</code></pre>

핵심은 제목을 예쁘게 쓰는 것이 아닙니다.

나중에 다시 찾을 수 있게 쓰는 것입니다.

## 4. 아카이브에 보관과 삭제의 차이

### 4.1 보관과 삭제는 다릅니다

많은 사용자가 `아카이브에 보관` 과 `삭제` 를 혼동합니다.

두 기능은 모두 사이드바에서 대화가 사라진다는 점에서는 비슷해 보입니다.

하지만 의미는 완전히 다릅니다.

> 아카이브에 보관은 숨기는 것이고, 삭제는 지우는 것입니다.

이 차이를 정확히 이해해야 합니다.

| 구분     | 아카이브에 보관                     | 삭제                       |
| ------ | ---------------------------- | ------------------------ |
| 의미     | 사이드바에서 숨김                    | 대화를 제거                   |
| 복구 가능성 | 다시 꺼낼 수 있음                   | 복구할 수 없음                 |
| 성격     | 비파괴 정리                       | 파괴적 정리                   |
| 사용 상황  | 지금은 자주 안 쓰지만 나중에 참고할 수 있는 대화 | 다시 볼 필요가 없거나 리스크가 있는 대화  |
| 예시     | 완료된 프로젝트, 리서치 결과, 참고 템플릿     | 일회성 질문, 실수로 민감정보를 입력한 대화 |

정리하면, 보관은 정리이고 삭제는 제거입니다.

### 4.2 아카이브에 보관이 적합한 경우

아카이브에 보관은 지금 당장 사이드바에 보일 필요는 없지만, 나중에 다시 볼 가능성이 있는 대화에 사용합니다.

예를 들어 다음과 같은 대화입니다.

* 완료된 프로젝트 대화
* 과거 리서치 결과 대화
* 참고 가치가 있는 템플릿 대화
* 당장은 끝났지만 나중에 재사용할 수 있는 프롬프트
* 특정 고객사나 과제 관련 기록

이런 대화는 사이드바에 계속 남겨두면 목록을 복잡하게 만듭니다. 하지만 삭제하기에는 아깝습니다.

이럴 때 아카이브에 보관을 사용합니다.

### 4.3 아카이브에 보관 실습

대화를 아카이브에 보관하는 절차는 다음과 같습니다.

1. 사이드바에서 보관할 대화를 찾습니다.
2. 대화 제목 옆의 더보기 메뉴를 클릭합니다.
3. `아카이브에 보관` 을 선택합니다.
4. 대화가 사이드바에서 사라지는 것을 확인합니다.

이때 대화가 삭제된 것은 아닙니다.

사이드바에서만 사라진 것입니다.

### 4.4 보관한 대화 다시 꺼내기

보관된 대화는 설정 메뉴에서 다시 찾을 수 있습니다.

기본 경로는 다음과 같습니다.

1. 화면 좌측 하단의 계정 아이콘을 클릭합니다.
2. `설정` 으로 이동합니다.
3. `데이터 제어` 항목을 선택합니다.
4. `아카이브에 보관된 채팅` 항목을 찾습니다.
5. `관리` 를 선택합니다.
6. 보관된 대화 목록에서 원하는 대화를 찾습니다.
7. `보관 해제` 를 선택합니다.

보관 해제를 하면 해당 대화가 다시 일반 대화 목록으로 돌아옵니다.

즉, 아카이브는 대화를 없애는 기능이 아니라 **사이드바를 정리하기 위해 잠시 치워 두는 기능** 입니다.

### 4.5 삭제가 적합한 경우

삭제는 더 신중하게 사용해야 합니다.

삭제는 복구할 수 없는 정리 방식입니다.

삭제가 적합한 경우는 다음과 같습니다.

* 다시 볼 필요가 전혀 없는 일회성 질문
* 테스트용으로 잠깐 만든 대화
* 잘못 만든 대화
* 실수로 민감한 정보를 입력한 대화
* 사이드바에 남겨둘 필요가 없는 잡담성 대화

특히 민감한 정보를 실수로 입력했다면 해당 대화를 삭제하는 것이 좋습니다.

하지만 중요한 점이 있습니다.

삭제는 사후 조치일 뿐입니다. 더 중요한 것은 애초에 민감한 정보를 입력하지 않는 것입니다.

### 4.6 삭제 실습

대화를 삭제하는 절차는 다음과 같습니다.

1. 사이드바에서 삭제할 대화를 찾습니다.
2. 대화 제목 옆의 더보기 메뉴를 클릭합니다.
3. `삭제` 를 선택합니다.
4. 확인 창이 나타나면 내용을 확인합니다.
5. 최종적으로 삭제를 진행합니다.

삭제 후에는 해당 대화를 복구할 수 없습니다.

따라서 삭제하기 전에는 정말 다시 볼 필요가 없는지 확인해야 합니다.

## 5. 삭제와 데이터 제어 설정의 차이

삭제와 데이터 제어 설정은 서로 다른 개념입니다.

많은 사용자가 대화를 삭제하면 데이터 관련 설정까지 모두 해결된다고 생각할 수 있습니다. 하지만 두 기능의 목적은 다릅니다.

| 구분    | 삭제                  | 데이터 제어 설정              |
| ----- | ------------------- | ---------------------- |
| 성격    | 사후 조치               | 사전 설정                  |
| 목적    | 특정 대화를 내 대화 목록에서 제거 | 앞으로의 데이터 사용 방식을 설정     |
| 적용 대상 | 이미 생성된 특정 대화        | 이후 대화나 계정 설정           |
| 예시    | 실수로 입력한 대화 삭제       | 대화가 모델 개선에 사용되지 않도록 설정 |

삭제는 내 화면에서 특정 대화를 지우는 조치입니다.

반면 데이터 제어 설정은 앞으로 내 대화가 어떻게 사용될지 설정하는 영역입니다.

민감한 정보를 실수로 입력했다면 삭제를 해야 합니다.

하지만 더 중요한 것은 예방입니다.

업무에서 ChatGPT 를 사용할 때는 다음 원칙을 지켜야 합니다.

* 고객 개인정보를 그대로 입력하지 않습니다.
* 회사 기밀을 입력하지 않습니다.
* 계약 조건이나 내부 시스템 정보를 그대로 입력하지 않습니다.
* 개인명, 회사명, 고객명은 익명화합니다.
* 필요한 경우 회사의 AI 사용 정책을 먼저 확인합니다.

삭제는 사후 조치이고, 예방이 먼저입니다.

## 6. 채팅 고정: 프롬프트 라이브러리 만들기

### 6.1 채팅 고정의 의미

채팅 고정은 자주 사용하는 대화를 사이드바 상단에 고정해 두는 기능입니다.

일반적으로는 즐겨찾기처럼 보일 수 있습니다.

하지만 업무 활용 관점에서는 더 중요한 의미가 있습니다.

채팅 고정은 반복 업무에 사용하는 프롬프트를 빠르게 불러오기 위한 **개인 프롬프트 라이브러리** 로 사용할 수 있습니다.

예를 들어 다음과 같은 업무가 있다고 해보겠습니다.

* 매주 회의록을 정리한다.
* 매월 실적 보고서를 요약한다.
* 고객 이메일을 자주 작성한다.
* 강의 자료 초안을 반복해서 만든다.
* 번역 후 문체를 다듬는 작업을 자주 한다.

이런 업무는 매번 새로운 프롬프트를 작성할 필요가 없습니다.

잘 만들어진 대화를 하나 만들고, 제목을 정리한 뒤, 고정해 두면 됩니다. 이후에는 그 대화를 열고 새로운 입력값만 넣으면 됩니다.

### 6.2 고정할 대화의 기준

모든 대화를 고정하면 오히려 불편해집니다.

고정 대화가 너무 많아지면 바로 찾는다는 장점이 사라집니다.

따라서 고정 대화는 3개에서 5개 정도로 제한하는 것이 좋습니다.

고정할 만한 대화는 다음 기준으로 선택합니다.

| 기준    | 설명                        |
| ----- | ------------------------- |
| 반복성   | 자주 반복되는 업무인가?             |
| 재사용성  | 같은 프롬프트 구조를 계속 사용할 수 있는가? |
| 중요도   | 업무 생산성에 직접적인 영향을 주는가?     |
| 안정성   | 결과물 형식이 비교적 일정한가?         |
| 관리 가치 | 사이드바 상단에 둘 만큼 자주 쓰는가?     |

예를 들어 다음과 같은 대화는 고정하기 좋습니다.

* 회의록 요약 템플릿
* 임원 보고서 요약 템플릿
* 고객 사과 메일 템플릿
* 강의 원고 구조화 템플릿
* 번역 후 비즈니스 문체 수정 템플릿

반대로 다음 대화는 고정하지 않는 것이 좋습니다.

* 한 번 쓰고 끝나는 일회성 질문
* 특정 날짜에만 필요한 단기 대화
* 내용이 불명확한 실험용 대화
* 거의 사용하지 않는 오래된 프롬프트

## 7. 고정 대화 실습: 회의록 요약 프롬프트 만들기

이번에는 반복 업무에 사용할 수 있는 고정 대화를 직접 만들어 보겠습니다.

예시는 회의록 요약입니다.

목표는 다음과 같습니다.

> 회의 메모를 붙여 넣으면 업무 보고용 회의록 초안으로 정리해 주는 고정 대화를 만든다.

### 7.1 새 대화에서 프롬프트 입력하기

먼저 새 대화를 시작합니다.

그리고 아래 프롬프트를 입력합니다.

<pre><code>나는 팀 회의 내용을 정리하는 실무자입니다.

앞으로 이 대화에서는 회의 메모를 업무 보고용 회의록으로 정리하려고 합니다.

아래 입력값을 기준으로 회의록 초안을 만들어 주세요.

조건:
- 핵심 의사결정 사항을 먼저 정리해 주세요.
- 담당자와 마감일을 분리해 주세요.
- 불명확한 내용은 "확인 필요"로 표시해 주세요.
- 임원 보고용으로 간결하게 작성해 주세요.

출력 형식:
1. 회의 개요
2. 핵심 결정 사항
3. 액션아이템 표
4. 확인 필요 사항

입력값:
[여기에 회의 메모를 붙여넣기]</code></pre>

이 프롬프트의 핵심은 단순히 회의록을 한 번 만들어 달라는 것이 아닙니다.

앞으로 이 대화에서 회의 메모를 붙여 넣으면 같은 방식으로 회의록을 정리하겠다는 **업무 규칙** 을 설정하는 것입니다.

### 7.2 테스트용 회의 메모 입력하기

이제 실제로 작동하는지 확인하기 위해 아래 가상의 회의 메모를 입력합니다.

<pre><code>6월 18일 오전 10시에 신제품 출시 준비 회의를 진행함.

참석자는 김민수 팀장, 이서연 매니저, 박지훈 대리, 정유진 과장, 그리고 디자인팀 최현아 책임이었음.
회의 주제는 7월 말 출시 예정인 B2B 고객관리 솔루션 신규 패키지의 출시 일정, 홍보 자료, 영업 준비 현황 점검이었음.

우선 출시일은 기존 7월 25일에서 7월 31일로 조정하기로 함. 개발팀에서 관리자 권한 설정 기능 테스트가 아직 끝나지 않았고, 영업팀에서도 데모 자료 준비 시간이 더 필요하다는 의견이 있었음. 다만 대표 보고 일정 때문에 7월 말 안에는 반드시 출시하는 방향으로 정리함.

가격 정책은 월 구독형 기준으로 베이직 9만 원, 프로 19만 원, 엔터프라이즈는 별도 견적 방식으로 가는 안이 유력함. 최종 확정은 아직 아님. 김민수 팀장이 재무팀과 다시 검토 후 확정하기로 함. 이번 주 금요일까지 가격안 정리 필요.

홍보 랜딩페이지는 기존 초안을 유지하되, 고객 사례 영역을 상단으로 올리기로 함. 디자인팀 최현아 책임이 페이지 구조 수정안을 다음 주 초까지 공유하기로 함. 구체적인 날짜는 회의 중 명확히 정하지 않음.

영업팀 데모 스크립트는 이서연 매니저가 작성 중이며, 현재 약 60% 완료된 상태라고 함. 다음 주 수요일까지 1차 버전을 공유하고, 이후 영업팀 내부 피드백을 반영하기로 함.

고객 대상 웨비나는 7월 셋째 주에 진행하는 방향으로 논의됨. 정확한 날짜와 시간은 아직 미정. 박지훈 대리가 가능한 일정 후보를 정리해서 공유하기로 함. 마감일은 따로 정하지 않았으나, 김민수 팀장이 가능하면 이번 주 안에 정리해 달라고 요청함.

기술 FAQ 문서는 개발팀에서 초안을 작성하고, 정유진 과장이 고객지원팀 관점에서 보완하기로 함. FAQ에는 권한 설정, 데이터 업로드, 결제 방식, 관리자 계정 관리 관련 질문을 반드시 포함해야 한다는 의견이 있었음. 정유진 과장은 7월 10일까지 고객지원팀 검토본을 만들기로 함.

출시 전 내부 교육은 온라인으로 1회 진행하기로 함. 대상은 영업팀과 고객지원팀이며, 교육 시간은 90분 정도로 예상함. 교육 담당자는 아직 확정되지 않음. 후보로 이서연 매니저와 정유진 과장이 언급됨.

이번 회의에서 결정된 주요 내용은 출시일을 7월 31일로 조정한다는 점, 랜딩페이지 구조를 수정한다는 점, 영업 데모 스크립트와 기술 FAQ를 출시 전까지 준비한다는 점임.

다음 회의는 6월 25일 오후 2시에 진행하기로 함. 장소는 8층 회의실 또는 온라인 병행으로 진행할 수 있음. 최종 회의 방식은 김민수 팀장이 다시 공지하기로 함.</code></pre>

ChatGPT 가 회의 메모를 다음 형식으로 정리하는지 확인합니다.

* 회의 개요
* 핵심 결정 사항
* 액션아이템 표
* 확인 필요 사항

여기서 중요한 것은 회의록 결과물이 완벽한지 여부가 아닙니다.

핵심은 이 대화가 앞으로 회의 메모를 넣으면 같은 방식으로 회의록을 만들어 주는 **반복 가능한 업무 도구** 가 되었다는 점입니다.

### 7.3 대화 이름 바꾸기

이제 이 대화의 제목을 바꿉니다.

더보기 메뉴에서 `이름 바꾸기` 를 선택하고 아래처럼 입력합니다.

<pre><code>[회의] 업무 보고용 회의록 요약 템플릿</code></pre>

이 제목은 업무 영역과 산출물을 모두 포함합니다.

나중에 사이드바에서 바로 찾기 쉽습니다.

### 7.4 채팅 고정하기

마지막으로 이 대화를 고정합니다.

1. 대화 제목 옆의 더보기 메뉴를 클릭합니다.
2. `채팅 고정` 을 선택합니다.
3. 사이드바 상단의 고정된 대화 영역에 해당 대화가 표시되는지 확인합니다.

이제 이 대화는 개인 프롬프트 라이브러리의 일부가 되었습니다.

다음에 회의록을 정리할 때는 새 프롬프트를 작성할 필요가 없습니다.

고정된 대화를 열고 회의 메모만 붙여 넣으면 됩니다.

## 8. 사이드바 운영 전략

대화 관리 기능을 개별적으로 아는 것도 중요하지만, 실제 업무에서는 하나의 운영 전략으로 사용하는 것이 더 중요합니다.

추천하는 운영 방식은 다음과 같습니다.

| 상황                  | 사용할 기능      |
| ------------------- | ----------- |
| 중요한 대화인데 제목이 모호함    | 이름 바꾸기      |
| 당장은 안 쓰지만 나중에 참고 가능 | 아카이브에 보관    |
| 다시 볼 필요가 없거나 리스크 있음 | 삭제          |
| 자주 반복해서 쓰는 프롬프트     | 채팅 고정       |
| 업무가 끝났지만 기록 가치 있음   | 이름 바꾸기 후 보관 |
| 반복 업무 템플릿으로 쓸 가치 있음 | 이름 바꾸기 후 고정 |

사이드바를 정리할 때는 다음 기준을 사용하면 좋습니다.

1. 다시 쓸 대화인가?
2. 나중에 참고할 가치가 있는가?
3. 제목만 보고 내용을 알 수 있는가?
4. 반복 업무에 사용할 수 있는가?
5. 보안상 남겨두면 안 되는 내용이 있는가?

이 질문에 따라 이름을 바꾸고, 보관하고, 삭제하고, 고정하면 됩니다.

## 9. 실습하기

이번 강의에서는 대화 관리 기능을 직접 사용해 보겠습니다.

### 9.1 이름 바꾸기 실습

기존 대화 중 하나를 골라 아래 형식으로 제목을 바꿉니다.

<pre><code>[업무영역] 구체적 목적 + 산출물</code></pre>

예시는 다음과 같습니다.

<pre><code>[이메일] 고객 배송 지연 사과 메일 템플릿</code></pre>

### 9.2 아카이브에 보관 실습

지금은 필요 없지만 나중에 참고할 수 있는 대화를 하나 골라 아카이브에 보관합니다.

그다음 설정 메뉴에서 다시 찾아 보관 해제까지 진행해 봅니다.

### 9.3 삭제 실습

다시 볼 필요가 없는 일회성 대화를 하나 선택해 삭제합니다.

삭제 전에는 복구할 수 없다는 점을 확인합니다.

### 9.4 채팅 고정 실습

회의록 요약 프롬프트를 새 대화에 입력합니다.

테스트용 회의 메모를 붙여 넣어 결과를 확인합니다.

대화 제목을 아래처럼 바꿉니다.

<pre><code>[회의] 업무 보고용 회의록 요약 템플릿</code></pre>

그다음 해당 대화를 고정합니다.

## 10. 실습 완료 기준

이번 강의의 실습은 다음 기준으로 완료할 수 있습니다.

* 대화 하나의 이름을 `[업무영역] 구체적 목적 + 산출물` 형식으로 바꿨다.
* 대화 하나를 아카이브에 보관했다.
* 보관된 대화를 설정 메뉴에서 다시 찾아 보관 해제했다.
* 다시 볼 필요가 없는 대화 하나를 삭제했다.
* 삭제는 복구할 수 없다는 점을 이해했다.
* 회의록 요약 프롬프트 대화를 만들었다.
* 테스트용 회의 메모로 결과를 확인했다.
* 해당 대화의 이름을 바꾸고 채팅 고정까지 완료했다.
* 사이드바 상단에서 고정된 대화를 확인했다.

## 11. 핵심 정리

* ChatGPT 를 업무에 자주 사용하면 사이드바에 대화가 빠르게 쌓입니다.
* 대화 관리는 ChatGPT 를 업무 도구로 쓰기 위한 기본 관리 역량입니다.
* 이름 바꾸기는 대화를 나중에 찾기 쉽게 만드는 기능입니다.
* 추천 제목 형식은 `[업무영역] 구체적 목적 + 산출물` 입니다.
* 아카이브에 보관은 대화를 숨기는 비파괴 정리 기능입니다.
* 삭제는 대화를 지우는 기능이며, 삭제 후에는 복구할 수 없습니다.
* 삭제와 데이터 제어 설정은 서로 다른 개념입니다.
* 민감정보는 삭제보다 입력하지 않는 예방이 먼저입니다.
* 채팅 고정은 반복 업무 프롬프트를 빠르게 불러오기 위한 기능입니다.
* 고정 대화는 3개에서 5개 정도로 제한하는 것이 좋습니다.
* 반복 업무 프롬프트를 고정하면 개인 프롬프트 라이브러리처럼 사용할 수 있습니다.
* ChatGPT 를 잘 쓰는 사람은 좋은 질문만 하는 사람이 아니라, 좋은 대화를 다시 찾고 재사용할 수 있게 관리하는 사람입니다.
