# 04-2. 그룹 채팅: 팀과 함께 ChatGPT 사용하기

## 1. 이 강의에서 배울 내용

이번 강의에서는 ChatGPT 의 **그룹 채팅** 기능을 다룹니다.

ChatGPT 를 혼자 사용할 때는 나와 ChatGPT 사이의 대화만 존재합니다. 하지만 실제 업무는 대부분 혼자 끝나지 않습니다. 팀원과 함께 논의하고, 의견을 모으고, 결과를 정리하고, 의사결정을 내려야 합니다.

그룹 채팅은 여러 사람과 ChatGPT 가 하나의 대화방에서 함께 대화할 수 있게 해 주는 협업 기능입니다.

이 강의를 통해 다음 내용을 익힐 수 있습니다.

* 그룹 채팅이 1:1 채팅과 무엇이 다른지 이해할 수 있습니다.
* 그룹 채팅이 팀 협업에 필요한 이유를 설명할 수 있습니다.
* 새 그룹 채팅을 만들고 초대 링크로 팀원을 초대할 수 있습니다.
* 기존 대화를 그룹 채팅으로 전환할 때 개인 대화가 어떻게 분리되는지 이해할 수 있습니다.
* `@ChatGPT` 호출 방식과 자동 응답 방식의 차이를 구분할 수 있습니다.
* 그룹 전용 맞춤 지침으로 ChatGPT 의 역할을 설정할 수 있습니다.
* 팀 논의를 회의록, 추천안, 실행 과제로 정리할 수 있습니다.
* 초대 링크, 파일 공유, 개인 메모리, 사용량과 관련된 보안 원칙을 이해할 수 있습니다.

## 2. 왜 그룹 채팅이 필요한가

회사에서 ChatGPT 를 쓰다 보면 이런 장면이 자주 생깁니다.

한 사람이 ChatGPT 에게 질문하고 좋은 답을 받습니다. 그리고 그 답을 복사해서 메신저나 이메일로 팀에 공유합니다.

그런데 팀원들이 보는 것은 대개 최종 답변뿐입니다.

어떤 프롬프트를 입력했는지, 어떤 조건을 줬는지, 어디를 다시 고쳤는지, 어떤 이유로 결과가 바뀌었는지는 보이지 않습니다.

그러다 보니 다음과 같은 문제가 생깁니다.

* 같은 질문을 여러 사람이 반복합니다.
* 팀원마다 서로 다른 AI 답변을 받습니다.
* 결과만 공유되어 프롬프트 작성 과정이 보이지 않습니다.
* AI 결과를 놓고 다시 별도 회의를 해야 합니다.
* AI 활용 노하우가 개인 안에만 머뭅니다.

문제는 ChatGPT 를 못 써서가 아닙니다.

핵심 문제는 **AI 를 쓰는 과정이 개인 안에 갇혀 있다는 것** 입니다.

그룹 채팅은 이 문제를 줄여줍니다.

그룹 채팅에서는 여러 사용자와 ChatGPT 가 하나의 대화방에서 함께 대화합니다. 팀원들이 같은 질문, 같은 답변, 같은 수정 과정, 같은 의사결정 흐름을 함께 볼 수 있습니다.

즉, 그룹 채팅은 AI 답변을 단순히 공유하는 기능이 아닙니다.

> 그룹 채팅은 팀원들과 ChatGPT 가 같은 대화 맥락을 공유하는 협업 공간입니다.

## 3. 그룹 채팅의 기본 개념

### 3.1 1:1 채팅과 그룹 채팅의 차이

ChatGPT 의 일반적인 대화는 1:1 채팅입니다.

사용자 한 명이 ChatGPT 와 대화하고, 그 대화의 맥락은 해당 사용자에게만 보입니다.

반면 그룹 채팅은 여러 명의 사용자와 ChatGPT 가 하나의 대화방에 함께 참여합니다.

| 구분       | 1:1 채팅              | 그룹 채팅                 |
| -------- | ------------------- | --------------------- |
| 참여자      | 사용자 1명 + ChatGPT    | 여러 사용자 + ChatGPT      |
| 대화 맥락    | 개인만 확인              | 참여자 모두 확인             |
| 적합한 상황   | 개인 질문, 개인 작업, 초안 작성 | 팀 회의, 브레인스토밍, 공동 검토   |
| AI 활용 방식 | 개인 비서               | 공동 정리자, 회의 서기, 분석 보조자 |
| 결과 공유    | 결과를 따로 복사해 공유       | 대화방 안에서 함께 확인         |

1:1 채팅은 개인 생산성에 강합니다.

그룹 채팅은 팀 생산성에 강합니다.

### 3.2 그룹 채팅이 적합한 업무

그룹 채팅은 다음과 같은 상황에 잘 맞습니다.

* 회의 중 나온 의견을 즉시 요약할 때
* 여러 팀원의 아이디어를 비교표로 정리할 때
* 같은 문서나 파일을 놓고 함께 검토할 때
* 회식 장소, 교육 주제, 캠페인 방향처럼 공동 의사결정이 필요할 때
* 팀원들이 서로의 프롬프트 작성 방식을 보고 학습해야 할 때
* 논의 결과를 실행 과제로 정리해야 할 때

반대로 모든 작업에 그룹 채팅이 적합한 것은 아닙니다.

장기적인 자료 축적, 복잡한 데이터 분석, 개인 맞춤형 작업은 1:1 채팅이나 프로젝트 기능이 더 적합할 수 있습니다.

## 4. 기존 대화를 그룹 채팅으로 만들면 어떻게 되는가

많은 사용자가 처음에 이런 걱정을 합니다.

> 내 기존 ChatGPT 대화에 팀원을 초대하면, 내 과거 질문이 전부 공개되는 것 아닌가?

핵심은 다음과 같습니다.

기존 1:1 채팅에 사람을 초대하면 원본 대화가 그대로 열리는 것이 아닙니다. 그 시점의 대화가 복사되어 별도의 그룹 채팅이 새로 만들어집니다.

이 개념을 `Copy-on-add` 라고 이해하면 됩니다.

쉽게 말해, 사람을 추가하는 순간 기존 대화를 복사한 새 그룹 채팅이 만들어지는 방식입니다.

| 구분             | 설명                    |
| -------------- | --------------------- |
| 원본 1:1 채팅      | 내 개인 대화로 그대로 유지       |
| 새 그룹 채팅        | 초대 시점의 대화를 복사해 별도로 생성 |
| 이후 그룹 대화       | 참여자들이 함께 볼 수 있음       |
| 개인 메모리         | 그룹 참여자에게 공유되지 않음      |
| 그룹에 올린 메시지와 파일 | 참여자에게 공유됨             |

따라서 개인 대화 전체가 그대로 공개되는 것은 아닙니다.

하지만 그룹 채팅이 만들어진 뒤 그 안에 직접 입력한 메시지와 업로드한 파일은 참여자에게 공유됩니다.

이 차이를 반드시 구분해야 합니다.

> 개인 대화와 개인 메모리는 공유되지 않습니다.
> 하지만 그룹 채팅 안에 올린 메시지와 파일은 참여자에게 공유됩니다.

## 5. 그룹 채팅 만들기

### 5.1 새 그룹 채팅 시작하기

그룹 채팅은 새 채팅에서 시작할 수도 있고, 기존 채팅에서 만들 수도 있습니다.

기본 흐름은 다음과 같습니다.

1. ChatGPT 에서 새 채팅을 시작합니다.
2. 간단한 메시지를 입력해 대화를 생성합니다.
3. 대화 화면에서 더보기 메뉴 또는 사람 아이콘을 선택합니다.
4. `그룹 채팅 시작` 을 선택합니다.
5. 안내 화면을 확인합니다.
6. 그룹 링크를 생성합니다.

화면 구성은 웹, 모바일 앱, 데스크톱 앱에 따라 조금씩 다를 수 있습니다. 중요한 것은 그룹 채팅을 시작하는 진입점을 찾는 것입니다.

### 5.2 안내 화면의 의미

그룹 채팅을 시작하면 다음과 같은 의미의 안내가 표시될 수 있습니다.

<pre><code>이 대화에서 그룹 채팅을 시작하세요.
이 대화만 공유됩니다.
개인 ChatGPT 메모리는 언제나 비공개로 유지됩니다.</code></pre>

이 안내의 핵심은 두 가지입니다.

첫째, 공유되는 것은 해당 그룹 채팅입니다.

둘째, 개인 메모리는 공유되지 않습니다.

즉, 기존의 모든 대화가 팀원에게 공개되는 것이 아니라, 그룹 채팅으로 만든 해당 대화만 공유됩니다.

### 5.3 그룹 이름 정하기

그룹 채팅이 만들어졌다면 이름을 주제에 맞게 바꾸는 것이 좋습니다.

예를 들어 다음과 같이 이름을 붙일 수 있습니다.

<pre><code>사내 AI 교육 주제 선정
신제품 캠페인 아이디어 회의
고객 제안서 목차 검토
팀 회식 장소 선정
워크숍 프로그램 구성</code></pre>

그룹 이름을 명확히 해두면 사이드바에서 개인 채팅과 구분하기 쉽습니다.

## 6. 초대 링크로 팀원 초대하기

그룹 채팅에 팀원을 초대하려면 초대 링크를 사용합니다.

기본 절차는 다음과 같습니다.

1. 그룹 채팅 상단의 그룹 이름 또는 사람 아이콘을 선택합니다.
2. 초대 링크를 생성합니다.
3. 링크 복사를 선택합니다.
4. 메신저, 이메일, 협업 도구를 통해 팀원에게 전달합니다.
5. 팀원이 링크로 접속해 본인의 계정으로 로그인합니다.
6. 팀원이 그룹 채팅에 입장했는지 확인합니다.

팀원이 들어오면 대화방에 입장 안내 메시지가 표시됩니다.

이 메시지가 보이면 초대가 정상적으로 완료된 것입니다.

### 6.1 초대 후 역할 입력하기

팀원이 들어온 뒤에는 각자 자신의 역할을 한 줄로 입력하게 하는 것이 좋습니다.

예를 들어 다음과 같이 입력할 수 있습니다.

<pre><code>저는 마케팅 관점에서 의견을 내겠습니다.</code></pre>

<pre><code>저는 영업 담당자 관점에서 검토하겠습니다.</code></pre>

<pre><code>저는 고객 관점에서 우려 사항을 제시하겠습니다.</code></pre>

<pre><code>저는 실행 가능성과 일정 관점에서 보겠습니다.</code></pre>

이렇게 역할을 미리 선언하면 이후 논의가 더 정돈됩니다.

### 6.2 실습 주제 선택 기준

처음 실습할 때는 민감하지 않은 가벼운 주제를 사용하는 것이 좋습니다.

예를 들어 다음 주제가 적합합니다.

* 팀 회식 장소 선정
* 사내 AI 교육 주제 선정
* 워크숍 프로그램 구성
* 신제품 캠페인 문구 아이디어
* 신규 업무 자동화 아이디어 발굴

처음부터 고객 정보, 계약 금액, 내부 전략, 실제 인사 정보가 들어간 주제로 실습하지 않습니다.

## 7. @ChatGPT 로 AI 호출하기

### 7.1 그룹 채팅에서 ChatGPT 를 부르는 방식

그룹 채팅에서는 사람들끼리 대화할 수 있습니다.

이때 ChatGPT 를 대화에 참여시키려면 메시지 입력창에서 `@` 를 입력합니다. 그러면 대화방 참여자와 ChatGPT 가 선택지로 나타납니다.

이 중 `ChatGPT` 를 선택한 뒤 요청을 입력하면, ChatGPT 가 해당 대화 맥락을 보고 답변합니다.

예를 들어 다음처럼 입력할 수 있습니다.

<pre><code>@ChatGPT 지금까지 나온 의견을 핵심 쟁점별로 정리해 주세요.</code></pre>

또는 다음처럼 요청할 수 있습니다.

<pre><code>@ChatGPT 지금까지의 논의를 바탕으로 최종 후보 2개와 추천 이유를 정리해 주세요.</code></pre>

이렇게 `@ChatGPT` 로 호출하면 ChatGPT 는 팀 대화에 참여하는 회의 서기나 정리자처럼 작동합니다.

### 7.2 왜 @ 호출이 중요한가

그룹 채팅에서는 ChatGPT 가 항상 모든 메시지에 답하는 것이 좋지 않을 수 있습니다.

사람들끼리 의견을 주고받아야 하는 상황에서 AI 가 너무 자주 끼어들면 대화 흐름이 끊깁니다.

따라서 실무 회의에서는 일반적으로 다음 방식이 안정적입니다.

1. 팀원들이 먼저 의견을 냅니다.
2. 충분한 의견이 쌓입니다.
3. 특정 시점에 `@ChatGPT` 를 호출합니다.
4. ChatGPT 에게 정리, 비교, 요약, 실행 과제 도출을 요청합니다.
5. 사람이 결과를 검토하고 의사결정합니다.

핵심은 ChatGPT 의 판단을 그대로 따르는 것이 아닙니다.

ChatGPT 가 만든 표와 요약을 기준으로 사람이 더 빠르게 의견을 좁히는 것입니다.

## 8. 자동 응답과 언급 모드

그룹 채팅에서 ChatGPT 의 참여 방식은 크게 두 가지로 이해할 수 있습니다.

| 방식    | 설명                                | 적합한 상황              |
| ----- | --------------------------------- | ------------------- |
| 자동 응답 | ChatGPT 가 대화 흐름을 보고 필요하다고 판단하면 응답 | 가벼운 아이디어 회의, 소규모 논의 |
| 언급 모드 | 사용자가 `@ChatGPT` 로 부를 때 응답         | 회의, 의사결정, 업무 논의     |

자동 응답은 자연스러운 대화에는 편리합니다.

예를 들어 회식 장소를 가볍게 논의하거나, 브레인스토밍 초반에 아이디어를 자유롭게 나누는 상황에서는 ChatGPT 가 중간중간 도와줘도 괜찮습니다.

하지만 회의나 의사결정 상황에서는 `@ChatGPT` 호출 중심이 더 안전합니다.

특히 다음 상황에서는 언급 모드를 권장합니다.

* 팀원 의견을 먼저 충분히 들어야 할 때
* 회의 흐름을 사람이 통제해야 할 때
* AI 가 중간에 끼어드는 것이 부담스러울 때
* 최종 정리 시점에만 AI 를 사용하고 싶을 때
* 공식 회의록이나 실행 과제를 만들 때

정리하면 다음과 같습니다.

> 가벼운 대화는 자동 응답도 괜찮습니다.
> 업무 회의와 의사결정은 `@ChatGPT` 호출 중심으로 운영하는 것이 안정적입니다.

## 9. 그룹 전용 맞춤 지침 설정하기

### 9.1 그룹 전용 맞춤 지침이란 무엇인가

그룹 채팅에서는 ChatGPT 의 역할과 응답 방식을 해당 그룹에 맞게 설정할 수 있습니다.

이를 그룹 전용 맞춤 지침이라고 이해하면 됩니다.

개인 계정의 맞춤 지침과는 별도로, 특정 그룹 채팅 안에서만 적용되는 지침입니다.

예를 들어 어떤 그룹에서는 ChatGPT 를 회의 서기로 사용할 수 있습니다. 다른 그룹에서는 아이디어 평가자, 제안서 검토자, 마케팅 전략가, 실행 과제 정리자로 사용할 수 있습니다.

즉, 그룹 전용 맞춤 지침은 ChatGPT 에게 이렇게 말하는 것과 같습니다.

> 이 방에서는 당신이 어떤 역할을 해야 하는지 정해 주겠습니다.

### 9.2 그룹 맞춤 지침 접근 방법

그룹 채팅에서 맞춤 지침을 설정하는 기본 흐름은 다음과 같습니다.

1. 그룹 채팅 상단의 대화 제목을 클릭합니다.
2. 그룹 설정 메뉴를 엽니다.
3. `ChatGPT 맞춤 설정` 또는 유사한 메뉴를 선택합니다.
4. 그룹에서 사용할 지침을 입력합니다.
5. 저장합니다.

설정 메뉴에서는 보통 다음 항목도 함께 확인할 수 있습니다.

* 참여자 목록
* 그룹 링크 관리
* 그룹 이름 바꾸기
* ChatGPT 맞춤 설정
* 자동 응답 설정

화면 이름은 업데이트에 따라 달라질 수 있지만, 핵심은 그룹 채팅 안에서 ChatGPT 의 역할을 별도로 정할 수 있다는 점입니다.

## 10. 그룹 맞춤 지침 예시

아래 지침은 ChatGPT 를 회의 서기이자 아이디어 정리자로 설정하는 예시입니다.

그룹 채팅 실습에서 그대로 사용할 수 있습니다.

<pre><code>당신은 이 그룹 채팅의 회의 서기이자 아이디어 정리자입니다.

역할:
- 참여자들이 낸 의견을 구조화합니다.

- 의사결정에 필요한 기준을 제안합니다.
- 마지막에는 실행 과제를 정리합니다.

응답 원칙:
- 사람들 간 대화에는 불필요하게 끼어들지 않습니다.
- 명시적으로 호출받으면 답변합니다.
- 답변은 짧고 실무적으로 작성합니다.

* 의견을 구조화합니다.
* 중복 의견을 묶습니다.
* 차이점을 구분합니다.
* 의사결정 기준을 제안합니다.
* 실행 과제를 정리합니다.
* 확인이 필요한 내용은 추정하지 않고 표시합니다.

이렇게 역할을 정해두면 매번 길게 설명하지 않아도 됩니다.

## 11. 팀원 의견을 먼저 입력하기

그룹 채팅을 사용할 때 중요한 운영 원칙이 있습니다.

> ChatGPT 를 먼저 부르지 말고, 팀원들이 먼저 의견을 냅니다.

예를 들어 `사내 AI 교육 주제 선정` 을 실습 주제로 정했다고 가정해 보겠습니다.

팀원들은 먼저 각자의 의견을 아래 형식으로 입력합니다.

<pre><code>제가 추천하는 교육 주제는 "ChatGPT 로 회의록 자동 정리하기"입니다.

추천 이유:
- 대부분의 팀이 회의록 작성에 시간을 많이 씁니다.
- 난이도가 낮아 초급자도 바로 따라 할 수 있습니다.
- 교육 후 바로 업무 적용이 가능합니다.

우려 사항:
- 보안상 실제 회의 내용을 그대로 입력하기 어려울 수 있습니다.</code></pre>

다른 팀원도 각자 같은 방식으로 의견을 입력합니다.

최소 3개 이상의 의견이 쌓인 뒤 ChatGPT 를 호출하면 좋습니다.

이렇게 하면 ChatGPT 가 사람들의 의견을 기준으로 정리할 수 있습니다.

반대로 의견이 충분히 쌓이기 전에 ChatGPT 를 먼저 호출하면, AI 가 논의를 주도해 버릴 수 있습니다.

팀 협업에서는 사람이 먼저 의견을 내고, ChatGPT 는 그 의견을 정리하는 역할로 두는 것이 안정적입니다.

## 12. 팀 논의 정리 요청하기

팀원들의 의견이 충분히 쌓였다면 ChatGPT 를 호출합니다.

아래 프롬프트를 사용할 수 있습니다.

<pre><code>@ChatGPT 지금까지 팀원들이 제안한 의견을 정리해 주세요.

목적:
우리는 이 논의를 바탕으로 최종 후보 2개를 선정하려고 합니다.

정리 기준:
- 업무 적용 가능성
- 초급자 실습 적합성
- 교육 후 결과물 명확성
- 보안 리스크
- 준비 난이도
- 기대 효과

출력 형식:
1. 의견별 요약표
2. 기준별 평가표
3. 최종 추천 후보 2개
4. 추천 이유
5. 실행 전 확인해야 할 사항
6. 다음 회의까지의 액션 아이템</code></pre>

이 프롬프트에는 다음 요소가 포함되어 있습니다.

| 요소    | 내용                         |
| ----- | -------------------------- |
| 호출 대상 | `@ChatGPT`                 |
| 작업    | 팀원 의견 정리                   |
| 목적    | 최종 후보 2개 선정                |
| 기준    | 업무 적용 가능성, 보안 리스크, 기대 효과 등 |
| 출력 형식 | 요약표, 평가표, 추천 후보, 액션 아이템    |

이렇게 요청하면 ChatGPT 는 단순 요약이 아니라 의사결정에 필요한 비교 자료를 만들 수 있습니다.

## 13. 결과를 더 간결하게 수정하기

ChatGPT 의 첫 답변이 너무 길거나, 회의록에 바로 붙여넣기 어렵다면 후속 요청으로 다듬을 수 있습니다.

핵심은 분량을 줄이되 중요한 내용은 유지하도록 조건을 주는 것입니다.

회의 결과가 실행으로 이어지려면 이 구조가 중요합니다.

## 16. 사용량 차감 원칙

그룹 채팅을 사용할 때 자주 나오는 질문이 있습니다.

> 방장이 상위 요금제면 팀원도 같은 성능을 쓰나요?

그렇지 않습니다.

ChatGPT 응답 사용량은 방장 기준이 아니라, ChatGPT 가 답변하고 있는 질문자의 계정 기준으로 적용됩니다.

예를 들어 상위 요금제 사용자가 만든 그룹 채팅에 무료 사용자가 참여했다고 해서, 무료 사용자가 상위 요금제 한도를 공유받는 것은 아닙니다.

따라서 실무에서는 다음 원칙을 적용할 수 있습니다.

* 긴 문서 요약은 상위 요금제 사용자가 질문합니다.
* 복잡한 분석이나 중요한 결과 생성도 상위 요금제 사용자가 담당합니다.
* 가벼운 의견 정리나 짧은 요약은 누구나 질문할 수 있습니다.
* 팀 회의에서는 질문 담당자를 정해 중복 호출을 줄입니다.
* 사람끼리 주고받는 메시지는 ChatGPT 응답 사용량과 별개입니다.

이 원칙을 알고 있어야 그룹 채팅을 더 효율적으로 운영할 수 있습니다.

## 17. 개인 메모리와 개인 맞춤 지침은 공유되지 않습니다

그룹 채팅에서 ChatGPT 는 사용자의 개인 1:1 채팅, 개인 메모리, 개인 맞춤 지침을 볼 수 없습니다.

이 점은 프라이버시 측면에서 중요합니다.

하지만 다시 강조해야 할 점이 있습니다.

그룹 안에 직접 입력한 메시지와 파일은 참여자에게 공유됩니다.

| 항목             | 공유 여부            |
| -------------- | ---------------- |
| 개인 1:1 채팅      | 공유되지 않음          |
| 개인 메모리         | 공유되지 않음          |
| 개인 맞춤 지침       | 공유되지 않음          |
| 그룹 채팅에 입력한 메시지 | 참여자에게 공유됨        |
| 그룹 채팅에 업로드한 파일 | 참여자에게 공유됨        |
| 초대 링크          | 링크를 가진 사람이 접근 가능 |

따라서 “개인 메모리는 공유되지 않는다” 와 “그룹에 올린 자료는 공유된다” 를 반드시 구분해야 합니다.

### 18.1 민감 정보는 올리지 않습니다

### 18.2 초대 링크 관리

초대 링크는 편리하지만 보안상 중요한 관리 대상입니다.

링크를 가진 사람이 그룹에 들어올 수 있고, 그룹 대화와 공유 파일을 볼 수 있기 때문입니다.

실습이나 회의가 끝났다면 링크 상태를 확인해야 합니다.

관리 방식은 다음과 같습니다.

| 관리 방식  | 의미                   | 사용 상황            |
| ------ | -------------------- | ---------------- |
| 링크 재설정 | 기존 링크를 무효화하고 새 링크 생성 | 링크가 외부에 넓게 퍼졌을 때 |
| 링크 삭제  | 링크를 통한 신규 참여 차단      | 회의나 실습 종료 후      |

실무에서는 필요한 인원이 모두 입장한 뒤, 회의가 끝나면 링크를 삭제하는 습관을 갖는 것이 좋습니다.

## 19. 그룹 채팅의 지원 기능과 한계

그룹 채팅은 팀 협업에 유용하지만, 모든 ChatGPT 기능이 그룹 채팅 안에서 동일하게 작동하는 것은 아닙니다.

그룹 채팅은 실시간 공동 대화와 빠른 합의에 초점이 맞춰져 있습니다.

따라서 일부 고급 기능은 제한될 수 있습니다.

| 구분          | 기능 예시                                                                              | 실무 의미                     |
| ----------- | ---------------------------------------------------------------------------------- | ------------------------- |
| 지원 가능 기능    | 검색, 이미지 업로드와 이해, 파일 업로드와 검색, 받아쓰기                                                  | 회의 자료와 문서 기반 논의 가능        |
| 제한될 수 있는 기능 | Canvas, 전체 음성 대화, Python 또는 데이터 분석, Deep Research, Tasks, Apps, Agent mode, 개인 메모리 | 고급 분석과 장기 작업은 다른 기능 사용 필요 |

이 제한은 단점이라기보다 용도의 구분으로 이해하는 것이 좋습니다.

* 실시간 논의와 빠른 합의: 그룹 채팅
* 장기 자료 관리와 반복 작업: 프로젝트
* 복잡한 데이터 분석: 1:1 채팅 또는 데이터 분석 환경
* 개인 맞춤형 작업: 개인 1:1 채팅

## 20. 흔한 오류와 해결 방법

| 오류 상황                         | 원인                              | 해결 방법                                            |
| ----------------------------- | ------------------------------- | ------------------------------------------------ |
| 팀원이 내 과거 1:1 채팅을 모두 볼까 봐 불안해함 | 그룹 채팅 생성 방식에 대한 오해              | 기존 채팅을 그룹으로 만들면 복사본이 생성되고 원본 개인 대화는 분리된다고 설명합니다. |
| 초대하지 않은 사람이 들어올 가능성이 있음       | 초대 링크가 외부로 전달됨                  | 필요한 인원이 입장한 뒤 링크를 재설정하거나 삭제합니다.                  |
| ChatGPT 가 너무 자주 끼어듦           | 자동 응답 상태에서 대화 흐름을 AI 가 질문으로 판단함 | 회의성 논의에서는 `@ChatGPT` 호출 중심으로 운영합니다.              |
| ChatGPT 가 중요한 의견을 누락함         | 팀원 의견이 길거나 기준이 불명확함             | 모든 참여자의 의견을 빠짐없이 표로 정리해 달라고 다시 요청합니다.            |
| 결과가 너무 일반적임                   | 역할, 목적, 기준, 출력 형식이 부족함          | 프롬프트에 역할, 업무 상황, 평가 기준, 최종 결과물 형식을 추가합니다.        |
| 파일을 올렸는데 고급 데이터 분석이 안 됨       | 그룹 채팅에서 데이터 분석 기능이 제한될 수 있음     | 데이터 분석은 1:1 채팅이나 별도 분석 도구에서 수행하고 결과를 그룹에 공유합니다.  |
| 그룹 채팅을 장기 프로젝트 저장소처럼 사용함      | 그룹 채팅과 프로젝트 기능의 목적을 혼동함         | 일회성 논의는 그룹 채팅, 장기 자료 관리는 프로젝트를 사용합니다.            |
| 민감 자료가 그룹에 공유됨                | 그룹 채팅의 공유 범위를 과소평가함             | 개인정보, 계약 정보, 내부 기밀은 업로드하지 않습니다.                  |
| 액션 아이템이 실행으로 이어지지 않음          | 담당자와 완료 기준이 없음                  | 담당자, 기한, 완료 기준이 포함된 실행 계획표로 다시 변환합니다.            |

## 21. 실습하기

이번 강의에서는 그룹 채팅을 직접 만들어 보고, ChatGPT 를 팀 회의 서기로 사용해 봅니다.

### 21.1 실습 주제 정하기

먼저 논의할 주제를 하나 정합니다.

초급자 실습에는 다음 주제가 적합합니다.

* 팀 회식 장소 선정
* 사내 AI 교육 주제 선정
* 워크숍 프로그램 구성

업무 적용형 실습에는 다음 주제가 적합합니다.

* 신제품 캠페인 문구 아이디어
* 고객 제안서 목차 작성
* 신규 업무 자동화 아이디어 발굴

중요한 기준은 민감 정보 없이 진행할 수 있는 주제여야 한다는 점입니다.

### 21.2 그룹 채팅 만들기

1. ChatGPT 에서 새 채팅을 시작합니다.
2. 그룹 채팅 시작 메뉴를 선택합니다.
3. 그룹 이름을 실습 주제에 맞게 바꿉니다.
4. 초대 링크를 생성합니다.
5. 실습 파트너에게 링크를 공유합니다.
6. 팀원들이 들어왔는지 확인합니다.

### 21.3 그룹 맞춤 지침 설정하기

아래 지침을 그룹 전용 맞춤 지침에 입력합니다.

<pre><code>당신은 이 그룹 채팅의 회의 서기이자 아이디어 정리자입니다.

역할:
- 참여자들이 낸 의견을 구조화합니다.
- 중복 의견을 묶고, 차이점을 구분합니다.
- 의사결정에 필요한 기준을 제안합니다.
- 마지막에는 실행 과제를 정리합니다.

응답 원칙:
- 사람들 간 대화에는 불필요하게 끼어들지 않습니다.
- 명시적으로 호출받으면 답변합니다.
- 답변은 짧고 실무적으로 작성합니다.
- 가능하면 표를 사용합니다.
- 확실하지 않은 내용은 추정하지 말고 "확인 필요"라고 표시합니다.

출력 형식:
1. 핵심 요약
2. 의견 분류표
3. 추천안
4. 리스크 또는 확인 사항
5. 다음 실행 과제</code></pre>

### 21.4 팀원 의견 입력하기

각 팀원이 먼저 의견을 입력합니다.

예시는 다음과 같습니다.

<pre><code>제가 추천하는 교육 주제는 "ChatGPT 로 회의록 자동 정리하기"입니다.

추천 이유:
- 대부분의 팀이 회의록 작성에 시간을 많이 씁니다.
- 난이도가 낮아 초급자도 바로 따라 할 수 있습니다.
- 교육 후 바로 업무 적용이 가능합니다.

우려 사항:
- 보안상 실제 회의 내용을 그대로 입력하기 어려울 수 있습니다.</code></pre>

최소 3개 이상의 의견이 쌓일 때까지 ChatGPT 를 호출하지 않습니다.

### 21.5 ChatGPT 호출하기

의견이 충분히 쌓이면 아래 프롬프트로 ChatGPT 를 호출합니다.

<pre><code>@ChatGPT 지금까지 팀원들이 제안한 의견을 정리해 주세요.

목적:
우리는 이 논의를 바탕으로 최종 후보 2개를 선정하려고 합니다.

정리 기준:
- 업무 적용 가능성
- 초급자 실습 적합성
- 교육 후 결과물 명확성
- 보안 리스크
- 준비 난이도
- 기대 효과

출력 형식:
1. 의견별 요약표
2. 기준별 평가표
3. 최종 추천 후보 2개
4. 추천 이유
5. 실행 전 확인해야 할 사항
6. 다음 회의까지의 액션 아이템</code></pre>

### 21.6 결과 수정하기

결과가 너무 길다면 아래처럼 요청합니다.

<pre><code>@ChatGPT 방금 답변을 회의록에 붙여 넣을 수 있도록 더 간결하게 다시 정리해 주세요.

조건:
- 전체 분량은 1페이지 이내
- 표는 2개 이하
- 최종 추천안과 액션 아이템은 반드시 유지</code></pre>

실행 계획으로 바꾸고 싶다면 아래처럼 요청합니다.

<pre><code>@ChatGPT 최종 추천안을 실행 계획으로 바꿔 주세요.

출력 형식:
| 작업 | 담당자 | 기한 | 필요한 자료 | 완료 기준 |
|---|---|---|---|---|

담당자가 명확하지 않은 작업은 "담당자 지정 필요"라고 표시해 주세요.</code></pre>

### 21.7 초대 링크 정리하기

실습이 끝나면 그룹 설정에서 초대 링크를 확인합니다.

필요한 인원이 모두 입장했고 실습이 끝났다면 링크를 삭제합니다.

외부로 넓게 공유되었다면 링크를 재설정합니다.

## 22. 실습 완료 기준

이번 강의의 실습은 다음 기준으로 완료할 수 있습니다.

* 그룹 채팅이 개인 1:1 채팅과 별도로 생성되었다.
* 실습 주제에 맞게 그룹 이름을 변경했다.
* 초대 링크로 2명 이상이 참여했다.
* 참여자들이 각자의 역할을 입력했다.
* 그룹 전용 맞춤 지침을 설정했다.
* 팀원들이 먼저 의견을 입력했다.
* `@ChatGPT` 를 호출해 의견을 정리했다.
* 결과물에 요약, 비교, 추천안, 액션 아이템이 포함되었다.
* 필요한 경우 결과를 더 간결하게 수정했다.
* 실행 계획표로 변환했다.
* 민감 정보나 기밀 파일을 업로드하지 않았다.
* 실습 종료 후 초대 링크를 재설정하거나 삭제했다.

## 23. 핵심 정리

* 그룹 채팅은 여러 사용자와 ChatGPT 가 같은 대화 맥락을 공유하는 협업 공간입니다.
* 그룹 채팅은 단순한 AI 답변 공유 기능이 아니라, 질문과 답변과 의사결정 과정을 팀이 함께 보는 공간입니다.
* 기존 1:1 채팅에 사람을 추가하면 원본 대화가 공개되는 것이 아니라 복사본 그룹 채팅이 만들어집니다.
* 개인 메모리와 개인 맞춤 지침은 그룹 참여자에게 공유되지 않습니다.
* 하지만 그룹 채팅 안에 직접 입력한 메시지와 업로드한 파일은 참여자에게 공유됩니다.
* 초대 링크는 외부로 퍼질 수 있으므로 회의나 실습이 끝나면 재설정하거나 삭제해야 합니다.
* 실무 회의에서는 자동 응답보다 `@ChatGPT` 호출 중심으로 운영하는 것이 안정적입니다.
* 그룹 전용 맞춤 지침을 설정하면 ChatGPT 를 회의 서기, 아이디어 정리자, 비교 분석자, 실행 과제 관리자처럼 사용할 수 있습니다.
* 사용량은 방장이 아니라 ChatGPT 가 답변하는 질문자 계정 기준으로 적용됩니다.
* 그룹 채팅은 실시간 논의와 빠른 합의에 적합하고, 장기 자료 관리나 복잡한 데이터 분석에는 프로젝트나 1:1 채팅이 더 적합할 수 있습니다.
* ChatGPT 의 판단을 그대로 따르는 것이 아니라, ChatGPT 가 정리한 표와 요약을 기준으로 사람이 더 빠르게 의사결정하는 것이 핵심입니다.
